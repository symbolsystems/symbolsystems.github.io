CERT_C-MSC12 
CERT-MSC12_C (REC) Detect and remove code that has no effect or is never executed
RULES

There shall be no unreachable code in "else" block [CERT_C-MSC12-a-3]
There shall be no unreachable code after 'return', 'break', 'continue', 'goto', 'throw' statements, and after calls to functions with the 'noreturn' attribute [CERT_C-MSC12-b-3]
There shall be no unreachable code in "if/else/while/for" block [CERT_C-MSC12-c-3]
There shall be no unreachable code in switch statement [CERT_C-MSC12-d-3]
There shall be no unreachable code in 'for' loop [CERT_C-MSC12-e-3]
There shall be no unreachable code after 'if' or 'switch' statement [CERT_C-MSC12-f-3]
There shall be no unreachable code after "if" or "switch" statement inside while/for/do...while loop [CERT_C-MSC12-g-3]
Avoid switch with unreachable branches [CERT_C-MSC12-h-3]
Avoid unreachable methods [CERT_C-MSC12-i-3]
Avoid conditions that always evaluate to the same value [CERT_C-MSC12-j-3]
All non-null statements shall either have at least one side-effect however executed or cause control flow to change [CERT_C-MSC12-k-3]
