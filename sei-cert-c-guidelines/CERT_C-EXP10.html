CERT_C-EXP10 
CERT-EXP10_C (REC) Do not depend on the order of evaluation of subexpressions or the order in which side effects take place
RULES

The value of an expression shall be the same under any order of evaluation that the standard permits [CERT_C-EXP10-a-2]
Don't write code that depends on the order of evaluation of function arguments [CERT_C-EXP10-b-2]
Don't write code that depends on the order of evaluation of function designator and function arguments [CERT_C-EXP10-c-2]
Don't write code that depends on the order of evaluation of expression that involves a function call [CERT_C-EXP10-d-2]
