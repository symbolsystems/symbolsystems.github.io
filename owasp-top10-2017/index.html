Enabled Rules

  • OWASP Top 10 Most Critical Web Application Security Risks (2017) [OWASP2017]
    • OWASP A1 Injection [OWASP2017-A1]
      • Avoid passing unvalidated binary data to log methods [OWASP2017-A1-a-2]
      • Protect against command injection [OWASP2017-A1-b-1]
      • Avoid printing tainted data on the output console [OWASP2017-A1-c-1]
      • Protect against environment injection [OWASP2017-A1-d-1]
      • Exclude unsanitized user input from format strings [OWASP2017-A1-e-1]
      • Protect against SQL injection [OWASP2017-A1-f-1]
    • OWASP A10 Insufficient Logging&Monitoring [OWASP2017-A10]
      • All exceptions should be rethrown or logged with standard logger [OWASP2017-A10-a-2]
    • OWASP A2 Broken Authentication [OWASP2017-A2]
      • Do not use weak encryption functions [OWASP2017-A2-a-2]
    • OWASP A3 Sensitive Data Exposure [OWASP2017-A3]
      • Properly seed pseudorandom number generators [OWASP2017-A3-a-2]
    • OWASP A4 XML External Entities (XXE) [OWASP2017-A4]
      • Disable resolving XML external entities (XXE) in libxerces-c [OWASP2017-A4-a-2]
    • OWASP A5 Broken Access Control [OWASP2017-A5]
      • Protect against file name injection [OWASP2017-A5-a-1]
      • Observe correct revocation order while relinquishing privileges [OWASP2017-A5-b-3]
      • Ensure that privilege relinquishment is successful [OWASP2017-A5-c-3]
    • OWASP A6 Security Misconfiguration [OWASP2017-A6]
      • Where multiple handlers are provided in a single try-catch statement or function-try-block for a derived class and some or all of its bases, the handlers shall be ordered most-derived to base class [OWASP2017-A6-a-3]
      • Do not leave 'catch' blocks empty [OWASP2017-A6-b-3]
      • Properly use errno value [OWASP2017-A6-c-2]
