APSC_DV-002560 
APSC_DV-002560 The application must not be subject to input handling vulnerabilities.
RULES

Protect against environment injection [APSC_DV-002560-a-1]
Protect against file name injection [APSC_DV-002560-b-1]
Protect against SQL injection [APSC_DV-002560-c-1]
Never use unfiltered data from an untrusted user as the format parameter [APSC_DV-002560-d-1]
Avoid tainted data in array indexes [APSC_DV-002560-e-1]
Protect against integer overflow/underflow from tainted data [APSC_DV-002560-f-1]
Avoid passing unvalidated binary data to log methods [APSC_DV-002560-g-1]
Protect against command injection [APSC_DV-002560-h-1]
Avoid printing tainted data on the output console [APSC_DV-002560-i-1]
Exclude unsanitized user input from format strings [APSC_DV-002560-j-1]
Untrusted data is used as a loop boundary [APSC_DV-002560-k-1]
