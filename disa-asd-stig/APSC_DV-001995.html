APSC_DV-001995 
APSC_DV-001995 The application must not be vulnerable to race conditions.
RULES

Avoid race conditions when using fork and file descriptors [APSC_DV-001995-a-2]
Avoid race conditions while checking for the existence of a symbolic link [APSC_DV-001995-b-2]
Avoid race conditions while accessing files [APSC_DV-001995-c-2]
Use locks to prevent race conditions when modifying bit fields [APSC_DV-001995-d-2]
Do not use global variable with different locks set [APSC_DV-001995-e-2]
Avoid using thread-unsafe functions [APSC_DV-001995-f-2]
Usage of functions prone to race is not allowed [APSC_DV-001995-g-2]
Avoid using the 'vfork()' function [APSC_DV-001995-h-2]
Properly define signal handlers [APSC_DV-001995-i-2]
