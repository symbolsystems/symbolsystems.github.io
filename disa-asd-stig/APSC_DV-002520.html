APSC_DV-002520 The application must protect from canonical representation vulnerabilities.
RULES

Protect against environment injection [APSC_DV-002520-a-2]
Protect against file name injection [APSC_DV-002520-b-2]
Protect against SQL injection [APSC_DV-002520-c-2]
Never use unfiltered data from an untrusted user as the format parameter [APSC_DV-002520-d-2]
Avoid tainted data in array indexes [APSC_DV-002520-e-2]
Protect against integer overflow/underflow from tainted data [APSC_DV-002520-f-2]
Avoid passing unvalidated binary data to log methods [APSC_DV-002520-g-2]
Protect against command injection [APSC_DV-002520-h-2]
Avoid printing tainted data on the output console [APSC_DV-002520-i-2]
Exclude unsanitized user input from format strings [APSC_DV-002520-j-2]
Untrusted data is used as a loop boundary [APSC_DV-002520-k-2]
