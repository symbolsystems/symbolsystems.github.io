A source file should not contain unused type declarations [MISRAC2012-RULE_2_3-b-4]

DESCRIPTION 

"If a type is declared but not used, then it is unclear to a reviewer if the 
type is redundant or it has been left unused by mistake."[MISRA C:2012 Rule 2.3]

This rule detects cases when a source file contains
unused type declarations. The rule does not report
violations for unused type declarations in header files.
SINCE 

v9.6
NOTES 

This rule does not check the types declared inside functions.
BENEFITS 

This rule improves legibility and increases efficiency of code.
EXAMPLE 

typedef int INT;             /* Violation */

void foo( void )
{
    int i;
}
REPAIR 

typedef int INT;

void foo( void )             /* OK */
{
    INT i;
}
REFERENCES 

1. MISRA C:2012 Guidelines for the use of the C language in critical systems
Section 8: Rules, Rule 2.3
