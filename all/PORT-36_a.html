The names of identifiers should not be longer than 31 characters [PORT-36_a-3]

DESCRIPTION 

"The ISO standard requires internal identifiers to be distinct in the first 31
characters to guarantee code portability. This limitation shall not be exceeded,
even if the compiler supports it. This rule shall apply across all name spaces.
The ISO standard requires external identifiers to be distinct in the first 6
characters, regardless of case, to guarantee optimal portability.
However this limitation is particularly severe and is considered unnecessary.
The intent of this rule is to sanction a relaxation of the ISO requirement
to a degree commensurate with modern environments and it shall be confirmed
that 31 character/case significance is supported by the implementation."
[MISRA C:2004 Rule 5.1]

This rule detects when an identifier has more than 31 characters in its name.

The rule checks the following identifiers:
- variables, functions, function parameters
- classes, structures, unions, typedefs
- enumerations and enumeration constants
- namespaces
- labels
- template parameters and template parameter constants
SINCE 

2020.2
NOTES 

This rule does not check the names of conversion operators.
BENEFITS 

This rule helps you improve code portability and readability.
EXAMPLE 

void f1_45678901234567890123456789012();        // Violation
void f1_45678901234567890123456789012(){}       // Violation

void f1(int p1_45678901234567890123456789012) { // Violation
    int v1_45678901234567890123456789012;       // Violation
}
REPAIR 

You can fix the code by modifying the names of identifiers.

void f1_4567890123456789012345678901();        // OK
void f1_4567890123456789012345678901(){}       // OK

void f1(int p1_4567890123456789012345678901) { // OK
    int v1_4567890123456789012345678901;       // OK
}
