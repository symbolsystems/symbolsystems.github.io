An assignment operator shall assign all data members [INIT-11-2]

DESCRIPTION 

"A class may contain many data members as well as exist within an inheritance
hierarchy. Hence the assignment operator must assign all members, including
those in base classes."
The rule reports a violation if the assignment operator does not assign
all data members. Data bases are checked by rule MRM-43.
The data from a class can be assigned directly inside body of assignment
operator or inside body of function called from assignment operator
(three levels of nested function's calls are checked).
See also: MRM-41, MRM-43
NOTES 

The rule assumes that member variable might be initialized 
by passing its non-const pointer to an external function.
BENEFITS 

"Ensure data members are properly handled under assignment."
EXAMPLE 

class C
{
public:
    C& operator=(const C&);
private:
    int mem_a;
    int mem_b;
    int mem_c;
    int mem_d;
};

C& C::operator=(const C&) // Violation - mem_a, mem_b, mem_c, mem_d - not assigned
{
}
REPAIR 

class C
{
public:
    C& operator=(const C&);
    void init();
    void init2();
    void getPtr(int *);
private:
    int mem_a;  // assigned inside operator= body
    int mem_b;  // assigned inside function 'init' called from operator=
    int mem_c;  // assigned inside function 'init2' called from 'init'
    int mem_d;  // its non-const pointer is passed to an external function.
};

void C::init()
{
    mem_b = 2;
    init2();
}

void C::init2()
{
    mem_c = 2;
}

C& C::operator=(const C&) // OK - all members are assigned
{
    mem_a = 1;
    init();
    getPtr(&mem_d);
}
