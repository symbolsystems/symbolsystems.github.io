Every switch statement will have at least two cases and a potential default [HICPP-6_1_4-a-3]

DESCRIPTION 

The rule reports a violation if a 'switch' statement has less than two 'case'
labels. 
A 'switch' statement with only one 'case' label can be replaced with 'if'
statement.
A 'switch' statement with no 'case' labels is unnecessary and can be removed.

See also: CODSTA-41, MISRA2004-15_5
SINCE 

v7.1
BENEFITS 

The rule enforces a simple and consistent structure on to the 'switch' statement.
EXAMPLE 

void foo(int i, int j) 
{
    switch(i){      // Violation
    default:
        i = i + 10;
        break;
    }
    
    switch(j){      // Violation
    case 1:
    	j = j + 5;
        break;
    default:
        j = j + 10;
        break;
    }
}
REPAIR 

void foo(int i, int j, int k) 
{
    // OK - switch statement is unnecessary
    i = i + 10;
    
    // OK - switch statement is replaced with if-else construct
    if(j == 1){       
    	j = j + 5;
    } else {
        j = j + 10;
    }  
    
    // OK - switch statement with two case labels
    switch(k){
    case 0:
    case 1:
    	k = k + 5;
        break;
    default:
        k = k + 10;
        break;
    }
}

1. AUTOSAR C++14 Coding Guidelines

   from the default label

   distinct from the default label
