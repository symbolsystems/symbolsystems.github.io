Do not assume that members are initialized in any special order in constructors [INIT-13-3]

DESCRIPTION 

Members of a class are initialized in the order in which they are declared,
not the order in which they appear in the initialization list. 
The member variable used in initializer of another member variable can be not
initialized.

The rule reports a violation if member variable is used to initialize other 
member variable in constructor initialization list.

See also: INIT-10
EXCEPTIONS 

Variables used in sizeof are excluded. The sizeof does not use a value
of the variable, but the type, so it does not matter that variable might
be not initialized.
There is allowed taking an address of variable.
BENEFITS 

Complying with this rule leads to safer and more readable code.
EXAMPLE 

class X {
public:
    X( int y );
private:
    int i;
    int j;
};

inline X::X( int y ) : j( y ), i( j ) {}    // Violation
REPAIR 

class X {
public:
    X( int y );
private:
    int i;
    int j;
};

inline X::X( int y ) : i( y ), j( y ) {}    // OK
