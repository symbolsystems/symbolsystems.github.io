There should be no space between an increment/decrement operator (++/--) and its operand in macro definition [FORMAT-46-3]

DESCRIPTION 

The rule reports a violation if there is space, tab, comment or new line
character between an increment/decrement operator (++/--) and its operand.
Both prefix and postfix operators are checked.

See also: FORMAT-17, FORMAT-18, FORMAT-21, FORMAT-22, FORMAT-45
SINCE 

v9.4
BENEFITS 

The rule promotes continuity in the relationship between the operators
and the operands on which they act.
DRAWBACKS 

It might be not possible to correctly recognize the operator version
(increment vs decrement) or recognize free text as the operator which
might result in false negative or false positive.
EXAMPLE 

#define M1(a) -- a /* Violation */
#define M2 a ++ /* Violation */
#define M3(a, b) \
    do { \
        ++/* Violation */a; \
        a \
        ++; /* Violation */ \
    } while( b() -- ); /* Violation */
#define M4 Just a simple text -- Violation
REPAIR 

#define M1(a) --a
#define M2 a++
#define M3(a, b) \
    do { \
        ++a; \
        a++; \
    } while( b()-- );
