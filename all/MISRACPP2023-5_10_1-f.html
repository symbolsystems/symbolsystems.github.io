A macro identifier should have an appropriate form [MISRACPP2023-5_10_1-f-2]

DESCRIPTION 

"Each identifier that contains a double underscore '__' or begins
with an underscore followed by an uppercase letter is reserved
to the implementation for any use." [ISO/IEC 14882:2017]

Macro names should be distinct from other identifiers and should
not collide with keywords or alternative representations, as this
may result in undefined behavior.

This rule detects when a macro name:
- contains a double underscore, or
- starts with an underscore followed by an uppercase letter, or
- contains characters other than uppercase letters, numbers, and underscores.
SINCE 

2023.2
BENEFITS 

This rule helps you improve code readability and prevent undefined behavior.
EXAMPLE 

#define pi 3.141592653    // Violation
#define _MAC 10           // Violation
REPAIR 

#define PI 3.141592653    // OK
#define MAC 10            // OK
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   5.10 Identifiers

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
