The identifiers va_list, va_arg, va_start, va_end, va_copy should not be used [MISRACPP2023-21_10_1-a-2]

DESCRIPTION 

"The standard header file <stdarg.h> shall not be #included, and none of the
features that are specified as being provided by <stdarg.h> shall be used.

The Standard lists many instances of undefined behaviour associated with
the features of <stdarg.h>, including:
- va_end not being used prior to end of a function in which va_start was used;
- va_arg being used in different functions on the same va_list;
- the type of an argument not being compatible with the type specified
  to va_arg." [MISRA C:2012 Rule 17.1 with Amendment 3]

The rule reports violations on identifiers named va_list, va_arg, va_start,
va_end or va_copy. The rule does not check if these identifiers are macros
from stdarg.h header, so violation will be reported even if <stdarg.h> is not
included or the identifiers are defined by the user.
SINCE 

v9.5
NOTES 

This rule is designed for C99 standard, for code conforming to C90 standard
please use CODSTA-136_b rule.
BENEFITS 

The rule prevents undefined behaviours.
EXAMPLE 

#include <stdarg.h>

void foo1(int i, ...){
  va_list ap;          /* Violation */
  
  va_start(ap, i);     /* Violation */
  
  va_arg(ap, int);     /* Violation */
  
  va_end(ap);          /* Violation */
}
REPAIR 

Do not use identifiers va_list, va_arg, va_start, va_end and va_copy.
