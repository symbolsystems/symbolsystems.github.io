The value of a complex expression of floating type should not be cast to a wider floating type [MISRA2004-10_4-3]

DESCRIPTION 

"The value of a complex expression of floating type may only be cast
to a narrower floating type." [MISRA-C:2004]

"A cast applied to the result of an expression does not change the type in which
the expression is evaluated, which may be contrary to developer expectations."
[MISRA C++:2008]

This rule detects when a complex expression of floating type is explicitly cast
to a wider floating type.
NOTES 

"The term 'complex expression' is defined to mean any expression that is not:
- a constant expression
- an lvalue (i.e. an object)
- the return value of a function" [MISRA-C:2004]
BENEFITS 

"Complex expressions are often a source of confusion and it is therefore
wise to be cautious. In order to comply with these rules, it may be necessary
to use a temporary variable and introduce an extra statement." [MISRA-C:2004]
EXAMPLE 

void f1( ) {
    float f32a, f32b;
    double f64a;

    f64a = (double)(f32a/f32b);   /* Violation */
}
REPAIR 

void f1( ) {
    float f32a, f32b;
    double f64a;

    f64a = (double)f32a/f32b;     /* OK */
}
