Avoid conversions of constant values to a narrower type [PORT-28-3]

DESCRIPTION 

The rule reports a violation if an integer or enumeration constant is passed
to a type that is not enough to hold its value.
See also: MISRA-043, MISRA2004-10_1_b, MISRA2004-10_1_d, MISRA2004-10_2_a,
MISRA2004-10_2_b, PORT-27
SINCE 

v9.2
NOTES 

The rule does not report violations on complex constant expressions.
BENEFITS 

The rule prevents a loss of data.
EXAMPLE 

void foo1(){
    unsigned char c = 256;  // Violation
    short s = 32768;        // Violation
    int i = 2147483648;     // Violation
}
REPAIR 

void foo1(){
    unsigned short c = 256;       // OK
    int s = 32768;                // OK
    unsigned int i = 2147483648;  // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A4-7-1. An integer expression shall not lead to data loss
