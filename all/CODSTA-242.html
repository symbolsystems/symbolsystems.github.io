The string handling functions from <cstring>, <cstdlib>, <cwchar> and <cinttypes> shall not be used [CODSTA-242-3]

DESCRIPTION 

String handling functions can be used incorrectly, which can result
in undefined behavior when they read or write outside the bounds
of an object passed as a parameter.
In all cases, the functionality provided by these functions
can be obtained by using other C++ Standard Library functions
which are less error-prone.

This rule detects when functions with the names: 'strcat', 'strchr', 'strcmp',
'strcoll', 'strcpy', 'strcspn', 'strerror', 'strlen', 'strncat', 'strncmp',
'strncpy', 'strpbrk', 'strrchr', 'strspn', 'strstr', 'strtok', 'strxfrm',
'strtol', 'strtoll', 'strtoul', 'strtoull', 'strtod', 'strtof', 'strtold',
'fgetwc', 'fputwc', 'wcstol', 'wcstoll', 'wcstoul', 'wcstoull', 'wcstod',
'wcstof', 'wcstold', 'strtoumax', 'strtoimax', 'wcstoumax', 'wcstoimax'
that are defined with extern C linkage (in an external library or locally by the
user) are used (called or their addresses are taken).

Additionally, it detects function-like macros with the above names, when one
of the headers <cstring>, <cstdlib>, <cwchar>, <cinttypes>, <string.h>,
<stdlib.h>, <wchar.h> or <inttypes.h> is included in a translation unit.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

#include <cstring>

void f1( char * buffer, size_t bufferSize )
{
    char const * str = "Message";

    if ( ( strlen( str ) + 5u ) < bufferSize )      // Violation
    {
        ( void ) strcpy( buffer, str );             // Violation
    }
}
REPAIR 

#include <string>

void f1( char * buffer, size_t bufferSize )
{
    std::string_view str = "Msg";

    if ( str.size() + 1u < bufferSize )             // OK
    {
        str.copy( buffer, str.size() );             // OK
        buffer[ str.size() ] = 0;
    }
}
