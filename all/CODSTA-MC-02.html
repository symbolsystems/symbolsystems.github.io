A declaration of an object should contain at most one explicit alignment specifier [CODSTA-MC-02-3]

DESCRIPTION 

"When multiple alignment specifiers occur in a declaration, the effective
alignment requirement is the strictest specified alignment."[ISO/IEC 9899:2011]

"If separate conditions require different minimum permitted alignments
for an object, they should be combined explicitly by the expression controlling
the specifier. Otherwise, the conflicting alignment specifiers risk obscuring
the intent of the declaration from a human reviewer." [MISRA C:2012 Amendment 3]

The rule detects declarations of variables with more than one alignment
specifier. The rule checks the standard alignment specifiers 'alignas'
and '_Alignas'.
SINCE 

2022.2
BENEFITS 

This rule helps you improve code readability and maintainability.
EXAMPLE 

#include <stdalign.h>

alignas (16) alignas (8) int g;                              // Violation
REPAIR 

You can fix the code by removing the redundant alignment specifier.

#include <stdalign.h>

alignas (8) int g;                                           // OK
