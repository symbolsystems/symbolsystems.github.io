Avoid function calls with incorrect argument order [PB-62-3]

DESCRIPTION 

This rule detects function calls that pass arguments in the incorrect order.
The rule reports a violation if the names of two arguments and their
corresponding parameters are reversed. This case is suspicious and may be
caused by incorrect order of arguments in a function call.
SINCE 

v9.5
BENEFITS 

Following this rule prevents mistakes. If an argument in a function call
is passed to the wrong parameter, then it may lead to incorrect behavior.
This problem typically occurs when the programmer makes a typo or a copy/paste
error.
EXAMPLE 

void foo1(int* first, int* second) {}

void foo_use(){
    int* first;
    int* second;

    // The names of parameters and arguments suggest the reverse
    // order of arguments in the function call
    foo1(second, first); // Violation
}
REPAIR 

void foo1(int* first, int* second) {}

void foo_use(){
    int* first;
    int* second;
    
    foo1(first, second);  // OK
}
