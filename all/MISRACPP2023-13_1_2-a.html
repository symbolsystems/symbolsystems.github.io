A base class shall not be both virtual and non-virtual in the same hierarchy [MISRACPP2023-13_1_2-a-2]

DESCRIPTION 

A base class shall not be both virtual and non-virtual
in the same hierarchy.
SINCE 

v7.1
BENEFITS 

Hierarchy becomes easier to comprehend and use.
EXAMPLE 

class A {};
class B : virtual A {};
class C : B, A {};           // Violation
REPAIR 

class A {};
class B : virtual A {};
class C : B, virtual A {};   // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M10-1-3. An accessible base class shall not be both virtual and non-virtual
   in the same hierarchy

2. General Principles of Software Validation;
   Final Guidance for Industry and FDA Staff
   Document issued on: January 11, 2002

3. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.10 Classes, AV Rule 89

4. MISRA C++:2008 - Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 10, Rule 10-1-3

5. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 13.1.2 An accessible base class shall not be both virtual and non-virtual in the same hierarchy
