Function types shall have named parameters [MISRA2012-RULE-8_2_b-2]

DESCRIPTION 

"This rule requires that names be specified for all the parameters
in a declaration. The parameter names can provide useful information
regarding the function interface and a mismatch between a declaration
and definition might be indicative of a programming error."[MISRA C:2012]

See also: MISRA2004-16_3, CODSTA-159
NOTES 

The rule checks function types other than in function declarations
and definitions.
BENEFITS 

The rule improves readability and clarity of code.
EXAMPLE 

typedef void foot(int*);        /* Violation */

void foo1(void (*fp1)(int)) {   /* Violation */
  void (*fp2)(float);           /* Violation */
  fp1 = (void (*)(int))fp2;     /* Violation */
}
REPAIR 

typedef void foot(int* p1);       /* OK */

void foo1(void (*fp1)(int p)) {   /* OK */
  void (*fp2)(float f);           /* OK */
  fp1 = (void (*)(int p))fp2;     /* OK */
}

/* Examples not reported by this rule */
void foo2(int);  /* OK - See rule MISRA2004-16_3 */
void foo3();     /* OK - See rule CODSTA-159 */
