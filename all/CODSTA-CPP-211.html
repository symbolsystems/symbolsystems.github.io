There shall be no conversion from type bool [CODSTA-CPP-211-3]

DESCRIPTION 

This rule reports if bool is converted to any other type
by an implicit or explicit conversion.
SINCE 

2023.2
EXCEPTIONS 

The rule does not report on equality operators where both operands have
type bool.

An explicit cast to class T of the following forms:
* T { bool },
* T ( bool ),
* static_cast< T >( bool ),
* ( T )bool
is permitted if T has a converting constructor with a parameter of type bool.

A value of type bool may be assigned to a bit-field of length 1.
BENEFITS 

The rule helps you prevent potential errors and avoid confusion.
EXAMPLE 

bool b1 = true;
bool b2 = false;

void f()
{
    if (b1 & b2) {}       // Violation - b1 and b2 converted to int
    if (b1 < true) {}    // Violation - b1 and 'true' converted to int
    static_cast<int>(b1); // Violation - b1 converted to int
}
REPAIR 

struct A
{
  explicit A( bool );
};

bool b1 = true;
bool b2 = false;

void f()
{
    if (b1 && b2) {}      // OK - no conversion
    if (b1 != true) {}    // OK - equality operator 
    static_cast<A>(b1);   // OK - call to converting constructor
}
