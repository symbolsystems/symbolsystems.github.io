Only those escape sequences that are defined in ISO/IEC 14882:2003 shall be used [CODSTA-CPP-60-3]

DESCRIPTION 

"The use of an undefined escape sequence leads to undefined behaviour.
The defined escape sequences (ISO/IEC 14882:2003) are: \n, \t, \v, \b, \r, \f,
\a, \\, \?, \', \", \<Octal Number>, \x<Hexadecimal Number>" [MISRA C++:2008]

The rule detects when a character constant or a string constant contains the
'\' character that is not used to define an escape sequence.

See also: NAMING-48
SINCE 

v7.2
NOTES 

The rule does not detect '\' characters used in #include directives.
These characters should not be used in any case in #include directives
and are disallowed by another rule.
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

void f1( char c, const char* s )
{
    c = '\k';           // Violation
    s = "dir1\dir2";    // Violation
    s = "\u1234";       // Violation
}
REPAIR 

/* Examples of correct code */
void f1( char c, const char* s )
{
    c = '\t';           // OK
    s = "name \n";      // OK
}
