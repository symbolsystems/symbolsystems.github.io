Storage type modifiers shall be associated with the type, not the variable or the function [MISRA2012-RULE-1_5_b-2]

DESCRIPTION 

"The placement of a storage-class specifier other than at the beginning
of the declaration specifiers in a declaration is an obsolescent feature."
[ISO/IEC 9899:2011]

The C/C++ Standard allows any order of specifiers in a declaration.
If a specifier (auto, extern, static, register) appears in a declaration,
it should not be placed directly before variable or function name.

This rule detects when one from the following specifiers: 'auto', 'extern',
'static', 'register' is placed directly before a name in the variable
or the function declaration.
NOTES 

The rule does not report violations when specifiers are not connected
directly with names of variables or functions:

int static const var = 0;   /* No Violation */
BENEFITS 

This rule improves the readability of code.
EXAMPLE 

int static i;       // Violation
void extern f1();   // Violation
REPAIR 

static int i;       // OK
extern void f1();   // OK
