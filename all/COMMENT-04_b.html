Document functions in comments that precede function declarations [COMMENT-04_b-3]

DESCRIPTION 

"Assumptions (limitations) made by functions should be documented in
the functionâ€™s preamble. . .  Maintenance efforts become very difficult if
the assumptions (limitations) upon which functions are built are unknown."
[JOINT STRIKE FIGHTER AV Rule 134]

This rule detects function declarations that are not preceded by a comment.
NOTES 

The rule only checks function declarations that are not function definitions.
The rule does not analyze the content of comments.
BENEFITS 

The rule helps you improve code readability and maintainability.
EXAMPLE 

In the following example, there are any comments before the function
declarations.

void foo();                                        // Violation
                                              
int delta(int a, int b, int c);                    // Violation 

// returns the value of delta
int delta(int a, int b, int c)
{
  return (b*b)-(4*a*c);
}
REPAIR 

To fix the code, precede the function declarations with a comment
that documents the function.

// just a foo function declaration
void foo();                                        // OK

// returns the value of delta
int delta(int a, int b, int c);                    // OK

// returns the value of delta
int delta(int a, int b, int c)
{
  return (b*b)-(4*a*c);
}
