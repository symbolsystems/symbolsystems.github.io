The '_Generic' operator should not be used [CODSTA-216-3]

DESCRIPTION 

"Updates to the C Standard have introduced new language features. The following
shall not be used:
- The _Generic operator;" [MISRA C:2012 Amendment 2]

The rule detects when the '_Generic' operator is used.
SINCE 

2020.1
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

#define GEN(T) _Generic( (T), int: 1, long: 2, default: 0)

int f1()
{
  int x = _Generic( (100), int: 1, long: 2, default: 0);  /* Violation */
  int y = GEN(100);                                       /* Violation */
}
REPAIR 

Do not use the '_Generic' operator.
