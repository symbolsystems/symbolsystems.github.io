Match the filename in a #include directive to the one on the filesystem [PREPROC-20-3]

DESCRIPTION 

"Some operating systems have case insensitive filesystems. Code initially
developed on such a system may not compile successfully when ported
to a case sensitive filesystem." [HIC++ Coding Standard]

This rule detects inconsistency in the letter case between the header file used
in the '#include' directive and the real name of this header in a filesystem.
SINCE 

v10.4
NOTES 

If a translation unit contains two or more identical '#include' directives, and
at least one of them is inconsistent with the letter case of the file it
includes, then rule violation is reported on each of the directives (including
the correctly cased ones).
BENEFITS 

This rule helps you improve portability and readability of code.
EXAMPLE 

// file.h
int i;

// file.cpp
#include "File.h"      // Violation

#include <CStdDef>     // Violation
REPAIR 

// file.h
int i;

// file.cpp
#include "file.h"      // OK

#include <cstddef>     // OK

1. High Integrity C++ Coding Standard
