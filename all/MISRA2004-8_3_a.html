Use identical types in declaration and definition [MISRA2004-8_3_a-3]

DESCRIPTION 

For each function parameter the type given in the declaration and definition
shall be identical, and the return types shall also be identical.
BENEFITS 

Rule prevents undefined behaviour and improves readability.
EXAMPLE 

typedef int MyInt;
typedef MyInt MyInt2;

int foo1( );        /* Violation */
MyInt foo1( ) {
    return 0;
}

MyInt2 foo2( );     /* Violation */
int foo2( ) {
    return 0;
}
REPAIR 

typedef int MyInt;
typedef MyInt MyInt2;

int foo1( );        /* OK */
int foo1( ) {
    return 0;
}

MyInt2 foo2( );     /* OK */
MyInt2 foo2( ) {
    return 0;
}
REFERENCES 

1. MISRA-C:2004 Guidelines for the use of the C language in critical systems
   Chapter 6, Section 8

2. Origin: Misra Guidelines - Rule 72
