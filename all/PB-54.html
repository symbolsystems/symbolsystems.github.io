Do not cast from or to incomplete class at the point of casting [PB-54-2]

DESCRIPTION 

The operand of a cast using the cast notation can be “pointer to incomplete
class type”. The destination type of a cast using the cast notation can be
“pointer to incomplete class type”. In such cases, even if there is a
inheritance relationship between the source and destination classes, whether
the static_cast or reinterpret_cast interpretation is used is unspecified.

The result of such cast is implementation defined.

Rule checks C-style casts and reports a violation if operand of cast or
destination type is pointer or reference to class or struct type which is not
complete at the point of casting.
SINCE 

v9.4.1
EXCEPTIONS 

For templates rule checks only if types are complete at the end of translation
unit - a violation is reported if only forward declaration is provided.
BENEFITS 

Rule prevents implementation defined behavior.
EXAMPLE 

class X;
class Y;
class Z {};
void foo(X* x, Y* y, Z* z)
{
    (X*)z; // Violation: cast to incomplete type
    (Z*)y; // Violation: cast from incomplete type (at the point of casting)
    (Y*)x; // Violation: cast from/to incomplete type
};
class Y {};
REPAIR 

// Provide complete type declaration or use C++-style casts
class X;
class Y {};
class Z {};
void foo(X* x, Y* y, Z* z)
{
    reinterpret_cast<X*>(z);    // OK
    (Z*)y;                      // OK
    reinterpret_cast<Y*>(x);    // OK
};
REFERENCES 

1. C++ standard (ISO/IEC 14882:2003)
   Section 5.4 Explicit type conversion (cast notation), point 6
