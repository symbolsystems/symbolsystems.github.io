Every switch statement shall have at least two switch-clauses [OPT-39-3]

DESCRIPTION 

"Every switch statement shall have at least two switch-clauses.
A switch statement with a single path is redundant and may be indicative
of a programming error."

A switch-clause in this rule is defined as a group of 'case' and 'default'
labels followed by at least one statement.

See also: CODSTA-35, CODSTA-119, MISRA2004-15_2, MISRA2004-15_5, OPT-21
SINCE 

v9.5
BENEFITS 

Prevents redundant and possibly incorrect code.
EXAMPLE 

void example ( int i )
{
  switch ( i )                        /* Violation */
  {
    case 0:
    case 1:
      /* A switch-clause */
      i = i + 1;
      break;
    default:
      /* There are no statements here so this is not counted
       * as a switch-clause */
  }

  switch ( i )                        /* Violation */
  {
    case 1:
    default:
      /* A switch-clause */
      i = i + 1;
      break;
  }
}
REPAIR 

void repair ( int i )
{
  /* The first switch was rewritten into an 'if' */
  if ( ( i == 0 ) || ( i == 1 ) )
  {
    i = i + 1;
  }

  /* The second switch has been modified to contain two switch-clauses */
  switch ( i )                        /* Compliant */
  {
    case 1:
      /* A switch-clause */
      break;
    default:
      /* A switch-clause */
      i = i + 1;
      break;
  }
}
