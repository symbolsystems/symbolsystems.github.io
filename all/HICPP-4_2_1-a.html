Apply the U suffix to literals used in a context that requires an unsigned integral expression [HICPP-4_2_1-a-3]

DESCRIPTION 

"If a literal is used to initialize a variable of an unsigned type or with an
operand of unsigned type in a binary operation, the U suffix should be
appended to the literal to circumvent an implicit conversion and make the
intent explicit." [HIC++ v4.0, Rule 4.2.1]
SINCE 

v10.4
BENEFITS 

Improves readability and removes confusion if an improper value (negative) is
assigned.
EXAMPLE 

#include <cstdint>
void foo() {
    uint32_t u(0); // Violation
    u += 2;        // Violation
    u += 2U;
}
REPAIR 

#include <cstdint>
void foo() {
    uint32_t u(0U); // OK
    u += 2U;        // OK
    u += 2U;
}
REFERENCES 

1. High Integrity C++ Coding Standard
   4.2.1. Ensure that the U suffix is applied to a literal used in a context
   requiring an unsigned integral expression
