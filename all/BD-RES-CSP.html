Avoid cyclic shared_ptr references [BD-RES-CSP-2]

DESCRIPTION 
"A cyclic structure of std::shared_ptr results in reference counting mechanism never
dropping to zero, which prevents from pointed object deallocation. Breaking such
cycles is done using std::weak_ptr which must be converted to std::shared_ptr in order
to access the referenced object." [AUTOSAR A20-8-7]

This rules detects circular references between two shared pointers.
SINCE 
v10.4.2
NOTES 
N/A
SECURITY RELEVANCE 
When shared pointers form a circular reference, the destructors of underlying shared pointers
are never called, which may result in memory leaks.
PARAMETERS 
N/A
BENEFITS 
This rule helps you prevent memory leaks (see SECURITY RELEVANCE).
DRAWBACKS 
N/A
EXAMPLE 
Here is an example that triggers a violation.

#include <memory>

template <template <typename> class T, typename U>
struct Base
{
    T<U> sp;
};

template <typename T>
using Shared = Base<std::shared_ptr, T>;

struct SBarSFoo;
struct SFooSBar : public Shared<SBarSFoo> {};
struct SBarSFoo : public Shared<SFooSBar> {};

int main()
{
    std::shared_ptr<SFooSBar> f = std::make_shared<SFooSBar>();
    std::shared_ptr<SBarSFoo> b = std::make_shared<SBarSFoo>();
    f->sp = b;
    b->sp = f;    // VIOLATION
    return 0;
}
REPAIR 
In the following code, std::weak_ptr pointer is used instead of std::shared_ptr.

#include <memory>

template <template <typename> class T, typename U>
struct Base
{
    T<U> sp;
};

template <typename T>
using Shared = Base<std::shared_ptr, T>;

template <typename T>
using Weak = Base<std::weak_ptr, T>;

struct WBarSFoo;
struct SFooWBar : public Shared<WBarSFoo> {};
struct WBarSFoo : public Weak<SFooWBar> {};

int main()
{
    std::shared_ptr<SFooWBar> f2 = std::make_shared<SFooWBar>();
    std::shared_ptr<WBarSFoo> b2 = std::make_shared<WBarSFoo>();
    f2->sp = b2;
    b2->sp = f2; // NO VIOLATION
    return 0;
}
REFERENCES 
AUTOSAR C++14 Coding Guidelines
Rule A20-8-7 A std::weak_ptr shall be used to represent temporary shared ownership.
