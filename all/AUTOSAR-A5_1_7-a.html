A lambda shall not be an operand to typeid [AUTOSAR-A5_1_7-a-2]

DESCRIPTION 

"Each lambda expression has a different unique underlying type,
and therefore the type is not to be used as a decltype or typeid argument."
[AUTOSAR A5-1-7]

The rule detects when an expression of lambda type is used as an operand
of the 'typeid' operator.
SINCE 

v10.4
BENEFITS 

This rule helps you obtaining unpredictable results and to comply with some
coding standards.
EXAMPLE 

In the following example, a lambda is used as an operand of typeid.

#include <typeinfo>

void foo() {
    auto lmdb = []() -> int { return 0; };
    typeid(lmdb);                              // Violation
}
REPAIR 

Do not pass a lambda as an operand of typeid.

#include <functional>

void foo() {
    std::function<int()> f = []() { return 0; };
    typeid(f);                                 // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A5-1-7. A lambda shall not be an operand to decltype or typeid
