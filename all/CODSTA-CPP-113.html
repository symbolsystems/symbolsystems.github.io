The 'std::locale::global' function should not be used [CODSTA-CPP-113-3]

DESCRIPTION 

Using the 'std::locale::global' function may introduce data races
with functions that use the locale.

The rule detects when the 'std::locale::global' function is called
or when its address is taken.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent unexpected behavior and data races.
EXAMPLE 

#include <locale>

void f1 (void)
{
    std::locale foo;
    foo.global(std::locale(""));  // Violation
}
REPAIR 

Do not use the 'std::locale::global' function.
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 25.5.1 The setlocale and std::locale::global functions shall not be called
