Use != instead of < to compare iterators [STL-38-3]

DESCRIPTION 

"Use the most generic and abstract means to implement a piece of functionality:
Use != instead of < to compare iterators. Using != is more general
and so applies to a larger class of objects."
[Sutter, Alexandrescu: C++ Coding Standards]
BENEFITS 

"Using < asks for ordering, and only random-access iterators can implement
'operator<'. If you use 'operator!=' your code will 'port' easily to other
kinds of iterators, such as forward and bidirectional iterators."
[Sutter, Alexandrescu: C++ Coding Standards]
EXAMPLE 

#include <vector>
using namespace std;

void foo( ) {
    vector<int> coll;
    vector<int> moll;

    vector<int>::iterator c_pos = coll.begin( );
    vector<int>::iterator m_pos = moll.begin( );
    if (c_pos < m_pos) {}                       // Violation
}
REPAIR 

#include <vector>
using namespace std;

void foo( ) {
    vector<int> coll;
    vector<int> moll;

    vector<int>::iterator c_pos = coll.begin( );
    vector<int>::iterator m_pos = moll.begin( );
    if (c_pos != m_pos) {}                      // OK
}
