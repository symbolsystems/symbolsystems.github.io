Member variable names in derived classes should not be the same as member variable names in base classes [CODSTA-CPP-110-3]

DESCRIPTION 

If a member variable uses the same name as a member variable from its base
class, then it hides the member variable in the base class, which may cause
confusion for developers.

The rule detects when a member variable reuses a member variable name from its
base class.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid errors and confusion.
EXAMPLE 

class Base {
public:
    int base_member;
};

class Derived : public Base {
public:
    int base_member;                // Violation
};
REPAIR 

class Base {
public:
    int base_member;
};

class Derived : public Base {
public:
    int derived_member;             // OK
};
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 6.4.1 A variable declared in an inner scope shall not hide a variable declared in an outer scope
