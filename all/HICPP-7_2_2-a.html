Initialize all, only the first, or none of the enumerators in an enumeration [HICPP-7_2_2-a-3]

DESCRIPTION 

Explicitly initializing some enumerators in an enumeration, while relying on
the compiler to initialize the rest, is prone to error. During maintenance, for
example, introducing implicitly-initialized enumerators that have the same
value as an existing enumerator that was initialized explicitly may result in
an error.
SINCE 

v10.4
EXCEPTIONS 

Defining additional enumerators after all others have been defined is actually
recommended when using an enumeration to define the size of, and to index, an
array. In this case, it is acceptable to define three additional enumerators
that represent the first and last elements, as well as the size of the array.

#include <cstdint>
// ok
enum Team : int32_t {
  Anna,
  Bob,
  Joe,
  John,
  Sandra,
  Tim,
  Team_First = Anna,
  Team_Last = Tim,
  Team_Size
};
int32_t performance[Team_Size];
BENEFITS 

Enumerations are always initialized to the correct value
EXAMPLE 

#include <cstdint>
enum E : int32_t {
    RED,
    ORANGE = 2, // Violation
    YELLOW
};
REPAIR 

#include <cstdint>
// OK
enum E : int32_t { RED = 1, ORANGE = 2, YELLOW = 3 };
REFERENCES 

1. High Integrity C++ Coding Standard
