A function type shall not be type qualified [MISRA2012-RULE-17_13-2]

DESCRIPTION 

"If the specification of a function type includes any type qualifiers,
the behavior is undefined." [ISO/IEC 9899:2011]

This rule detects when the 'const', 'volatile,' or 'restrict' qualifier is
specified for a function type.
SINCE 

2021.1
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

typedef int func( void );

typedef const func func_t;         // Violation

void f1( void ) {
    volatile func* pfunc;          // Violation
}
REPAIR 

To fix the code, remove the 'const', 'volatile', and 'restrict' qualifiers
specified for the function type.

typedef const int func( void );   // OK

typedef func func_t;              // OK

void f1( void ) {
    func* volatile pfunc;         // OK
}
