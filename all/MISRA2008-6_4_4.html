A switch-label shall only be used when the most closely-enclosing compound statement is the body of a switch statement [MISRA2008-6_4_4-2]

DESCRIPTION 

"The Standard permits a switch label, i.e. a case label or default label,
to be placed before any statement contained in the body of a switch statement,
potentially leading to unstructured code. In order to prevent this,
a switch label shall only appear at the outermost level of the compound
statement forming the body of a switch statement." [MISRA C:2012 Rule 16.2]

The rule detects nested switch-labels.
BENEFITS 

This rule helps you improve readability and maintainability of code.
EXAMPLE 

void foo( int a, int b )
{
    switch(a)
    {
        case 1:
        {
            case 6:       /* Violation */
                ;
            default:      /* Violation */
                break;
        }
            break;

        case 2:
            if (b == 1)
            {
                 case 3:  /* Violation */
                 break;
            }
            break;
    }
}
REPAIR 

Do not use nested 'case'/'default' statements.
