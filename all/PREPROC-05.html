In a macro function, use parentheses before and after multiplication or division [PREPROC-05-2]

DESCRIPTION 

This rule checks whether parentheses are used before and after division
or multiplication in function-like macros.
EXCEPTIONS 

The rule does not report violation if as an operand of division
or multiplication is used a number constant or other division
or multiplication.
BENEFITS 

Improves the readability of code and ensures operations order.
EXAMPLE 

#define DIVISION_1(x)  ( x / (x))    // Violation
#define DIVISION_2(x)  ((x) / 2 + 5) // Violation
#define DIVISION_3(x)  (256 / x)     // Violation
#define MULTI_1(x)  (x * (x))        // Violation
#define MULTI_2(x)  ((x) * 2 + 5)    // Violation
#define MULTI_3(x)  (256 * x)        // Violation
REPAIR 

#define DIVISION_1(x)  ( (x) / (x))     // OK
#define DIVISION_2a(x)  ((x) / (2 + 5)) // OK
#define DIVISION_2b(x)  (((x) / 2) + 5) // OK
#define DIVISION_3(x)  (256 / (x))      // OK
#define MULTI_1(x)  ( (x) * (x))        // OK
#define MULTI_2a(x)  ((x) * (2 + 5))    // OK
#define MULTI_2b(x)  (((x) * 2) + 5)    // OK
#define MULTI_3(x)  (256 * (x))         // OK
