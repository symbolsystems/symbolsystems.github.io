Every 'switch' statement shall have a 'default' label [MISRA2012-RULE-16_4_a-2]

DESCRIPTION 

"Every switch statement shall have a default label.
The requirement for a default label is defensive programming.
Any statements following the default label are intended to take
some appropriate action. " [MISRA C:2012 Rule 16.4]
NOTES 

This rule only checks if 'default' is present. It does not impose any
requirements on location of the 'default' nor the content of the 'default'.
BENEFITS 

Rule implements defensive programming standards which limit possibility of
erroneous code.
EXAMPLE 

enum E {
  E1, E2
};
void foo ( int x, enum E e )
{
  switch ( x )  /* Violation: missing 'default' */
  {
    case 0:
      // ...
      break;
  }
  switch ( e )  /* Violation: missing 'default' */
  {
    case E1:
      // ...
      break;
    case E2:
      // ...
      break;
  }
}
REPAIR 

enum E {
  E1, E2
};
void foo ( int x, enum E e )
{
  switch ( x )  /* OK: 'default' present */
  {
    case 0:
      // ...
      break;
    default:
      // ...
      break;
  }
  switch ( e )  /* OK: 'default' present */
  {
    case E1:
      // ...
      break;
    case E2:
      // ...
      break;
    default:
      // ...
      break;
  }
}
