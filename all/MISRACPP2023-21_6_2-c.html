The 'std::unique_ptr::release' function should not be used [MISRACPP2023-21_6_2-c-2]

DESCRIPTION 

The dynamic memory should be managed automatically using facilities
such as 'std::make_unique' or 'std::vector'.

This rule detects when the 'std::unique_ptr::release' function
is called or when its address is taken.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent memory leaks.
EXAMPLE 

#include <memory>

struct S
{
    S();
    ~S();
};

void del(S* foo)
{
    delete foo;
}

void f1()
{
    std::unique_ptr<S> managed_foo(new S);
    del(managed_foo.release());                      // Violation
}
REPAIR 

Do not use the 'std::unique_ptr::release' function.

#include <memory>

struct S
{
    S();
    ~S();
};

void f()
{
    std::unique_ptr<S> v1 = std::make_unique<S>();    // OK
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 21.6.2 Dynamic memory shall be managed automatically
