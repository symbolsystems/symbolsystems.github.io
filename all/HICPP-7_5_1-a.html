The 'asm' declaration should not be used [HICPP-7_5_1-a-3]

DESCRIPTION 

"Use of inline assembly should be avoided since it restricts the portability
of the code." [High Integrity C++]

This rule detects when the 'asm' declaration is used.
SINCE 

2023.2
NOTES 

This rule detects only 'asm' declarations according to the C++ standard:
  asm ( string-literal ) ;
Any language extensions that use assembly language are not detected.
BENEFITS 

This rule helps you improve code portability and avoid implementation-defined
behavior.
EXAMPLE 

int f1(int par) {
    int res;
    asm("/*add par+res */");         // Violation
    return res;
}
REPAIR 

int f1(int par) {
    int res;
    res += par;                      // OK
    return res;
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines

2. High Integrity C++ Coding Standard
