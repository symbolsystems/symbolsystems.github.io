Eliminate unused parameters [MISRACPP2023-0_2_2-a-2]

DESCRIPTION 

The rule reports violations on parameters that are not used inside body 
of function. It is recommended to remove unused parameters or to omit 
their names (possible only in C++ language).
NOTES 

The rule does not trigger on parameters captured by lambda expressions. 
It assumes that lambdas use every parameter they capture.
EXCEPTIONS 

The rule does not report violations on virtual functions and on functions
that are used as callback (an address of function is taken in translation unit).
The rule does not report violations on unnamed parameters.
BENEFITS 

The rule improves legibility and increases efficiency of code.
EXAMPLE 

int foo(int i, int k) {      // Violation
    i = 5 * i;
    return i;
}
REPAIR 

int foo(int i) {      // OK
    i = 5 * i;
    return i;
}

// Exceptions

class A {
public:
    virtual void foo(int a);
};

void A::foo(int a){          // OK - virtual function
}

int callback(int a, int b){} // OK - address of function is taken

void useCb(){
    int(*fp)(int, int);
    fp = callback;
}

int unpar(int a, int){       // OK - unnamed parameter
    a++;
    return a;
}
