The volatile qualifier shall be used appropriately [CODSTA-306-1]

DESCRIPTION 

This rule detects when:
* a local variable, or
* a function parameter, or
* a function return type, or
* a member function, or
* a structured binding
is declared with the volatile qualifier.
SINCE 

2023.2
NOTES 

Since C++20, some of the cases detected by this rule are also deprecated by the standard.
BENEFITS 

This rule helps you avoid unexpected or undefined behavior.
EXAMPLE 

volatile int f1(volatile int a) // Violation x2
{
    volatile int v01;           // Violation
    int* volatile v02;          // Violation
    return v01;
}

class C
{
    void f2() volatile;         // Violation
};

int arr[2] = {1, 2};
volatile auto [a1, a2] = arr;   // Violation
REPAIR 

int f1(int a)                   // OK
{
    int v01;                    // OK
    int* v02;                   // OK

    // Examples of the correct code:
    volatile int* v03;          // OK - a pointer-to-volatile is not volatile
    volatile int& v04 = v01;    // OK - a reference-to-volatile is not volatile
    return a;
}

class C {
  void f2();                    // OK
};

int arr[2] = {1, 2};
auto [a1, a2] = arr;            // OK
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
