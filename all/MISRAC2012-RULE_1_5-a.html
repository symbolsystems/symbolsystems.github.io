The static storage class specifier shall be used in definitions and declarations of objects and functions that have internal linkage [MISRAC2012-RULE_1_5-a-2]

DESCRIPTION 

The C standard specifies that if an identifier declared with the storage-class
specifier 'extern' in a scope in which a prior declaration of that identifier
is visible, the linkage of that identifier is the same as the linkage of the
prior declaration.

For example, if a function is first declared as 'static' and then defined with
'extern' or without any specifier at all, the linkage will be internal. This
can be confusing: if you see only the definition, you might wrongly assume
that the function has external linkage.

It is a good practice to apply the 'static' storage class specifier to all
objects and functions with internal linkage.

See also: CODSTA-81
EXCEPTIONS 

This rule does not trigger on:
- definitions of const variables in C++ code.
- declarations of objects and functions in a block scope that have internal
  linkage.
BENEFITS 

Mitigates the risk of incorrectly interpreting linkage of objects and
functions.
EXAMPLE 

static int x = 0;
extern int x;        /* Violation */

static int foo1();
extern int foo1();   /* Violation */

static void foo2();
void foo2() {}       /* Violation */
REPAIR 

static int x = 0;
static int x;          /* OK */

extern int y;
int y = 0;             /* OK */

static int foo1();
static int foo1();     /* OK */

static void foo2();
static void foo2() {}  /* OK */
