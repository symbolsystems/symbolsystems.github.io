All structures should be typedef'd [CODSTA-47-3]

DESCRIPTION 

In C language, a structure declaration looks similar to a variable
declaration of a struct type. Using a typedef for each structure
makes variable declarations of struct types consistent with variable
declarations of built-in types, which improves readability of code.
Additionally, using typedefs prevents programming errors caused by
typos in variable declarations.
This rule detects when a structure does not have a typedef declaration
in the current file.
EXCEPTIONS 

This rule applies only to C code.
BENEFITS 

This rule improves readability and maintainability of code.
EXAMPLE 

struct A {    // Violation - no typedef
    int i;
};

struct B {    // Violation - no typedef
    int i;
};
REPAIR 

typedef struct A {    // OK
    int i;
} A_t;

struct B {            // OK
    int i;
};
typedef struct B B_t;
