Do not initialize a reference to an object whose address can be changed [INIT-05-1]

DESCRIPTION 

A reference should not be initialized to an object whose address can be
changed. For example, if a reference is initialized to an object that is
obtained by dereferencing a pointer and the object that this pointer points to
is deleted, the reference will refer to a non-existing object.

This rule detects when a reference is initialized to an object accessed by
dereferencing a pointer.
BENEFITS 

This rule helps you prevent illegal access to the memory that has been freed.
EXAMPLE 

int * bar();

void foo(char * a)
{
   int * ptr = new int(1);
   int & rintptr1 = *ptr;                      // Violation
   int & rintptr2 = ptr[0];                    // Violation
   int & rintptr3 = *bar();                    // Violation
   char & rcharptr = *a;                       // Violation
   if (ptr) delete ptr;
   // ...
}
REPAIR 

/* Examples of correct code */ 
int & bar();

void foo(char & a)
{
   int val = 1;
   int &rval1 = val;                           // OK
   int &rval2 = bar();                         // OK
   char &rchar = a;                            // OK
}
