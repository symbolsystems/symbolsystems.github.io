Braces ("{}") which enclose a block will not have an empty line after "{" nor before "}" [FORMAT-44-3]

DESCRIPTION 

Rule reports a violation if braces ("{}") which enclose a block have
an empty line after "{" or before "}". Only lines between matching
braces are checked, so no violation is reported if braces are placed
in one line or if there are no lines between them (the block is
completely empty and the closing brace is in line directly after the
opening brace).

The rule checks opening and closing braces only for blocks. Not all 
braces are "block" braces. The block is a compound statement and can
be used, for example:
 - by itself
 - to enclose a body of statements:
   'if', 'else', 'while', 'do-while', 'for', 'switch', 'try', 'catch'
 - to enclose a function's body
Examples of braces which are not "block" braces:
 - class definition
 - variable initializer

See also: FORMAT-02, FORMAT-03, FORMAT-34, FORMAT-42, FORMAT-43
SINCE 

v9.4
BENEFITS 

Rule improves readability and maintainability.
EXAMPLE 

void bar()
{ // Violation

} // Violation
int foo(int i) {                // Violation

    if (i > 0) {                // Violation

        {                       // Violation

            int j; }            // Violation
        return i;

    } else if (i < 0) {         // Violation (opening and closing brace)


    } else                      // Violation
    {   return 0;               // Violation

    }                           // Violation
}
REPAIR 

void bar()
{ // OK - completely empty block
}
int foo(int i) {
    if (i > 0) {
        // OK - this line has a comment

        { int j; } // OK - the braces are in the same line
        return i;
    } else if (i < 0) {
        // OK - this line has a comment
    } else
    {   return 0;
    }
    return 0;
}

// Braces that are not "block" braces are not checked
enum E {EN1 = 2, EN2 = 1};
class FooBar{};
struct S{ int i; float f; } ss;
int array[] = {1,2,3,4,5};
