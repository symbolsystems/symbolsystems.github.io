Do not use deprecated STL library features [HICPP-1_3_4-a-3]

DESCRIPTION 

The following STL features were deprecated in the 2011 C++ standard and may be
removed completely in a later versions:

  * std::auto_ptr
  * std::bind1st
  * std::bind2nd
  * std::ptr_mem_fun
  * std::ptr_mem_fun_ref
  * std::unary_function
  * std::binary_function

The most notable type deprecated in this version is std::auto_ptr. Searching
for this type and replacing it with std::unique_ptr may uncover latent bugs
related to the incorrect use of std::auto_ptr.
SINCE 

v10.4
BENEFITS 

You should keep your code up to date with the version of your programming
language standard. Doing so ensures the longevity of your application, as well
as addresses defects and vulnerabilities that may emerge as deployment
environments evolve.
EXAMPLE 

#include <cstdint>
#include <memory>

void foo() {
    std ::auto_ptr<int32_t> p1(new int32_t(0)); // Violation
}
REPAIR 

#include <cstdint>
#include <memory>

void foo() {
    std ::unique_ptr<int32_t> p2(new int32_t(0)); // OK
}
REFERENCES 

1. High Integrity C++ Coding Standard
   1.3.4. Do not use deprecated STL library features
