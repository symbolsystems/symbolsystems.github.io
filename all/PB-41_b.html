Do not pass an expression with array type to a function with a pointer or array type parameter [PB-41_b-3]

DESCRIPTION 

"When an array is bound to a function parameter of pointer type the array
is implicitly converted to a pointer to the first element of the array.
In order to retain the array dimension, the parameter should be changed
to a reference type or changed to a user defined type such as std::array."
[High Integrity C++]

This rule detects when an argument of an array type is passed to a function
call as a parameter of an array or pointer type.

See also: PB-41
SINCE 

v9.5
NOTES 

This rule allows to pass an argument as a parameter of a reference-to-array
type, because in that case array decay does not happen.
BENEFITS 

This rule helps you improve the readability of the code and avoid mistakes.
EXAMPLE 

#include <cstdint>

void f1(int32_t a[10]);
void f2(int32_t a[]);
void f3(int32_t *a);
void f4(int32_t *const & a );

void f()
{
    int32_t a1[10];
    f1(a1);             // Violation
    f2(a1);             // Violation
    f3(a1);             // Violation
    f4(a1);             // Violation
}
REPAIR 

#include <cstdint>
#include <array>

void f1(int32_t (&a)[10]);
void f2(std::array<int32_t, 10> &a);

void f()
{
    int32_t a1[10];
    std::array < int32_t, 10 > a2;
    f1(a1);                            // OK
    f2(a2);                            // OK
}
