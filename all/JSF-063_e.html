There will be no space between a unary operator '&', '*', '+', '-' and its operand [JSF-063_e-3]

DESCRIPTION 

The rule reports a violation if there is space, tab, comment or new line
character between a unary operator "&", "*", "+", "-" and its operand.

See also: FORMAT-17, FORMAT-18, FORMAT-21, FORMAT-22, FORMAT-46
SINCE 

v9.4
NOTES 

The rule does not report violations on unary operators used inside preprocessor
directives.

The rule does not report a violation on "operator" construct (declaration and
explicit call), for example:
int operator + (const A&); // no violation reported
int x = operator+ (a); // no violation reported
BENEFITS 

The rule promotes continuity in the relationship between the operators
and the operands on which they act.
DRAWBACKS 

The rule can report incorrect violation if in the same line is a unary
operator and a character with the same representation as the unary operator, 
but with different meaning (e.g. '-' used both as binary and unary operator).
EXAMPLE 

int * foo(int i) {
    int * p;
    i = + 10;           // Violation
    i = -  (10);        // Violation
                        // Violation in next line
    i = *
        p;
    i = i - * p;        // Violation
}

#define XXX(a) - a      // OK - no violation on macro
class A;
int operator + (const A&); // OK - no violation on "operator" construct
void bar(A& a) {
    int x = operator+ (a); // OK - no violation on "operator" construct
    x = + a;               // Violation
}
REPAIR 

int * foo(int i) {
    int * p;
    i = +10;        // OK
    i = -(10);      // OK
    i = *p;         // OK
    i = i - *p;     // OK
    p = &i;         // OK
}

#define XXX(a) -a   // OK
class A;
int operator+(const A&);   // OK
void bar(A& a) {
    int x = operator+(a);  // OK
    x = +a;                // OK
}
