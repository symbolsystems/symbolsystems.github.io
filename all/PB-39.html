A function shall not return a reference or a pointer to a parameter that is passed by reference or const reference [PB-39-3]

DESCRIPTION 

"A function shall not return a reference or a pointer to a parameter that is
passed by reference or const reference. It is implementation-defined behaviour
whether the reference parameter is a temporary object or a reference to the
parameter. If the implementation uses a local copy (temporary object), this
will be destroyed when the function returns. Any attempt to use such an object
after its destruction will lead to undefined behaviour."
SINCE 

v7.2
BENEFITS 

Rule prevents non-deterministic behaviour.
EXAMPLE 

int* foo2 ( int& x )
{
    return ( &x );     // Violation
}
REPAIR 

Do not return a reference or a pointer to a parameter that is passed by
reference or const reference
REFERENCES 

1. MISRA C++:2008 - Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 7, Rule 7-5-3
