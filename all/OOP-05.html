Do not use multiple inheritance [OOP-05-2]

DESCRIPTION 

Using multiple inheritances can result in strange class hierarchies and less 
flexible code. Because in C++ there may be an arbitrary number of instances
of a given type, it could be that direct inheritance from a class may only
be used once (see the example below). This rule detects if you use multiple
inheritances.
See also: OOP-07, OOP-06, OOP-04, OOP-03
BENEFITS 

Clarity, maintainability. Following this rule prevents ambiguity when classes 
derive from one object through multiple objects as in the "diamond of death".
EXAMPLE 

class A {};
class B {};
class C : public A, public B {}; // Violation
REPAIR 

class B {};
class A : public B {}; // OK
class C : public B {}; // OK
