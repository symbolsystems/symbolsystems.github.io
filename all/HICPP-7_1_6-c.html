Use class types or typedefs to abstract scalar quantities and standard integer types [HICPP-7_1_6-c-3]

DESCRIPTION 

"Where the auto type specifier is used in a declaration, and the initializer is
a constant expression, the declaration should not be allowed to resolve to a
standard integer type.  The type should be fixed by casting the initializer to
a size specific type." [HIC++ v4.0, Rule 7.1.6]
SINCE 

v10.4
BENEFITS 

Improves readability and prevents the compiler from resolving an integer to the
wrong type.
EXAMPLE 

#include <cstdint>
void foo() {
    auto a(0);  // violation: int
    auto b(0L); // violation: long
    auto c(0U); // violation: unsigned int
}
REPAIR 

#include <cstdint>
void foo() {
    auto a(static_cast<int32_t>(0)); // OK
    auto b(static_cast<int64_t>(0)); // OK
    auto d(static_cast<int32_t>(0)); // OK
}
REFERENCES 

1. High Integrity C++ Coding Standard

   standard integer types
