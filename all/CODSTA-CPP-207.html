An argument passed via ellipsis shall have an appropriate type [CODSTA-CPP-207-2]

DESCRIPTION 

This rule reports when an argument passed to a function via ellipsis is
a class type, which:
* has virtual functions, or
* has non-trivial copy or move operations, or
* has a non-trivial destructor.

Passing objects of such types via ellipsis is conditionally supported and
results in implementation-defined behavior.
SINCE 

2023.2
NOTES 

The rule does not report if the call is part of unevaluated context (e.g. inside
sizeof).
BENEFITS 

The rule helps you avoid implementation-defined behavior and portability
problems.
EXAMPLE 

class C
{
public:
    ~C();
};
void f(C c, int i, ...);

void g()
{
    C c;
    f(c, 10, c); // Violation
}
REPAIR 

class C
{
public:
    ~C();
};
void f(C c, int i, ...);

void g()
{
    C c;
    f(c, 10, &c); // OK
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
