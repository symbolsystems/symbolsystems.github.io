The \ character should not occur in a header file name [HICPP-16_1_2-b-3]

DESCRIPTION 

"It is undefined behaviour if the \ character is used between < and >
delimiters or between the " delimiters in a header name preprocessing token."

See also: NAMING-46, MISRA2004-19_2
SINCE 

v7.2
NOTES 

Some environments use \ as a file name delimiter. Compilers for these
environments often support the use of / in #include directives.
BENEFITS 

The rule prevents undefined behaviour.
EXAMPLE 

#include "fi\\le.h" // Violation
REPAIR 

#include "file.h"  // OK
