Output parameters shall not be used [AUTOSAR-A8_4_8-a-2]

DESCRIPTION 

"Output parameters are passed to a function as non-const references
or pointers that can denote either in-out or out-only parameter.
Using return value prevents from possible misuse of such a parameter."
[AUTOSAR A8-4-8]

This rule detects output parameters in functions with 'void' return type.
SINCE 

v10.4
NOTES 

This rule assumes that an output parameter is a parameter of non-const
reference type that is assigned in a function via an assignment operator
and is not used within this function in any other way.
BENEFITS 

This rule helps you prevent misuse of output parameters and improve code
clarity and readability.
EXAMPLE 

In the following example, the 'p1' output parameter is used in a void function.

void f1(int a, int b, int& p1) {  // Violation
    p1 = a + b;
}
REPAIR 

You can fix the code by not using the output parameter in the void function.

In the following example, the 'f1' function has a return value instead of
the output parameter.

int f1(int a, int b) {            // OK
    return (a + b);
}

In the following example, the 'p1' parameter of the 'f2' function is
an input-output parameter.

void f2(int a, int b, int& p1) {  // OK
    p1 = a + b + p1;
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
