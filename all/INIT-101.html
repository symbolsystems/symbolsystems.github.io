All non-class variables with automatic storage duration should be explicitly initialized [INIT-101-3]

DESCRIPTION 

This rule reports when a variable with automatic storage duration
does not have an explicit initializer.

See also: INIT-03
SINCE 

2023.2
EXCEPTIONS 

The rule does not report on variables of class, struct or union
type.
BENEFITS 

This rule helps you avoid unexpected or undefined behavior.
Initializing all variables reduces the risk of a variable
being used without a value assigned.
EXAMPLE 

int test()
{
    int i;           // Violation
    if (i > 0) {
        int j;       // Violation
        int a[10];   // Violation
        return a[i];
    }
    int* p;          // Violation
    return *p;
}
REPAIR 

int test()
{
    int i(0);        // OK
    if (i > 0) {
        int j = i;   // OK
        int a[10]{}; // OK
        return a[j];
    }
    int* p{&i};      // OK
    return *p;
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
