The Standard Library input/output functions shall not be used [CODSTA-110-3]

DESCRIPTION 

"This rule applies to the functions that are specified as being provided
by <stdio.h> and, in C99, their wide-character equivalents specified 
in Sections 7.24.2 and 7.24.3 of the C99 Standard as being
provided by <wchar.h>."

The rule reports violations on calls of functions declared in stdio.h header
and on the following functions declared in wchar.h header: fwprintf, fwscanf, 
wprintf, wscanf, swprintf, swscanf, vwprintf, vfwprintf, vswprintf, vwscanf,
vfwscanf, vswscanf, fgetwc, fgetws, fputwc, fputws, getwc, putwc, ungetwc,
fwide, getwchar, putwchar.

See also: MISRA2004-20_9
SINCE 

v9.5
BENEFITS 

The rule prevents unspecified, undefined and implementation-defined behaviours.
EXAMPLE 

#include <stdio.h>

void foo1 ()
{
    char str[20];
    scanf ("%s",str);                 // Violation
    printf("string %s", str);         // Violation
}

void foo2 ()
{
    FILE * pFile;
    pFile = fopen ("myfile.txt","w"); // Violation 
    if (pFile!=NULL)
    {
        fputs ("string",pFile);       // Violation 
        fclose (pFile);               // Violation 
    }
}
REPAIR 

Do not use Standard Library input/output functions.
