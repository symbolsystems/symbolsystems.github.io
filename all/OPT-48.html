Destructor shall have a non-empty body [OPT-48-4]

DESCRIPTION 

If a user destructor is defined, then it should contain any statements
that will allow for managing resources. If a resource management in
a class is not required, then the destructor should be declared as default.

This rule detects when the user-defined destructor does not contain
any statements other than compound statements and null statements.
SINCE 

2023.2
BENEFITS 

This rule helps you improve the readability and maintainability of the code.
EXAMPLE 

class C
{
public:
  ~C()                       // Violation
  {
  }
};
REPAIR 

// Examples of correct code

class C1
{
public:
  ~C1() = default;           // OK
};

class C2
{
public:
  ~C2() = delete;            // OK
};

class C3
{
public:
  ~C3()                      // OK
  {
    if (ptr) delete ptr;
  }
private:
  int *ptr;
};
