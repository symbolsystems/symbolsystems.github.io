Pass small objects with a trivial copy constructor by value [CODSTA-CPP-108_b-3]

DESCRIPTION 

"Because passing by const reference involves an indirection, it will be
less efficient than passing by value for a small object with a trivial
copy constructor." [High Integrity C++]

This rule detects when a parameter of const reference to a small type
is declared.
SINCE 

2023.1
NOTES 

This rule assumes that a small type is a class or a struct type
whose size is no larger than 64 bits.

This rule detects a trivial copy constructor according to its definition
in the C++11 standard.

This rule does not check parameters of const reference to incomplete
or non-instantiated template types.
BENEFITS 

This rule helps you to improve the efficiency of your code.
EXAMPLE 

#include <cstdint>

class C {
  public:
    C(C const &) = default; // Trivial copy constructor

  private:
    int32_t m_j;
};

void foo(const C &c) {}     // Violation
REPAIR 

#include <cstdint>

class C {
  public:
    C(C const &) = default; // Trivial copy constructor

  private:
    int32_t m_j;
};

void foo(C c) {}            // OK - passing object by value
