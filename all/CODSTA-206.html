The '_Noreturn' function specifier should not be used [CODSTA-206-3]

DESCRIPTION 

"Updates to the C Standard have introduced new language features. The following
shall not be used:
- The _Noreturn function specifier and the <stdnoreturn.h> header file;"
[MISRA C:2012 Amendment 2]

This rule detects when one of the following is used in code:
- the '_Noreturn' function specifier
- a macro that expands to the '_Noreturn' function specifier (such as
  the 'noreturn' macro from the 'stdnoreturn.h' library)
SINCE 

2020.1
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

_Noreturn int  f1() { return 3; }    /* Violation */
_Noreturn void f2() { }              /* Violation */

#include <stdnoreturn.h>
noreturn  int  f3() { return 3; }    /* Violation */
noreturn  void f4() { }              /* Violation */
REPAIR 

To fix the code, do not use the '_Noreturn' function specifier or the macro that
expands to the '_Noreturn' specifier.

int  f1() { return 3; }              /* OK */
void f2() { }                        /* OK */

#include <stdnoreturn.h>
int  f3() { return 3; }              /* OK */
void f4() { }                        /* OK */
