The C library shall not be used [HICPP-1_3_3-a-3]

DESCRIPTION 

"The C standard library headers are included in C++ for compatibility.
However, their inclusion was deprecated in the 1998 C++ Language Standard
and thus they may be withdrawn in a later version.
Instead of <name.h> prefer to use <cname>." [High Integrity C++]

This rule detects following standard C library headers:
<assert.h>, <ctype.h>, <errno.h>, <float.h>, <iso646.h>, <limits.h>,
<locale.h>, <math.h>, <setjmp.h>, <signal.h>, <stdarg.h>, <stddef.h>
<stdio.h>, <stdlib.h>, <string.h>, <time.h>, <wchar.h>, <wctype.h>
SINCE 

v7.2
NOTES 

This rule does not trigger when a file contains the __cplusplus macro used
in a #ifdef, #ifndef, #if, or #elif directive and includes an appropriate
header from the C++ standard library.

#ifdef __cplusplus
#include <cassert>
#else
#include <assert.h>  // No Violation
#endif
BENEFITS 

This rule helps you avoid using deprecated headers from the C standard library.
EXAMPLE 

#include <stdio.h> // Violation
REPAIR 

#include <cstdio>  // OK
