A compatible declaration shall be visible when an object or function with external linkage is defined [MISRA2012-RULE-8_4_a-2]

DESCRIPTION 

"If a declaration for an object or function is visible when that object
or function is defined, a compiler must check that the declaration
and definition are compatible. In the presence of function prototypes
checking extends to the number and type of function parameters."
[MISRA C:2012]

This rule reports a violation if an object or a function with external linkage
does not have a declaration that precedes its definition. The rule does not
check if a declaration is compatible. Violations on incompatible declarations
are reported by rule MISRA2004-8_4.

This rule is only enabled for C language and does not report violations
for the 'main' function.

See also: MISRA-023, MISRA-071_a, MISRA2004-8_4
NOTES 

"The recommended method of implementing declarations of objects and functions
with external linkage is to declare them in a header file, and then include
the header file in all those code files that need them, including the one
that defines them." [MISRA C:2012]
BENEFITS 

The rule prevents undefined behaviours.
EXAMPLE 

/* file.c */
int i1;                   /* Violation - no declaration */
extern int ei1 = 1;       /* Violation - no declaration */
void foo1(int x){}        /* Violation - no declaration */
void foo2(int x){}        /* Violation - declaration after definition */
void foo2(int x);
REPAIR 

/* file.h */
extern int i1;
extern int ei1;
extern void foo1(int x);
extern void foo2(int x);

/* file.c */
#include "file.h"
int i1;                   /* OK */
int ei1 = 1;              /* OK */
void foo1(int x){}        /* OK */
void foo2(int x){}        /* OK */
