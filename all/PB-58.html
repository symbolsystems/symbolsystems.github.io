Within an enumerator list, the value of an implicitly-specified enumeration constant shall be unique [PB-58-3]

DESCRIPTION 

"If implicitly-specified and explicitly-specified constants are mixed within
an enumeration list, it is possible for values to be replicated. 
Such replication may be unintentional and may give rise to unexpected behaviour.
This rule requires that any replication of enumeration constants be made
explicit, thus making the intent clear."
SINCE 

v9.5
BENEFITS 

The rule improves readability of code and prevents unexpected behaviours.
EXAMPLE 

enum En1 {E11 = 1, E12, E13 = 2};             /* Violation */
enum En2 {E21 = -2, E22, E23 = -1};           /* Violation */
enum En3 {E31, E32, E33 = E31};               /* Violation */
REPAIR 

enum En1 {E11 = 1, E12 = 1, E13 = 2};         /* OK */
enum En2 {E21 = -2, E22 = -1, E23 = -1};      /* OK */
enum En3 {E31 = 0, E32, E33 = E31};           /* OK */
REFERENCES
