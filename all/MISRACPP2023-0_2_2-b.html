There shall be no unused named parameters in virtual functions [MISRACPP2023-0_2_2-b-2]

DESCRIPTION 

"Unused named parameters are often a result of a design changes
and can lead to mismatched parameter lists." [AUTOSAR A0-1-5]

For this reason, it is recommended that you remove unused parameters
from a virtual function or leave them unnamed.

This rule detects when a named parameter of a virtual member function
is never used inside the body of that function.
SINCE 

v10.4
NOTES 

The rule does not report parameters captured by lambda expressions.
It assumes that lambdas use every parameter they capture.
The rule does not report parameters from template functions that
contain assembler code.
BENEFITS 

This rule helps you improve code readability and increase code efficiency.
EXAMPLE 

class C1 {
public:
    virtual int f1(int x, int y) = 0;
    virtual int f2(int x, int y) = 0;
};

class C2 : public C1 {
public:
    virtual int f1(int x, int y) {  // Violation
        return x;
    }
    virtual int f2(int x, int y) {  // Violation
        return y;
    }
};
REPAIR 

To fix the code, remove the unused parameter or skip the name of the unused
parameter.

class C1 {
public:
    virtual int f1(int x) = 0;
    virtual int f2(int x, int y) = 0;
};

class C2 : public C1 {
public:
    virtual int f1(int x) {               // OK
        return x;
    }
    virtual int f2(int /* x */, int y) {  // OK
        return y;
    }
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A0-1-5. There shall be no unused named parameters in the set of parameters
   for a virtual function and all the functions that override it.

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 0.2.2 A named function parameter shall be used at least once
