The 'binder1st' and 'binder2nd' identifiers should not be used [CODSTA-MCPP-07_b-3]

DESCRIPTION 

"All code shall conform to ISO/IEC 14882:2014 - Programming Language C++ and
shall not use deprecated features." [AUTOSAR-A1-1-1]

The 'binder1st' and 'binder2nd' template classes from the Standard Library
were deprecated in the C++11 and removed in the C++17 version of the C++
Language Standard.

This rule detects when the 'binder1st' and 'binder2nd' identifiers are used
in code.
SINCE 

v10.4
NOTES 

This rule reports all occurrences of the 'binder1st' and 'binder2nd'
identifiers. For this reason, the rule may report 'binder1st' or 'binder2nd'
that does not represent the class from the Standard Library. For example:

void binder1st (); // Violation
void binder2nd (); // Violation
BENEFITS 

This rule helps you ensure that your code complies with the up-to-date version
of the C++ Language Standard. This allows you to increase longevity of
an application and address the defects and vulnerabilities that may emerge
as deployment environments evolve.
EXAMPLE 

#include <functional>

void foo ()
{
    std::binder1st<equal_to<int>> equal_to_10 (equal_to<int>(),10); // Violation
    std::binder2nd<less<int>> is_negative (less<int>(),0);          // Violation
}
REPAIR 

Do not use the 'binder1st' and 'binder2nd' identifiers.
