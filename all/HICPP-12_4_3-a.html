Do not specify both an NSDMI and a member initializer in a constructor for the same non-static member [HICPP-12_4_3-a-3]

DESCRIPTION 

Non-static members should not be initialized more than once in a constructor.
Initializing a data member in an initializer list and in the base class will
create confusion as to which value is being set to the data member.
SINCE 

v10.4
NOTES 

If a non static member is being initialized in the base class, then it should
not be in the initialization list.
EXCEPTIONS 

"The move/copy constructors are exempt from this rule, as in this context the
rule conflicts with Rule 12.4.2: ”Ensure that a constructor initializes
explicitly all base classes and non-static data members”, which takes
precedence.” [HIC++ v4.0, Rule 12.4.3]
BENEFITS 

Improves clarity about which value was used to initialize the data member.
EXAMPLE 

#include <cstdint>

class A {
  public:
    // initialization list
    A()
        : member_1(1), member_2(1) // Violation
    {}

  private:
    int32_t member_1;
    int32_t member_2 = 0; // Violation
    int32_t member_3 = 0;
};
REPAIR 

#include <cstdint>

class A {
  public:
    // initialization list
    A()
        : member_1(1) // OK. member_2 was removed from initialization list
    {}

  private:
    int32_t member_1;
    int32_t member_2 = 0; // OK
    int32_t member_3 = 0;
};
