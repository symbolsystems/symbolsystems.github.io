The '_Pragma' operator should not be used [MISRACPP2023-19_6_1-b-4]

DESCRIPTION 

The use of the '_Pragma' operator is implementation-specific and causes
the program to behave in an implementation-defined manner.

The rule detects the '_Pragma' identifiers.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

_Pragma( "once" )          // Violation
REPAIR 

#ifndef FOO_H              // OK
#define FOO_H
/* ... */
#endif
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 19.6.1 The #pragma directive and the _Pragma operator should not be used
