Only those escape sequences that are defined in ISO/IEC 14882:2014 shall be used [AUTOSAR-A2_13_1-a-2]

DESCRIPTION 

"The use of an undefined escape sequence leads to undefined behavior.
The defined escape sequences (ISO/IEC 14882:2014) are: \', \", \?, \\, \a, \b,
\f, \n, \r, \t, \v, \<Octal Number>, \x<Hexadecimal Number>." [AUTOSAR A2-13-1]

The rule detects when a character constant or a string constant contains the
'\' character that is not used either to define an escape sequence or to create
a universal character name.

See also: NAMING-48
SINCE 

2022.1
NOTES 

"Universal-character-names (\u hex-quad and \U hex-quad hex-quad) are also
allowed in character and string literals" [AUTOSAR A2-13-1]

This rule allows every use of '\' before 'u' or 'U' and does not check
if the value of the universal character name is correct.

The rule does not detect '\' characters in raw strings. In raw strings any
escape sequences are not processed.

The rule does not detect '\' characters used in #include directives.
These characters should not be used in any case in #include directives
and are disallowed by another rule.
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

void f1( char c, const char* s )
{
    c = '\k';           // Violation
    s = "dir1\dir2";    // Violation
    s = " \ unknown";   // Violation
}
REPAIR 

/* Examples of correct code */
void f1( char c, const char* s )
{
    c = '\t';           // OK
    s = "name \n";      // OK
    s = "\u1234";       // OK
    s = R"(abc \def)";  // OK
}
