A 'try' statement of a 'try-block' should contain a potentially-throwing statement [EXCEPT-27-3]

DESCRIPTION 

If a 'try' statement of a 'try-block' does not contain a potentially-throwing
statement, then all its catch-handlers are unreachable.

This rule detects when the body of a 'try' compound statement does not contain
any statements that are potentially-throwing.
SINCE 

2023.2
NOTES 

This rule does not report on function-try-blocks of constructors
and destructors.
BENEFITS 

This rule helps you avoid unreachable code.
EXAMPLE 

void bar() noexcept;
void foo()
{
    try {
        bar();
    } catch (...) {     // Violation
        // ...
    }
}
REPAIR 

void bar() noexcept;
void foo()
{
    bar();              // OK
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 0.0.1 A function shall not contain unreachable statements
