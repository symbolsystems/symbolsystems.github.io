The 'getenv()' function from the 'stdlib.h' or 'cstdlib' library shall not be used [MISRA2004-20_11_c-3]

DESCRIPTION 

"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<stdlib.h> shall not be used." [MISRA-C:2004]
"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<cstdlib> shall not be used. The use of these functions leads to
implementation-defined behaviour." [MISRA C++:2008]

This rule detects calls to the 'getenv()' function from the 'stdlib.h'
and 'cstdlib' standard libraries.

See also: CODSTA-185_a, BD-PB-INVRET, SECURITY-48_b, PB-75, PB-75_b
SINCE 

v10.4
NOTES 

The standard "MISRA C:2012 Amendment 1" allows you to use the 'getenv'
function if it is compliant with rules 21.19 and 21.20. However, this
rule reports every use of this function.
BENEFITS 

This rule helps you prevent undefined or implementation-defined behavior.
EXAMPLE 

#include  <stdlib.h>

void f1( int i ) {
    char *libvar;
    libvar = getenv( "LIB" );  /* Violation */
}
REPAIR 

You can fix the code by removing the call to the 'getenv()' function.
