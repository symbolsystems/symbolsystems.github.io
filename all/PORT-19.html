Instead of casting a long type operand of an expression to int cast the result of the expression [PORT-19-3]

DESCRIPTION 

The rule reports a violation if you use explicit cast to 'int' on operand
of long type. When you cast an operand from a long to an int type, then
some data could be removed and the result of expression could be not according
to expectations.
BENEFITS 

The rule prevents loss of data.
EXAMPLE 

void foo() 
{
   int int1, int2;
   long longValue;
   int2 = ((int)longValue)/int1; // Violation
}
REPAIR 

void foo() 
{
   int int1, int2;
   long longValue;
   int2 = (int)(longValue/int1); // OK
}
