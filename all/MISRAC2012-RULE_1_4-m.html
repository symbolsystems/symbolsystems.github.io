Do not use following macros: RSIZE_MAX, L_tmpnam_s, TMP_MAX_S [MISRAC2012-RULE_1_4-m-2]

DESCRIPTION 

"Updates to the C Standard have introduced new language features. The following
shall not be used:
- Other than defining __STDC_WANT_LIB_EXT1__ to '0', the facilities of Annex K
(Boundschecking interfaces) shall not be used." [MISRA C:2012 Amendment 2]

The Annex K specifies following macros:
- 'RSIZE_MAX' defined the <stdint.h> header
- 'L_tmpnam_s' and 'TMP_MAX_S' defined in <stdio.h> header

This rule detects when 'RSIZE_MAX', 'L_tmpnam_s' and 'TMP_MAX_S' identifiers
are used in the code.
SINCE 

2020.1
NOTES 

This rule reports all occurrences of the 'RSIZE_MAX', 'L_tmpnam_s'
and 'TMP_MAX_S' identifiers. For this reason, the rule may detect 'RSIZE_MAX',
L_tmpnam_s' or 'TMP_MAX_S' that does not represent the macro from 'stdint.h' 
or 'stdio.h' library header files.
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

#include <stdint.h>
int i = RSIZE_MAX;                                      // Violation

#include <stdio.h>
int j = TMP_MAX_S;                                      // Violation
int k = L_tmpnam_s;                                     // Violation
REPAIR 

To fix the code, do not use 'RSIZE_MAX', 'L_tmpnam_s' and 'TMP_MAX_S' macros.
