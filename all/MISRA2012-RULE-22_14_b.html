The second argument passed to the 'mtx_init()' function call should be either 'mtx_plain', 'mtx_timed', 'mtx_plain | mtx_recursive', or 'mtx_timed | mtx_recursive' [MISRA2012-RULE-22_14_b-1]

DESCRIPTION 

"The 'mtx_init' function creates a mutex object with properties indicated
by type, which must have one of the six values:
- 'mtx_plain' for a simple non-recursive mutex,
- 'mtx_timed' for a non-recursive mutex that supports timeout,
- 'mtx_plain | mtx_recursive' for a simple recursive mutex, or
- 'mtx_timed | mtx_recursive' for a recursive mutex that supports timeout."
[ISO/IEC 9899:2011]

If the 'mtx_init' function is called with the second argument with a value
different than listed above, then behavior is undefined.

This rule detects when the 'mtx_init' function is called with the second
argument different than 'mtx_plain', 'mtx_timed', 'mtx_plain | mtx_recursive'
or 'mtx_timed | mtx_recursive'.
SINCE 

2023.1
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

#include <threads.h>

void foo(mtx_t* m){
    mtx_init(m, 0x01);                        // Violation
    mtx_init(m, mtx_recursive);               // Violation
    mtx_init(m, mtx_plain | mtx_timed);       // Violation
};
REPAIR 

#include <threads.h>

void foo(mtx_t* m){
    mtx_init(m, mtx_plain);                   // OK
    mtx_init(m, mtx_timed);                   // OK
    mtx_init(m, mtx_plain | mtx_recursive);   // OK
    mtx_init(m, mtx_recursive | mtx_timed);   // OK
};
