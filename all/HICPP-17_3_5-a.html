Do not create an rvalue reference of std::array [HICPP-17_3_5-a-3]

DESCRIPTION 

"The std::array class is a wrapper for a C style array. The cost of moving
std::array is linear with each element of the array being moved. In most cases,
passing the array by & or const & will provide the required semantics without
this cost." [High Integrity C++ Coding Standard]

This rule detects when an rvalue reference of 'std::array' is used as one of 
the following:
- the return type of a function
- the type of a variable or parameter (in its declaration)
SINCE 

2021.1
BENEFITS 

This rule helps you improve code readability and efficiency.
EXAMPLE 

#include <array>

std::array<int, 10> garr;
std::array<int, 10> && rvgarr = std::move(garr);     // Violation

void foo(std::array<int, 10> &&);                    // Violation

std::array<int, 10> && bar() {                       // Violation
  std::array<int, 10> && rvlarr = std::move(garr);   // Violation
  return std::move(rvlarr);
}
REPAIR 

To fix the code, use 'std::array&' instead of 'std::array&&'.

#include <array>

std::array<int, 10> garr;
std::array<int, 10> & rgarr = garr;                  // OK

void foo(std::array<int, 10> &);                     // OK

std::array<int, 10> & bar() {                        // OK
  std::array<int, 10> & rlarr = garr;                // OK
  return rlarr;
}
REFERENCES 

1. High Integrity C++ Coding Standard
