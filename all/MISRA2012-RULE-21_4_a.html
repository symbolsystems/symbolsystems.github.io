The standard header file <setjmp.h> shall not be used [MISRA2012-RULE-21_4_a-2]

DESCRIPTION 

"The standard header file <setjmp.h> shall not be #included, and none of the
features that are specified as being provided by <setjmp.h> shall be used."
[MISRA C:2012 Rule 21.4 with Amendment 3]

"'setjmp' and 'longjmp' allow the normal function call mechanisms to be
bypassed. Their use may lead to undefined and unspecified behaviour."
[MISRA C:2012]

The rule detects the use of the following facilities:
- the 'longjmp' function
- the 'jmp_buf' type
- the 'setjmp' macro
NOTES 

This rule detects all identifiers that have the same names as the facilities
from the <setjmp.h> header file.
BENEFITS 

The rule helps you prevent implementation-defined and undefined behavior.
EXAMPLE 

#include <setjmp.h>
jmp_buf mark;                       // Violation

void f1( void ) {
    int jmpret;
    jmpret = setjmp( mark );        // Violation

    longjmp( mark, -1 );            // Violation
}
REPAIR 

Do not use the facilities from the <setjmp.h> header.
