If an identifier refers to a type, it shall not also refer to an object or a function in the same scope [CODSTA-CPP-87_c-3]

DESCRIPTION 

"For C compatibility, it is possible in C++ for a name to refer
to both a type and object or a type and function. This can lead
to confusion." [MISRA C++:2008 Rule 2-10-6]

"A class name or enumeration name can be hidden by the name of an object, 
function, or enumerator declared in the same scope. If a class or enumeration
name and an object, function, or enumerator are declared in the same scope
(in any order) with the same name, the class or enumeration name is hidden
wherever the object, function, or enumerator name is visible." [ISO/IEC 14882]

The rule reports a violation if in the same local scope is declared a type
(class/struct/union or enumeration) and an object, a function or an enumeration
constant with the same name.

Identifiers are declared in the same local scope if they are declared in the
same block in a function body. Members of the anonymous union declared
in the block are detected as identifiers declared in this block.

The rule does not check neither function parameters (their scopes are not
the same as scopes of identifiers declared in a function's body) nor variables
declared in the for-init-statement and in the condition of if, while, for, 
and switch statements (their scopes are not the same as scopes of identifiers
declared in blocks following these statements). 

See also: CODSTA-CPP-83, CODSTA-CPP-84, CODSTA-CPP-85, MISRA2004-5_2_a,
          MISRA2004-5_2_b
SINCE 

v9.5
BENEFITS 

The rule improves readability of code and prevents confusions.
EXAMPLE 

void foo1() {
    class A1 {      // Violation - 'A1' refers also to the function
        int m1;
    };

    int A1();       // Violation - 'A1' refers also to the class

    struct S1 {     // Violation - 'S1' refers also to the member
                    //             of anonymous union
        int m1;
    };

    union {
        int S1;     // Violation - 'S1' refers also to the struct
    };

    struct S2 {     // Violation - 'S2' refers also to the object
        int m1;
    } S2;           // Violation - 'S2' refers also to the struct

    enum E1 {       // Violation - 'E1' refers also to the enumeration constant
        E1          // Violation - 'E1' refers also to the enumeration type
    };
};
REPAIR 

void foo1(int i) {
    class A1 {        // OK - 'A1' does not refer to any object, function
                      //      nor enumeration constant declared in a local scope
        int m1;
    };

    int a1();         // OK - 'a1' does not refer to any type declared
                      //      in a local scope

    {
        int A1;       // OK - the object is declared in an inner local scope
    };

    if(i > 0){
        class a1{};   // OK - the type is declared in an inner local scope
    };
    
    for(int GN1 = 0; GN1 > 10; GN1++) {  // OK - variables declared in the
                                         // for-init-statement are not checked
        class GN1{};                     // OK - the type is declared in
                                         // different scope than object GN1
    }
};
