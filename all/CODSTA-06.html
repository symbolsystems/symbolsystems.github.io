Avoid using the '?:' operator [CODSTA-06-3]

DESCRIPTION 

This rule detects if you use the '?:' operator.
BENEFITS 

Readability. The '?:' operator is difficult to read
and leads to code obfuscation.
EXAMPLE 

int main( ) 
{
    return (1 ? 1 : 0); // Violation
};
REPAIR 

int main( ) 
{
    if (1)             // OK
        return 1;
    else
        return 0;
};
