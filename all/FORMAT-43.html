Braces ("{}") which enclose a block will be placed in the same column [FORMAT-43-3]

DESCRIPTION 

Rule reports a violation if braces ("{}") which enclose a block are not placed
in the same column.

The rule checks opening and closing braces only for blocks. Not all 
braces are "block" braces. The block is a compound statement and can
be used, for example:
 - by itself
 - to enclose a body of statements:
   'if', 'else', 'while', 'do-while', 'for', 'switch', 'try', 'catch'
 - to enclose a function's body
Examples of braces which are not "block" braces:
 - class definition
 - variable initializer

See also: FORMAT-02, FORMAT-03, FORMAT-34, FORMAT-42, FORMAT-44
SINCE 

v9.4
BENEFITS 

Rule improves readability and maintainability.
EXAMPLE 

void bar()
{                               // Violation
    }
int foo(int i) {                // Violation
    if (i > 0)
     {                          // Violation
        { int j; }              // Violation
        return i;
    } else if (i < 0) {         // Violation
        i /= 2;
        return -i;
    } else
    { return 0; }               // Violation
}
REPAIR 

// Rule does not check the indentation, only if column number matches:
void bar()
    {
    }
int foo(int i)
{
    if (i > 0)
    {
            {
            int j;
            }
        return i;
    } else if (i < 0)
    {
        i /= 2;
        return -i;
    } else {
        return 0;
           }
}

// Braces that are not "block" braces are not checked
enum E {EN1 = 2, EN2 = 1};
class FooBar{};
struct S{ int i; float f; } ss;
int array[] = {1,2,3,4,5};
