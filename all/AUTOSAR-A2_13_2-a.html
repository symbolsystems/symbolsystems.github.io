String literals with different encoding prefixes shall not be concatenated [AUTOSAR-A2_13_2-a-2]

DESCRIPTION 

"Concatenation of wide and narrow string literals leads to undefined behavior."
[AUTOSAR A2-13-2]

"If both string literals have the same encoding-prefix, the resulting
concatenated string literal has that encoding-prefix. If one string-literal
has no encoding-prefix, it is treated as a string-literal of the same
encoding-prefix as the other operand. If a UTF-8 string literal token
is adjacent to a wide string literal token, the program is ill-formed.
Any other concatenations are conditionally-supported with
implementation-defined behavior." [ISO/IEC 14882:2011]

This rule detects when string literals with different encoding prefixes are
concatenated.

The rule can recognize following encoding prefixes:
'u8', 'u', 'U', 'L'.
SINCE 

v10.4
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

wchar_t array[] = U"Hello"              // Violation
                  L"World";
REPAIR 

wchar_t array[] = L"Hello"              // OK
                  L"World";
