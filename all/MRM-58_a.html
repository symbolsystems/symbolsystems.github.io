The result of the memory allocation function should be cast immediately [MRM-58_a-3]

DESCRIPTION 

"Because objects returned by the C Standard memory allocation functions are
implicitly converted into any object type, we recommend casting the results
of these functions into a pointer of the allocated type because it increases
the chances that the compiler will catch and diagnose a mismatch between the
intended type of the object and the actual type of the object." [CERT-MEM02-C]

This rule detects when the result of a call to a standard C memory allocation
function (malloc, calloc, realloc or align_alloc) is not immediately cast
to some type.
SINCE 

2023.1
BENEFITS 

This rule helps you avoid a mismatch between the intended type of the
object and the actual type of the object.
EXAMPLE 

#include <stdlib.h>
void bar()
{
    int *p = malloc(sizeof(int));                  // Violation
}
REPAIR 

#include <stdlib.h>
void bar ()
{
    int *p = (int*)malloc(sizeof(int));            // OK
}
