Function templates shall not be explicitly specialized [MISRACPP2023-17_8_1-a-2]

DESCRIPTION 

Explicit template specializations are selected in overload resolution
only when a best match is chosen from the set of primary template functions.
This solution can be misleading for developers.

This rule detects explicit template specializations.
SINCE 

2023.2
NOTES 

This rule does not apply to non-template member functions of class
templates.
BENEFITS 

This rule prevents developer confusion.
EXAMPLE 

template <typename T> void f ( T );
template <> void f<int*> ( int* );      // Violation
REPAIR 

template <typename T> void f ( T );
void f( int* );                         // OK
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 17.8.1 Function templates shall not be explicitly specialized
