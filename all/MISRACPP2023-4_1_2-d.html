A constexpr static data member should not be declared outside the class definition [MISRACPP2023-4_1_2-d-4]

DESCRIPTION 

"For compatibility with prior C++ International Standards, a constexpr
static data member may be redundantly redeclared outside the class with
no initializer. This usage is deprecated." [ISO/IEC 14882:2017]

This rule detects static constexpr data members declared outside of
their class definitions.
SINCE 

2023.2
NOTES 

This rule does not apply to data members of template classes.
BENEFITS 

This rule helps you improve code readability and remove
unnecessary code.
EXAMPLE 

struct S {
    static constexpr int n = 5;
};
constexpr int S::n;              // Violation
REPAIR 

To fix your code, remove the declaration outside of the class.

struct S {
    static constexpr int n = 5;  // OK
};
