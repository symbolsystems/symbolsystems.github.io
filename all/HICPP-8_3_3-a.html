Do not use default arguments [HICPP-8_3_3-a-3]

DESCRIPTION 

"Use of default arguments can make code maintenance and refactoring more
difficult. Overloaded forwarding functions can be used instead without having
to change existing function calls." [HIC++ v4.0, Rule 8.3.3]

This rule identifies cases where default arguments are being used.
SINCE 

v10.4
BENEFITS 

Improves code maintainability.
EXAMPLE 

#include <cstdint>
void bar(int32_t i, int32_t j = 0); // Violation
REPAIR 

#include <cstdint>

void bar(int32_t i, int32_t j); // OK
void bar(int32_t i) {
    bar(i, 0); // OK - call other bar with second argument set to zero
}
REFERENCES 

1. High Integrity C++ Coding Standard
   8.3.3. Do not use default arguments
