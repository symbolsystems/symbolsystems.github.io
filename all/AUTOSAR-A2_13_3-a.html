Type wchar_t shall not be used [AUTOSAR-A2_13_3-a-2]

DESCRIPTION 

"Width of wchar_t type is implementation-defined.
Types char16_t and char32_t should be used instead"
[AUTOSAR A2-13-3]

The rule detects wchar_t types.
SINCE 

v10.4
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

wchar_t s[] = L"ABC";              // Violation
REPAIR 

char16_t s1[] = u"ABC";            // OK
char32_t s2[] = U"ABC";            // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A2-13-3. Type wchar_t shall not be used
