Derived classes that do not need further explicit initialization and require all the constructors from the base class shall use inheriting constructors [AUTOSAR-A12_1_6-a-2]

DESCRIPTION 

"Reimplementing constructors that do not need further initialization is
error-prone and may lead to using wrong base class constructor accidentally."
[AUTOSAR-A12-1-6]

Derived classes should inherit constructors from the base class if the
derived class doesn't need to initialize new data members and if
all of the constructors from the base class are required.
SINCE 

v10.4
BENEFITS 

This rule improves code readability and consistency with the C++ standard
library. This rule also helps prevent unexpected side effects and enforces
compliance with some safety-critical standards.
EXAMPLE 

class A
{
public:
    A(int x, int y) : x(x + 8), y(y) {}
    explicit A(int x) : A(x, 0) {}

private:
    int x;
    int y;
};

class B : public A                // Violation
{
public:
    B(int x, int y) : A(x, y) {}
    B(int x) : A(x) {}
};
REPAIR 

class A
{
public:
    A(int x, int y) : x(x + 8), y(y) {}
    explicit A(int x) : A(x, 0) {}

private:
    int x;
    int y;
};

class B : public A               // OK
{
public:
    using A::A;
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines

            and require all the constructors from the base class shall use
