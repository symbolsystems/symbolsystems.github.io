Document functions in comments that precede function definitions [COMMENT-04-3]

DESCRIPTION 

"Assumptions (limitations) made by functions should be documented in
the functionâ€™s preamble. . .  Maintenance efforts become very difficult if
the assumptions (limitations) upon which functions are built are unknown."
[JOINT STRIKE FIGHTER AV Rule 134]

This rule detects function definitions that are not preceded by a comment.
NOTES 

The rule does not analyze the content of comments.


Exception:

If a function has visible declaration preceded by a comment, then
the rule does not trigger on the uncommented definition.
BENEFITS 

The rule helps you improve code readability and maintainability.
EXAMPLE 

int foo1(int x){                                       // Violation
    return x++;
}

int foo2(int x);
int foo2(int x){                                       // Violation
    return x++;
}
REPAIR 

// returns squared x
int foo1(int x){                                       // OK
    return x*x;
}

// returns x multiplied by two
int foo2(int x);

int foo2(int x){                                       // OK
    return x*2;
}

int foo3(int x);

// returns x multiplied by three
int foo3(int x){                                       // OK
    return x*3;
}
