The literal value zero shall be the only value assigned to errno [CODSTA-239-3]

DESCRIPTION 

'errno' can be set to a non-zero value in the function. This may indicate
that an error has been detected. 'errno' should not be used as a mechanism for
reporting errors because C++ provides better mechanisms for error handling.

This rule detects when a value other than a literal zero is assigned to 'errno'.
SINCE 

2023.2
BENEFITS 

This rule helps you apply preferable mechanisms for error handling.
EXAMPLE 

#include <cerrno>
void f1(){
    errno = EINVAL;    // Violation
}

static const int VALUE = 0;

void f2(){
    errno = VALUE;    // Violation
}
REPAIR 

// Examples of correct code

#include <cerrno>
void f1(){
    errno = 0;      // OK
}

#define VALUE 0

void f2(){
    errno = VALUE; // OK
}
