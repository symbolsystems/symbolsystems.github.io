Do not increase the accessibility of overridden or hidden methods [OOP-54-3]

DESCRIPTION 

Increasing the accessibility of overridden or hidden methods allows greater
access than was originally intended for a method.

This rule detects when:
- a private method from a base class is overridden or hidden by a public or
protected method in a derived class
- a protected method from a base class is overridden or hidden by a public
method in a derived class.
SINCE 

v10.4
NOTES 

A method from a base class is hidden by a method in a derived class
if both methods have the same signature.
BENEFITS 

This rule helps you improve encapsulation and prevent unauthorized access to
restricted methods.
EXAMPLE 

class Base {
protected:
    virtual void f1();
    void f2(){ }
private:
    void f3();
};

class Derived : public Base {
public:
    virtual void f1();          // Violation
    using Base::f2;             // Violation
protected:
    void f3();                  // Violation
};
REPAIR 

class Base {
protected:
    virtual void f1();
    void f2(){ }
private:
    void f3();
};

class Derived : public Base {
protected:
    virtual void f1();          // OK
    using Base::f2;             // OK
private:
    void f3();                  // OK
};
