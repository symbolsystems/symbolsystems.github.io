Trivial accessor and mutator functions should be inlined [AUTOSAR-A3_1_6-a-4]

DESCRIPTION 

This rule detects trivial accessor and mutator functions that are not defined
as inline.
SINCE 

v7.1
NOTES 

This rule assumes that:
- an accessor function is a function that contains exactly one statement
  in which it returns the value of a data member.
- a mutator function is a function that contains exactly one argument
  and one statement in which that argument is assigned to a class member.
EXCEPTIONS 

This rule does not report virtual and template functions.
BENEFITS 

"Inlining trivial accessors and mutators saves time and space, as it reduces
multiple syntax elements that has to be repeated." [AUTOSAR A3-1-6]
EXAMPLE 

class A1 {
public:
    int getMember();
    void setMember(int param);
private:
    int m1;
};

int A1::getMember() {              // Violation
    return m1;
}

void A1::setMember(int param) {    // Violation
    m1 = param;
}
REPAIR 

To fix the code, define the functions as inline.

class A1 {
public:
    inline int getMember();
    inline void setMember(int param);
private:
    int m1;
};

inline int A1::getMember() {              // OK
    return m1;
}

inline void A1::setMember(int param) {    // OK
    m1 = param;
}
