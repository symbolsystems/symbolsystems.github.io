Do not use casts between essentially character types and essentially floating types [MISRA2012-RULE-10_5_c-4]

DESCRIPTION 

"An explicit cast may be introduced for legitimate functional reasons.
However, some explicit casts are considered inappropriate:
- Converting between floating and character types is not meaningful
  as there is no precise mapping between the two representations."
  [MISRA C:2012 Rule 10.5]

This rule detect cases when: 
- an expression of an essentially floating type is explicitly cast
  to an essentially character type
- an expression of an essentially character type is explicitly cast
  to an essentially floating type

The term "essential type" is defined by the MISRA C:2012 standard.
SINCE 

v9.6
BENEFITS 

This rule prevents confusion and helps avoid inappropriate explicit casts.
EXAMPLE 

void foo()
{
    char ch;
    float f;
    double complex dc;

    (float)ch;                         /* Violation */
    (double)'A';                       /* Violation */
    (float complex)'B';                /* Violation */
    (char)1.2;                         /* Violation */
    (char)f;                           /* Violation */
    (char)dc;                          /* Violation */
}
REPAIR 

Do not use casts between essentially character types and essentially floating types.
REFERENCES 

1. MISRA C:2012 Guidelines for the use of the C language in critical systems
   Section 8: Rules, Rule 10.5

2. MISRA C:2012 Amendment 3 Updates for ISO/IEC 9899:2011/2018
   Phase 2 - New C11/C18 Features
   2.3 Section 8 - Rules, 2.3.10 Rule 10.5
