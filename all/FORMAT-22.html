There should be no space between a increment/decrement operator (++/--) and its operand [FORMAT-22-3]

DESCRIPTION 

The rule reports a violation if there is space, tab, comment or new line
character between a increment/decrement operator and its operand. The rule
checks prefix and postfix operators.

See also: FORMAT-17, FORMAT-18, FORMAT-21, FORMAT-45, FORMAT-46
NOTES 

The rule does not report violations on operators used inside preprocessor
directives.

The rule does not report a violation on "operator" construct (declaration and
explicit call), for example:
A& operator ++ (A&); // no violation reported
operator++ (a); // no violation reported
BENEFITS 

The rule promotes continuity in the relationship between the operators
and the operands on which they act.
DRAWBACKS 

The rule can report incorrect violation if in the same line is a prefix
and postfix increment/decrement operator.
EXAMPLE 

void foo(int x) {
    x ++; /* Violation */
    x     /* Violation */
    --;
    ++ \
    x;    /* Violation in line above */
}

#define M1(x) (x) ++; /* OK - macro definitions not checked */
class A;
A& operator ++ (A&);  /* OK - no violation on "operator" construct */
void bar(A& a) {
    operator++ (a);   /* OK - no violation on "operator" construct */
    ++ a;             /* Violation */
}
REPAIR 

void foo(int x) {
    x++;    /* OK */
    --x;    /* OK */
    x--;    /* OK */
    ++x;    /* OK */
}

#define M1(x) (x)++; /* OK */
class A;
A& operator++(A&);   /* OK */
void bar(A& a) {
    operator++(a);   /* OK */
    ++a;             /* OK */
}
