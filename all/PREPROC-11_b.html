The #if preprocessor directive should not be used [PREPROC-11_b-3]

DESCRIPTION 

"Limit the use of the pre-processor to those cases where it is necessary."
[JOINT STRIKE FIGHTER]

Conditional code compilation can hinder testing and maintenance and should
therefore be kept to a minimum.

This rule detects when the #if preprocessor directive is used in code.
SINCE 

v7.1
BENEFITS 

This rule helps you improve code maintainability.
EXAMPLE 

#if M > 100        // Violation
// ..
#endif
REPAIR 

Do not use the #if preprocessor directive.
REFERENCES 

1. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.6 Pre-Processing Directives, AV Rule 26
