Identifiers shall be given for all of the parameters in a function prototype declaration [MISRA2012-RULE-8_2_a-2]

DESCRIPTION 

"This rule requires that names be specified for all the parameters
in a declaration. The parameter names can provide useful information
regarding the function interface and a mismatch between a declaration
and definition might be indicative of a programming error."[MISRA C:2012]

See also: CODSTA-158, CODSTA-159
NOTES 

Function types other than in function declarations are checked
by rule CODSTA-158.
BENEFITS 

The rule improves readability and clarity of code.
EXAMPLE 

void foo1(int);    /* Violation */
REPAIR 

void foo1(int p);       /* OK */

/* Examples not reported by this rule */
void foo2(int){}        /* OK - this code is compilable in C++ only */
void foo3(void);        /* OK */
void foo4(int p, ...);  /* OK */   
typedef void foot(int); /* OK - reported by rule CODSTA-158 */
