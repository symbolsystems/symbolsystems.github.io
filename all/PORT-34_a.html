User-specified C++ external identifiers should differ in the first 64 characters [PORT-34_a-3]

DESCRIPTION 

"User-specified identifiers (internal and external) will not rely
on significance of more than 64 characters." [JSF]

This rule detects when two external identifiers that are declared or defined
in the same translation unit have the same first 64 characters in their names.

The rule checks the names of the following identifiers:
- variables and functions with internal or external linkage
- classes, structures, unions, typedefs, enumerations, and enumeration
  constants that are defined in a namespace or class scope
- namespaces
SINCE 

2020.2
NOTES 

The rule compares identifiers across all namespaces.
The rule does not report identifiers with identical names.

This rule does not the check names of conversion operators.
BENEFITS 

This rule helps you improve code portability.
EXAMPLE 

extern void NG_4567890123456789012345678901234567890123456789012345678901234_v1();
static void NG_4567890123456789012345678901234567890123456789012345678901234_v2();  // Violation

namespace N1 {
    void  NA_4567890123456789012345678901234567890123456789012345678901234_v1();
}
namespace N2 {
    class NA_4567890123456789012345678901234567890123456789012345678901234_v2{};    // Violation
}
REPAIR 

You can fix the code by modifying the names of identifiers.

extern void V1_4567890123456789012345678901234567890123456789012345678901234_v1();
static void V2_4567890123456789012345678901234567890123456789012345678901234_v2();  // OK

namespace N1 {
    void  N1_4567890123456789012345678901234567890123456789012345678901234_v1();
}
namespace N2 {
    class N2_4567890123456789012345678901234567890123456789012345678901234_v2{};    // OK
}
