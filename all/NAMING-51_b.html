A user-defined literal suffix should be directly preceded by a literal operator [NAMING-51_b-3]

DESCRIPTION 

"Each identifier that begins with an underscore is reserved
to the implementation for use as a name in the global namespace."
[ISO/IEC 14882:2017]

It is recommended to begin a user-defined literal suffix with
a single underscore. However, it should not be preceded by a space,
comment or another character, because this results in the creation
of a reserved identifier.

This rule detects when a user-defined literal suffix is not
directly preceded by the literal operator 'operator""'.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid using reserved identifiers.
EXAMPLE 

double operator "" _ld(long double);    // Violation
REPAIR 

double operator ""_ld(long double);     // OK
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   5.10 Identifiers

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 5.10.1 User-defined identifiers shall have an appropriate form
