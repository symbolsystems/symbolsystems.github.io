Do not access iterator out of range [BD-CO-ITOUT-1]

DESCRIPTION 
This rule detects when an iterator is accessed out of range. A violation 
is reported when a begin iterator moves backward or when an end iterator 
is accessed or moves forward.
SINCE 
v10.1
NOTES 
N/A
SECURITY RELEVANCE 
N/A
PARAMETERS 
N/A
BENEFITS 
N/A
DRAWBACKS 
N/A
EXAMPLE 
Here is an example that triggers a violation:

#include <list>

using namespace std;

int* itout_bad(list<int> &list_to_reverse)
{
    int* tab = new int [list_to_reverse.size()];
    unsigned int i = 0;

    list<int>::iterator iter = list_to_reverse.end();
    while (i < list_to_reverse.size()) {
        tab[i] = *iter; // VIOLATION
        iter--;
        i++;
    }
    return tab;
}


The iterator "iter" is set to the end of "list_to_reverse" then accessed. 
But the end element is only a marker for the end list and doesn't 
represent any real value.
REPAIR 
You should move the iterator backward before accessing it or change the 
code so that it uses reverse_iterator.

#include <list>

using namespace std;

int* itout_fixed(list<int> &list_to_reverse)
{
    int* tab = new int [list_to_reverse.size()];
    unsigned int i = 0;

    list<int>::reverse_iterator iter = list_to_reverse.rbegin();
    while (i < list_to_reverse.size()) {
        tab[i] = *iter; // FIXED
        ++iter;
        ++i;
    }
    return tab;
}
