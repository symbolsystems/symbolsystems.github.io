The values returned by string comparison functions should be compared only to zero [PB-64-3]

DESCRIPTION 

"The 'memcmp', 'strcmp', 'strncmp', 'strcoll' functions return an integer
greater than, equal to, or less than zero." [ISO/IEC 9899:1999]

"The function basic_string::compare returns the nonzero result if the result
of the comparison is nonzero." [ISO/IEC 14882:2003]

The rule reports a violation if the result of global function 'memcmp',
'strcmp', 'strncmp', 'strcoll' or member function 'compare' from class
'basic_string' is compared to an expression other than:
- integer or enumeration constant with value '0'
- const variable of integer type that was initialized by integer
  or enumeration constant with value '0'
SINCE 

v9.5
BENEFITS 

Following this rule prevents unintentional behavior.
EXAMPLE 

#include <string.h>

void foo1() {
    char* ch1 = "str1";
    char* ch2 = "str2";
    int x = 0;
    
    if (strcmp(ch1, ch2) == 1) {                    // Violation
        // ... 
    }
    
    if (memcmp(ch1, ch2, sizeof(ch1)) > -1) {       // Violation
        // ... 
    }
    
    if (strcmp(ch1, ch2) == x) {                    // Violation
        // ... 
    }
}
REPAIR 

#include <string.h>

void foo1() {
    char* ch1 = "str1";
    char* ch2 = "str2";
    const int x = 0;
    
    if (strcmp(ch1, ch2) < 0) {                    // OK
        // ... 
    }
    
    if (memcmp(ch1, ch2, sizeof(ch1)) >= 0) {      // OK
        // ... 
    }

    if (strcmp(ch1, ch2) == x) {                   // OK
        // ... 
    }
}
