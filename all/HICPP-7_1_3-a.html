Do not place type specifiers before non-type specifiers in a declaration [HICPP-7_1_3-a-3]

DESCRIPTION 

"The C++ Standard allows any order of specifiers in a declaration. However, to
improve readability when a non-type specifier (typedef, friend, constexpr,
register, static, extern, thread local, mutable, inline, virtual, explicit)
appears in a declaration, it should be placed in the left-most position of the
declaration." [HIC++ v4.0, Rule 7.1.3]
SINCE 

v10.4
BENEFITS 

Improves readability
EXAMPLE 

typedef int int32_t; // OK
int typedef int32_e; // Violation
class C {
  public:
    virtual inline void f1(); // OK
    inline virtual void f2(); // OK
    void inline virtual f3(); // Violation
  private:
    int32_t mutable _i; // Violation
};
REPAIR 

typedef int int32_t; // OK
typedef int int32_e; // OK
class C {
  public:
    virtual inline void f1(); // OK
    virtual inline void f2(); // OK
    inline void virtual f3(); // OK
  private:
    mutable int32_t _i; // OK
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A7-1-8. A non-type specifier shall be placed before a type specifier in a
   declaration

2. High Integrity C++ Coding Standard
   7.1.3. Do not place type specifiers before non-type specifiers in a
   declaration
