An integer-literal of type long long shall not use a single 'L' or 'l' in any suffix [PORT-38-3]

DESCRIPTION 

The type of an integer-literal is dependent on a complex combination of factors
including:
- the magnitude of the integer-literal
- the implemented sizes of the integer types
- the presence of any suffixes
- the number base in which the value is expressed (i.e., decimal, octal or
  hexadecimal)

The 'L' (or 'l') suffix means the 'long' type, but in some cases, the real type
of the integer-literal can be the 'long long' type. In this case, the suffix
should be 'LL' (or 'll') to match the real type of the integer-literal.
SINCE 

2023.2
NOTES 

This rule does not apply to user-defined-literals.
This rule triggers only if 'long' and 'long long' types have different sizes.
BENEFITS 

The rule improves code readability and maintainability.
EXAMPLE 

This example assumes that 'long' is 32-bits and 'long long' is 64-bits.

auto k1 = 12345678998L;  // Violation
REPAIR 

You can fix the code by adding the 'LL' suffix.

auto k1 = 12345678998LL; // OK
