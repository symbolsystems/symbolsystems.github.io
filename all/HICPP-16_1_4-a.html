Use <> brackets for system and standard library headers. Use quotes for all other headers [HICPP-16_1_4-a-3]

DESCRIPTION 

Use angle brackets for compiler-provided headers (i.e., #include <. . .>) and
quotation marks for user-provided files (i.e., #include “. . .”). Adhering to
this convention improves the clarity and maintainability of the code.
SINCE 

v10.4
BENEFITS 

This rule provides a strict delineation between compiler provided headers and
user provided files. This provides an easy way to track down headers or other
files should the need arise.
EXAMPLE 

#include "cstddef" // Violation
#include <types.h> // Violation
REPAIR 

#include "types.h" // OK
#include <cstddef> // OK
REFERENCES
