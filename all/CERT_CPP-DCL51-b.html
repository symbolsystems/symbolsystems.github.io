Do not redefine reserved words [CERT_CPP-DCL51-b-3]

DESCRIPTION 

"It is generally bad practice to #undef or #define names which are C reserved words"
BENEFITS 

Redefinition of reserved words can lead to errors and confusion.
EXAMPLE 

#define break 1     // Violation
#define continue 1  // Violation
#undef while        // Violation
#undef return       // Violation
REPAIR 

Do not #define or #undef reserved words
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A17-0-1. Reserved identifiers, macros and functions in the C++ standard
   library shall not be defined, redefined or undefined

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 17, Rule 17-0-1

3. MISRA-C:2004 Guidelines for the use of the C language in critical systems
   Chapter 6, Section 20

4. Origin: Misra Guidelines - Rule 114
