The identifiers used in the declaration and definition of a function shall be identical [MISRA2012-RULE-8_3_b-2]

DESCRIPTION 

If identifiers are given for any of the parameters, then the identifiers used
in the declaration and definition shall be identical.
BENEFITS 

Rule improves readability and clarity of code.
EXAMPLE 

void foo(int a, int b);
void foo(int x, int y ) {}  /* Violation */
REPAIR 

void foo(int a, int b);    /* OK */
void foo(int a, int b) {}

/* Examples of correct code */
void foo1();               /* OK */
void foo2(int  , int   );  /* OK */
void foo2(int x, int y ) {}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M8-4-2. The identifiers used for the parameters in a re-declaration of a
   function shall be identical to those in the declaration

2. High Integrity C++ Coding Standard
   8.2.1. Make parameter names absent or identical in all declarations

3. MISRA C++:2008 Guidelines for the use of the C++ language in critical

4. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 13.3.3 The parameters in all declarations or overrides of a function shall either be unnamed or have identical names

5. MISRA C:2012 Guidelines for the use of the C language in critical systems
