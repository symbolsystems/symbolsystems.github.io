If you have a non-trivial copy constructor or copy assignment operator, you should also declare a destructor [MRM-40_c-3]

DESCRIPTION 

If you define a non-trivial copy constructor or copy assignment operator, you
should probably have a destructor as well. If the "special" work in the copying
functions is intended to allocate or duplicate some resources, you probably need
to deallocate it in the destructor.
SINCE 

v9.5
NOTES 

This rule only checks if the copy constructor or copy assignment operator is
non-empty. It does not check if you actually allocate some resources there.
BENEFITS 

Following this rule prevents memory management problems.
EXAMPLE 

struct A // Violation - no destructor
{
    int *p;
    A& operator=(const A&o) {
        p = new int(*o.p);
        return *this;
    }
};
REPAIR 

struct A // OK - destructor declared
{
    int *p;
    A& operator=(const A&o) {
        p = new int(*o.p);
        return *this;
    }
    ~A();
};

class B // OK - empty copy constructor
{
    B(const B&) {
    }
};
