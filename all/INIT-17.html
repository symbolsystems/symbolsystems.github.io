The data members initialized with the same constant values across all constructors should be initialized using NSDMI [INIT-17-3]

DESCRIPTION 

"If all user-defined constructors of a class initialize data members with
constant values that are the same across all constructors, then data
members shall be initialized using NSDMI instead." [AUTOSAR A12-1-3]

This rule checks if data members in a class are intialized to the same
constant value in all user-defined constructors.
SINCE 

v10.4
BENEFITS 

This rule improves code readability and consistency with the C++ standard.
The rule also allows the compiler to generate functions that are more
efficient than user-defined constructors.
DRAWBACKS 

This rule does not support member variables being intialized to nothing and
member variables being initialized with numeric operators.
EXAMPLE 

#include <cstdint>
#include <string>

class B2                 // Violation
{
public:
    B2() : x(0), y(0.0F), str("a") {}
    B2(int a) : x(0), y(0.0F), str("a") {}

private:
    std::int32_t x;
    float y;
    std::string str;
};
REPAIR 

#include <cstdint>
#include <string>

class B2                 // OK
{
public:
    B2() {}
    B2(int a) {}

private:
    std::int32_t x = 0;
    float y = 0.0F;
    std::string str = "a";
};
