A for-range-initializer shall contain at most one function call [MISRACPP2023-9_5_2-a-2]

DESCRIPTION 

This rule reports when the initializer expression of a range-based
'for' statement contains more than one function call.

The following expressions are considered as function calls:
- explicit function calls
- implicit function calls (for example constructors and operators)
- any expression creating an object of class type,
  even if no constructor is actually called
SINCE 

2023.2
BENEFITS 

This rule helps you prevent undefined behavior related to object lifetime
violations when the for-range-initializer of a range-based 'for' statement
creates a temporary object.
EXAMPLE 

struct S {
    const char* begin() const;
    const char* end() const;
    const S& operator[](int) const;
    operator int() const;
    const S& baz() const;
};
int bar();
S make(int);
void foo(S s)
{
    for (auto i: make(bar()));      // Violation: make(), bar()
    for (auto i: make(0)[1]);       // Violation: make(), operator[]
    for (auto i: (new S)->baz());   // Violation: class creation, baz()
    for (auto i: make(s));          // Violation: operator int(), make()
}
REPAIR 

Use at most one function call in the for-range-initializer.
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
