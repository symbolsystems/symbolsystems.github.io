Narrow and wide string literals shall not be concatenated [PB-38-3]

DESCRIPTION 

Concatenation of wide and narrow string literals leads to undefined behaviour.
SINCE 

v7.2
BENEFITS 

Rule prevents undefined behaviour.
EXAMPLE 

wchar_t array[] = "Hello" L"World";    // Violation
REPAIR 

wchar_t array[] = L"Hello" L"World";   // OK
REFERENCES 

1. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 2, Rule 2-13-5

2. SEI CERT C Coding Standard
   STR10-C. Do not concatenate different type of string literals
   https://wiki.sei.cmu.edu/confluence/display/c/STR10-C.+Do+not+concatenate+different+type+of+string+literals
