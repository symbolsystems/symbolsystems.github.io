User-specified C++ internal identifiers should differ from external identifiers in the first 64 characters [JSF-046_b-2]

DESCRIPTION 

"User-specified identifiers (internal and external) will not rely
on significance of more than 64 characters." [JSF]

This rule detects when an internal identifier has the same first 64 characters
in its name as an external identifier declared or defined in the same
translation unit.

The rule checks the following external identifiers:
- variables and functions with internal or external linkage
- classes, structures, unions, typedefs, enumerations, and enumeration
  constants that are defined in a namespace or class scope
- namespaces

The rule checks the following internal identifiers:
- variables and functions with no linkage
- function parameters
- labels
- classes, structures, unions, typedefs, enums, and enumeration constants
  that are defined in a local scope
- template parameters and template parameter constants
SINCE 

2020.2
NOTES 

The rule compares identifiers across all namespaces.
The rule does not report identifiers with identical names.

This rule does not check the names of conversion operators.
BENEFITS 

This rule helps you improve code portability.
EXAMPLE 

void NE_4567890123456789012345678901234567890123456789012345678901234_v1();
void f1() {
    int NE_4567890123456789012345678901234567890123456789012345678901234_v2;       // Violation
}

static void NS_4567890123456789012345678901234567890123456789012345678901234_v1();
void f1 (int NS_4567890123456789012345678901234567890123456789012345678901234_v2);  // Violation
REPAIR 

You can fix the code by modifying the names of identifiers.

void NE_4567890123456789012345678901234567890123456789012345678901234_v1();
void f1() {
    int NL_4567890123456789012345678901234567890123456789012345678901234_v2;       // OK
}

static void NS_4567890123456789012345678901234567890123456789012345678901234_v1();
void f1 (int NP_4567890123456789012345678901234567890123456789012345678901234_v2);  // OK
