Each expression-statement will be on a separate line [JSF-042-3]

DESCRIPTION 

"Each expression statement and identifier declaration shall be placed
on a separate line." [AUTOSAR A7-1-7]

This rule detects lines that contain one of the following:
- more than one semicolon.
- a semicolon that is not the last token on the line.
EXCEPTIONS 

The rule assumes that two semicolons that separate expressions in the header
of the 'for' loop can be placed on the same line, for example:

  void f() {
      int i;
      for( i = 0; i < 10; ++i) {   // OK
          //...
      }
  }
BENEFITS 

"Declaring an identifier on a separate line makes the identifier declaration
easier to find and the source code more readable." [AUTOSAR A7-1-7]
EXAMPLE 

void bar(int arg);

void foo()
{
    int i; char c;                   // Violation
    if (i == 0) { i++; bar(i); }     // Violation
}
REPAIR 

You can fix your code by placing each statement on a separate line.

void bar(int arg);

void foo()
{
    int i;                           // OK
    char c;                          // OK
    if( i == 0) {                    // OK
        i++;                         // OK
        bar(i);                      // OK
    }
}
