An enumeration shall be defined with an explicit underlying type [MISRACPP2023-10_2_1-a-2]

DESCRIPTION 

This rule reports when an enum is defined without an explicit underlying type.
SINCE 

2023.2
EXCEPTIONS 

The rule does not report when:
* an enum is declared in an extern "C" block, or
* all of the enumerators use their default values.
BENEFITS 

This rule helps you avoid unexpected or implementation-defined behavior.
EXAMPLE 

enum class Enum             // Violation
{
  E0 = 0,
  E1,
  E2
};
REPAIR 

enum class Enum : uint8_t  // OK
{
  E0 = 0,
  E1,
  E2
};

// Examples of code compliant by exception:

enum class Enum2             // OK
{
  E0,
  E1,
  E2
};

extern "C"
{
  enum class Enum3            // OK
  {
    E0 = 0,
    E1,
    E2
  };
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 10.2.1 An enumeration shall be defined with an explicit underlying type
