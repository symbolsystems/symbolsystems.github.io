A thread synchronization object should not be initialized by a thread [BD-TRS-THRINIT-3]

DESCRIPTION 
This rule detects cases when a thread synchronization object is initialized inside of a thread.

For a list of recognized thread-creating functions, see the rule BD-TRS-THRTHR.

This rule recognizes the following initialization functions:

<threads.h> (C Standard Library)
    int mtx_init( mtx_t* mutex, int type );
    int cnd_init( cnd_t* cond );

<thread> (C++ Standard Library)
    std::mutex::mutex();
    std::timed_mutex::timed_mutex();
    std::recursive_mutex::recursive_mutex();
    std::recursive_timed_mutex::recursive_timed_mutex();
    
<pthread.h> (POSIX)
    int pthread_mutex_init(pthread_mutex_t *mutex, const pthread_mutexattr_t *attr);
    int pthread_cond_init(pthread_cond_t *restrict cond, const pthread_condattr_t *restrict attr);
    
<synhapi.h> (Windows)
  HANDLE CreateMutex (LPSECURITY_ATTRIBUTES lpMutexAttributes, BOOL bInitialOwner, LPCTSTR lpName);
  HANDLE CreateMutexA (LPSECURITY_ATTRIBUTES lpMutexAttributes, WINBOOL bInitialOwner, LPCSTR lpName);
  HANDLE CreateMutexW (LPSECURITY_ATTRIBUTES lpMutexAttributes, WINBOOL bInitialOwner, LPCWSTR lpName);
  HANDLE CreateMutexExA (LPSECURITY_ATTRIBUTES lpMutexAttributes, LPCSTR lpName, DWORD dwFlags, DWORD dwDesiredAccess);
  HANDLE CreateMutexExW (LPSECURITY_ATTRIBUTES lpMutexAttributes, LPCWSTR lpName, DWORD dwFlags, DWORD dwDesiredAccess);    
  
  HANDLE CreateSemaphore(LPSECURITY_ATTRIBUTES lpSemaphoreAttributes, LONG  lInitialCount, LONG lMaximumCount, LPCTSTR lpName);
  HANDLE CreateSemaphoreA(LPSECURITY_ATTRIBUTES lpSemaphoreAttributes, LONG  lInitialCount, LONG lMaximumCount, LPCSTR lpName);
  HANDLE CreateSemaphoreExW (LPSECURITY_ATTRIBUTES lpSemaphoreAttributes, LONG lInitialCount, LONG lMaximumCount, LPCWSTR lpName, DWORD dwFlags, DWORD dwDesiredAccess);

  VOID InitializeConditionVariable (PCONDITION_VARIABLE ConditionVariable);

  VOID InitializeCriticalSection (LPCRITICAL_SECTION lpCriticalSection);
  WINBOOL InitializeCriticalSectionAndSpinCount (LPCRITICAL_SECTION lpCriticalSection, DWORD dwSpinCount);
  WINBOOL InitializeCriticalSectionEx (LPCRITICAL_SECTION lpCriticalSection, DWORD dwSpinCount, DWORD Flags);

  WINBOOL InitializeSynchronizationBarrier (LPSYNCHRONIZATION_BARRIER lpBarrier, LONG lTotalThreads, LONG lSpinCount);
    
If any of these functions is called (directly or indirectly) from a function which itself
is a thread entry point, the rule will report a violation.
SINCE 
2023.1
SECURITY RELEVANCE 
Initializing a thread synchronization object inside of a thread can cause 
undefined behavior. It can cause a thread synchronization object to be accessed 
before it is initialized.
BENEFITS 
This rule helps you prevent threads from accessing thread synchronization objects
with indeterminate state.
EXAMPLE 

#include<pthread.h>

pthread_t thread;
pthread_mutex_t mutex;           

void *foo( void* ignore )
{
    pthread_mutex_init(&mutex, NULL); // VIOLATION
    return 0;
}

int main(void)
{
    pthread_create(&thread, NULL, &foo, NULL);
    return 0;
}
REPAIR 
To repair the code, initialize the thread synchronization object outside of the created thread.

#include<pthread.h>

pthread_t thread;
pthread_mutex_t mutex;           

void *foo( void* ignore )
{             
    return 0;
}

int main(void)
{
    pthread_mutex_init(&mutex, NULL); // NO VIOLATION
    pthread_create(&thread, NULL, &foo, NULL);             
    return 0;
}
