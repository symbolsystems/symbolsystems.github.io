Do not use static keyword except inside functions and classes [CODSTA-CPP-50-3]

DESCRIPTION 

"In C and older C++ programs, the keyword 'static' is (confusingly) used
to mean 'use internal linkage'. Don’t use 'static' except inside functions
and classes. An unnamed namespace can be used to make names local to
a compilation unit. The effect of an unnamed namespace is very similar to that
of internal linkage."

"The use of 'static' to indicate 'local to translation unit' is deprecated
in C++. Use unnamed namespaces instead." [The C++ Programming Language]

This rule detects when a non-const global object or global function
is explicitly declared as static.
SINCE 

v7.0
EXCEPTIONS 

"Anonymous unions declared in a named namespace or in the global namespace
shall be declared static." [ISO/IEC 14882]
BENEFITS 

The rule prevents using deprecated language structures.
EXAMPLE 

static int var = 10;         // Violation
​
static int f1();             // Violation
static int f1()              // Violation
{
  return 1;
}
REPAIR 

namespace
{
  int var = 10;              // OK
​
  int f1();                  // OK
  int f1()                   // OK
  {
    return 1;
  }
}

// Examples of code not detected by this rule
​
static const int VAR = 100;  // OK 
​
class A {
public:
  static const int mem;      // OK
};
​
void foo() {
  static int i;              // OK
}
​
static union {               // OK
  int memU;
};
