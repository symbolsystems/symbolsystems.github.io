Neither operand of an equality operator (== or !=) shall be a pointer to a virtual member function [CODSTA-CPP-79-3]

DESCRIPTION 

The rule reports a violation if either operand of an equality operator
(== or !=) is a pointer to a virtual member function. The result of such
operation is unspecified according to C++ standard section 5.10(2):

  "If both operands are null, they compare equal.
   Otherwise if only one is null, they compare unequal.
   Otherwise if either is a pointer to a virtual member function,
   the result is unspecified."

Note that although the standard allows comparing pointer to member with null,
this rule reports a violation - such comparison is pointless as it is always
false.
SINCE 

v9.4
BENEFITS 

The rule prevents undefined behaviours.
EXAMPLE 

class A {
public:
    virtual void foov();
};
class B : public A {
public:
    virtual void foov();
};
typedef void (A::*MFP)();
int bar(MFP fp){
    if(fp == A::foov){ // Violation
        return 1;
    }
    if(A::foov == B::foov){ // Violation
        return 0;
    }
    if(0 != &A::foov){ // Violation
        return 1;
    }
    if(A::foov == 0){ // Violation
        return 0;
    }
    if(fp != 0){ // OK - rule does not check variables
        return 1;
    }
    if(!A::foov) { // OK - rule checks explicit "==", "!=" operators
        return 0;
    }
}
REPAIR 

Neither operand of an equality operator (== or !=) shall be a pointer
to a virtual member function
