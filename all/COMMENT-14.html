All declarations of types, data members, and functions should be preceded by a comment annotated with the '@brief' tag [COMMENT-14-3]

DESCRIPTION 

"All declarations of 'user-defined' types, static and non-static data members,
functions and methods shall be preceded by documentation." [AUTOSAR-A2-7-3]

"Every declaration needs to provide a proper documentation. This is compatible
with the C++ standard library documentation. This forces a programmer to
provide a clarification for defined types and its data members
responsibilities, methods and functions usage. . . " [AUTOSAR-A2-7-3]

The rule detects when a user-defined type, a data member, or a function is not
preceded by a comment that contains the '@brief' tag.
SINCE 

v10.4
BENEFITS 

This rule helps you improve code readability and maintainability.
EXAMPLE 

class C                          // Violation
{
public:
    int member;                  // Violation
};

void foo(int arg);               // Violation
REPAIR 

You can fix the code by adding a comment that contains the '@brief' tag before
the declarations.

// @brief description of class C
class C                          // OK
{
public:
    // @brief description of variable m
    int m;                       // OK
};

// @brief description of a function foo
void foo(int arg);               // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A2-7-3 All declarations of "user-defined" types, static and non-static data
   members, functions and methods shall be preceded by documentation.
