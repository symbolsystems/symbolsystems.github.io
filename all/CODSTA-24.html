The macro FALSE should be defined as 0 (zero) [CODSTA-24-5]

DESCRIPTION 

The rule reports a violation if the macro 'FALSE' is not defined as 0 (zero).
It is allowed to use decimal, octal or hexadecimal values (e.g. 0000, 0x0000)
and the number 0 can be followed by a suffix (e.g. 0u, 0l, 0UL).

See also: CODSTA-25, CODSTA-42, CODSTA-43
BENEFITS 

The rule improves readability and maintainability of code.
EXAMPLE 

#define FALSE 1   // Violation
REPAIR 

#define FALSE 0   // OK
