Whenever an object is declared or defined, its type shall be explicitly stated [MISRAC2012-RULE_8_1-b-2]

DESCRIPTION 

The rule reports violations on declarations of parameters and variables where 
the type is not explicitly stated.
BENEFITS 

Rule improves readability and maintainability.
EXAMPLE 

extern x;     /* Violation  */
const y;      /* Violation  */
z;            /* Violation  */

void foo(const i);  /* Violation  */
void foo(const i){} /* Violation  */
REPAIR 

extern int x;     /* OK  */
const int y;      /* OK  */
int z;            /* OK  */

void foo(const int i);  /* OK  */
void foo(const int i){} /* OK  */
REFERENCES 

1. MISRA-C:2004 Guidelines for the use of the C language in critical systems
   Chapter 6, Section 8

2. MISRA C:2012 Guidelines for the use of the C language in critical systems
   Section 8: Rules, Rule 8.1
