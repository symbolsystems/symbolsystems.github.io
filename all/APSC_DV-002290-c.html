The 'random_shuffle' identifier should not be used [APSC_DV-002290-c-2]

DESCRIPTION 

"All code shall conform to ISO/IEC 14882:2014 - Programming Language C++ and
shall not use deprecated features." [AUTOSAR-A1-1-1]

The 'random_shuffle' function template from the Standard Library was deprecated
in the C++11 and removed in the C++17 version of the C++ Language Standard.

This rule detects when the 'random_shuffle' identifier is used in code.
SINCE 

v10.4
NOTES 

This rule reports all occurrences of the 'random_shuffle' identifier. For this
reason, the rule may report 'random_shuffle' that does not represent a function
from the Standard Library. For example:

int random_shuffle; // Violation
BENEFITS 

This rule helps you ensure that your code complies with the up-to-date version
of the C++ Language Standard. This allows you to increase longevity of
an application and address the defects and vulnerabilities that may emerge
as deployment environments evolve.
EXAMPLE 

#include <vector>

void foo()
{
    std::vector<int> myvector;
    std::random_shuffle ( myvector.begin(), myvector.end() );   // Violation
}
REPAIR 

Do not use the 'random_shuffle' identifier.
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
