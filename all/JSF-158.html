The operands of a logical && or || shall be parenthesized if the operands contain binary operators [JSF-158-2]

DESCRIPTION 

The rule reports a violation if an operand of a logical operator '&&' or '||'
containing the binary operator is not parenthesized.

Where an expression consists of either a sequence of only logical '&&'
or a sequence of only logical '||', extra parentheses are not required.

See also: CODSTA-90, MISRA2004-12_5
SINCE 

v9.5
BENEFITS 

"Parenthesizing operands (that contain binary operators) of the logical
operators '&&' or '||' enhances readability"
EXAMPLE 

void foo( int x, int y, int z )
{
    if ( x != y && x != z );   // Violation
    if ( x == y || x == z );   // Violation
}
REPAIR 

void foo( int x, int y, int z )
{
    if ( (x != y) && (x != z) );   // OK
    if ( (x == y) || (x == z) );   // OK
}
