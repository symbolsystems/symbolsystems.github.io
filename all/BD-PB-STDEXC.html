Always throw the created std::exception object [BD-PB-STDEXC-3]

DESCRIPTION 
This rule detects when objects of type std::exception 
or any derived type are created, but are not thrown. 
"If an instance of an object inheriting from std::exception 
is created, then such an object must appear in a throw expression."
[High Integrity C++ Coding Standard; 15.1.1 Only use instances
of std::exception for exceptions]
SINCE 
v10.4
NOTES 
N/A
SECURITY RELEVANCE 
N/A
PARAMETERS 
The "Report unvalidated violations" parameter can result in more violations,
but possibly also more false positives, being reported. In this mode Flow 
Analysis will not check whether the path of the violation can be reached from
the beginning of the function. In some complicated cases, restricted analysis
depth may prevent Flow Analysis from completing the violation validation
procedure. If this happens, violation can be found only with this parameter 
turned on.
BENEFITS 
N/A
EXAMPLE 
Here is an example that triggers a violation:

#include <exception>

class MyException : public std::exception
{

};

void f1 ()
{
    MyException myExcp;    // VIOLATION
}
REPAIR 
To repair the code, use "myExcp" in the throw statement:

#include <exception>

class MyException : public std::exception
{

};

void f2 ()
{
    MyException myExcp;    // NO VIOLATION
    throw myExcp;
}
