Type long double shall not be used [PORT-32-3]

DESCRIPTION 

"The width of long double type, and therefore width of the significand,
is implementation-defined.

The width of long double type can be either:
    - 64 bits, as the C++14 Language Standard allows long double to provide
      at least as much precision as type double does, or
    - 80 bits, as the IEEE 754 standard allows extended precision formats, or
    - 128 bits, as the IEEE 754 standard defines quadruple precision format"
[AUTOSAR A0-4-2]

The rule detects 'long double' types.
SINCE 

v10.4
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

long double d;                // Violation
REPAIR 

double d;                     // OK
