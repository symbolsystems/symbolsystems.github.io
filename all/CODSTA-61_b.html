The first or the last label of a switch statement should be the 'default' label unless all enumeration values are tested [CODSTA-61_b-3]

DESCRIPTION 

Using the 'default' label in every switch statement is a defensive programming
practice and ensures that the programmer will consider a scenario where none of
the other options have been selected. Placing the 'default' label as the first
or the last label in a switch makes the code easier to review.

This rule detects when a 'switch' statement does not contain the 'default'
label or when the 'default' label is not the first or the last label
in a switch.
SINCE 

2023.2
EXCEPTIONS 

The 'default' label is not required when the condition of a switch contains
an expression of an unscoped enumeration type that does not have a fixed
underlying type, and all its enumerators are used as indexes in case labels.
BENEFITS 

This rule helps you to improve the readability and maintainability of the code.
EXAMPLE 

enum Colours { RED, BLUE, GREEN } colour;

void f1(int i)
{
    switch( i )              // Violation
    {
        case 0:
            break;
        case 1:
        case 2:
            break;
    }

    switch( colour )         // Violation
    {
        case RED:
            break;
        case GREEN:
            break;
    }
}
REPAIR 

enum Colours { RED, BLUE, GREEN } colour;

void foo(int i)
{
    switch( i )              // OK
    {
        case 0:
            break;
        case 1:
        case 2:
            break;
        default:
            // Suitable comment
            break;
    }

    switch( colour )         // OK - Exception
    {
        case RED:
            break;
        case BLUE:
            break;
        case GREEN:
            break;
    }
}
