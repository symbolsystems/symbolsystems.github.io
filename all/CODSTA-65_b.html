An object with enumeration type shall not be converted to an object with pointer type [CODSTA-65_b-3]

DESCRIPTION 

Converting from an enumeration type to a pointer to an object leads
to unspecified behavior.

This rule detects when an expression of enumeration type is explicitly
cast to a pointer to object or pointer to void type.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid unspecified behavior.
EXAMPLE 

enum E { E1, E2 };

struct S{
    int x;
};

void f (E e)
{
    S * s1 = reinterpret_cast< S * >( e ); // Violation
    S * s2 = (S*)e;                        // Violation
}
REPAIR 

Do not cast an object with enumeration type to an object with pointer type.
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 8.2.6 An object with integral, enumerated, or pointer to void type shall not be cast to a pointer type
