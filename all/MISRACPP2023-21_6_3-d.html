Advanced memory management operators 'new' and 'delete' should not be used [MISRACPP2023-21_6_3-d-2]

DESCRIPTION 

Using the advanced memory management operators 'new' and 'delete' causes
a number of complex issues, such as alignment or object lifetimes, which
may result in undefined behavior.

This rule detects when the advanced memory management operator 'new'
or 'delete' is used.
SINCE 

2023.2
NOTES 

The advanced memory management operators are all overloads of operator
'new' and operator 'delete' that are not listed below:

void * operator new  ( std::size_t count );
void * operator new[]( std::size_t count );
void * operator new  ( std::size_t count, const std::nothrow_t & tag );
void * operator new[]( std::size_t count, const std::nothrow_t & tag );
void operator delete  ( void * ptr ) noexcept;
void operator delete[]( void * ptr ) noexcept;
void operator delete  ( void * ptr, std::size_t sz ) noexcept;
void operator delete[]( void * ptr, std::size_t sz ) noexcept;
void operator delete  ( void * ptr, const std::nothrow_t & tag ) noexcept;
void operator delete[]( void * ptr, const std::nothrow_t & tag ) noexcept;
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

#include <new>
 
int foo()
{
  void* pVoid1;
  void* pVoid2;
  int *pInt;
  char buf[sizeof(int)*3] ;
  
  pInt = new (buf) int(3);                                     // Violation
  pV2 = ::operator new(sizeof(pV1), pV1);                      // Violation
  operator delete(pV2, pV1);                                   // Violation
}
REPAIR 

Use only the allowed versions of 'new' and 'delete'.
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 21.6.3 Advanced memory management shall not be used
