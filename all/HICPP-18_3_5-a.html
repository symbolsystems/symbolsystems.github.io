Do not access the members of std::mutex directly [HICPP-18_3_5-a-3]

DESCRIPTION 

std::lock_guard and std::unique_lock provide RAII scoping mechanisms to
automatically acquire and release the mutex lock when required. Manually
locking or unlocking mutexes is error-prone and leads to code instability.
SINCE 

v10.4
BENEFITS 

This rule improves the stability and quality of the code. It prevents race
conditions and deadlocks, thereby improving code security.
EXAMPLE 

#include <mutex>

void f() {
    static std::mutex mut;
    mut.lock(); // Violation
    // ...
    mut.unlock(); // Violation
}
REPAIR 

#include <mutex>

std::mutex mut;

void f() {
    static std::mutex mut;
    std::lock_guard<std::mutex> guard(mut); // OK
    // ...
    return; // mutex automatically unlocked here.
}
REFERENCES 

1. High Integrity C++ Coding Standard
   18.3.5. Do not access the members of std::mutex directly
