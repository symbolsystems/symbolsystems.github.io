All the members of a structure (or class/union) shall be named [MISRA-113-3]

DESCRIPTION 

"All the members of a structure (or class/union) shall be named and shall only
be accessed via their names. Reading/writing to unnamed locations in memory
is error prone."

"Provided the elements of the structure are only ever accessed by their name,
the programmer needs to make no assumptions about the way the bit fields are
stored within the structure."

The rule reports a violation if an unnamed member variable of bit field type
is declared in class/struct/union.

See also: MISRA2004-3_5, MISRA2004-6_4
EXCEPTIONS 

Unnamed bit-fields of width zero are allowed.
BENEFITS 

Prevents using unaccessible data.
EXAMPLE 

typedef signed int t;

struct S {
    unsigned :4;          /* Violation */
    const t:5;            /* Violation */
    enum E{E1, E2} :5;    /* Violation */
};
REPAIR 

typedef signed int t;

struct S {
    unsigned       bf1:4;    /* OK */
    const t        bf2:5;    /* OK */
    enum E{E1, E2} bf3:5;    /* OK */
    
    unsigned :0; /* OK - Exception */
};
