Universal character names shall be used only inside character or string literals [NAMING-52-3]

DESCRIPTION 

"Using universal-character-names to define a language identifier
can be confusing for a developer and may be troublesome to use
this identifier in the source code." [AUTOSAR A2-13-6]

This rule detects universal character names, unless they are used inside:
- comments.
- character or string literals.

See also: PREPROC-21
SINCE 

v10.4
NOTES 

Universal character names have the form of "\uhhhh" or "\Uhhhhhhhh",
where h stands for a hexadecimal digit.
BENEFITS 

The rule helps you improve code readability and prevent confusion.
EXAMPLE 

void \u1111();        // Violation
int var\U00011111;    // Violation
REPAIR 

Do not use universal character names.

// Exceptions:
void f1() {
    '\u0001';      // OK
    "\U00011111";  // OK
    /* \u1111 */   // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A2-13-6. Universal character names shall be used only inside character or
   string literals
