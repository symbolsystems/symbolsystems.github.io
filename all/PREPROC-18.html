The input/output library stdio.h shall not be included [PREPROC-18-3]

DESCRIPTION 

"Streams and file I/O have a large number of unspecified, 
undefined and implementation-defined behaviours associated with them.
It is assumed that they will not normally be needed in production 
code in embedded systems.
If any of the features of stdio.h need to be used in production code, 
then the issues associated with the feature need to be understood."

This rule recognizes following #include directives:
    #include <stdio.h>
    #include <cstdio>

See also: MISRA2004-20_9
SINCE 

v9.5
NOTES 

1. The only recognized form of #include directive is the form with the '< >'
delimiters. Please see Example section.

2. This rule cross-checks original source files (#include directives) and
preprocessed output (#line's). A violation is reported only if the stdio header
is actually included in current translation unit (a #line for stdio.h or cstdio
file is present).

If the stdio header is actually included in current translation unit,
a violation is reported on all #include directives, even if they are 
conditionally compiled. For example:
   /* header.h: */
   #include <stdio.h>    /* Violation reported */
   /* source.c: */
   #include "header.h"
   #if 0
     /* A violation is reported on following #include directive because in 
      * current translation unit the stdio file is included somewhere    
      */
     #include <stdio.h>  /* Violation reported */
   #endif
BENEFITS 

Prevents form problems associated with a large number of unspecified,
undefined and implementation-defined behaviour associated 
with streams and file I/O.
EXAMPLE 

#include <stdio.h>  /* Violation */
#include <cstdio>   /* Violation */

/* #include directives which use quotes as delimiters are not recognized: */
#include "stdio.h"  /* OK - "stdio.h" not recognized */
#include "cstdio"   /* OK - "cstdio" not recognized */

/* #include direcives which use macro to specify header are not recognized: */
#define HEADER <stdio.h>
#include HEADER     /* OK - macros not recognized */
REPAIR 

Do not use stdio library.
