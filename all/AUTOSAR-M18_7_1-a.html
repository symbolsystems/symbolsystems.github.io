The signal handling facilities of <signal.h> shall not be used [AUTOSAR-M18_7_1-a-2]

DESCRIPTION 

"The standard header file <signal.h> shall not be #included, and none of the
features that are specified as being provided by <signal.h> shall be used."
[MISRA C:2012 Rule 21.5 with Amendment 3]

The rule detects the use of the following facilities:
- the 'signal' and 'raise' functions
- the 'sig_atomic_t' type
- the SIGABRT, SIGFPE, SIGILL, SIGINT, SIGSEGV, SIGTERM, SIG_DFL, SIG_IGN,
  SIG_ERR macros
SINCE 

v9.5
NOTES 

This rule detects all identifiers that have the same names as the facilities
from the <signal.h> header file.
BENEFITS 

The rule helps you prevent implementation-defined and undefined behavior.
EXAMPLE 

#include <signal.h>

void f1 (int sig)
{
    signal (sig, SIG_IGN);       // Violation
}
REPAIR 

Do not use facilities from the <signal.h> header.
