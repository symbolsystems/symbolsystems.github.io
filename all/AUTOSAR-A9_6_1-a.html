Types of bit-fields should have defined sizes [AUTOSAR-A9_6_1-a-2]

DESCRIPTION 

The use of bit-fields is allowed for interfacing with hardware or conforming
to communication protocols, so they should have only types that have a defined
size.
This rule detects bit-fields that have types whose size is not defined.
SINCE 

2023.2
NOTES 

This rule assumes that types with defined sizes are only fixed width integer
types: std::int8_t, std::int16_t, std::int32_t, std::int64_t, std::uint8_t,
std::uint16_t, std::uint32_t, std::uint64_t, and enumeration types if they
have been explicitly declared with a fixed width integer type.
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

enum E
{
    E1, E2
};

struct S
{
    char var1 : 2;     // Violation
    E var2 : 2;        // Violation
};
REPAIR 

#include <cstdint>
enum E : std::int32_t
{
    E1, E2
};

struct S
{
    std::int8_t var1 : 2;     // OK
    E var2 : 2;               // OK
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A9-6-1. Data types used for interfacing with hardware or conforming to
   communication protocols shall be trivial, standard-layout and only contain
   members of types with defined sizes.
