The declaration should not contain more than one level of pointer indirection [CODSTA-89-3]

DESCRIPTION 

Use of more than one level of pointer indirection can seriously impair
the ability to understand the behaviour of the code, and should therefore
be avoided.

See also: CODSTA-05, CODSTA-94, CODSTA-95, MISRA2004-17_5
SINCE 

v7.3
EXCEPTIONS 

This rule does not apply to arguments of the 'main' function.

int main(int argc, char** argv) {       // OK - Exception
    return 0;
}
BENEFITS 

This rule helps you improve the readability and maintanability of your
code.
EXAMPLE 

typedef int** INTPTR;  /* Violation */
int** ptr;             /* Violation */
int* a[];              /* Violation */
struct S{
    int** s;           /* Violation */
};

struct S** ps;         /* Violation */

int** foo();           /* Violation */
int(**pfunc1)();       /* Violation */
int**(*pfunc2)();      /* Violation */
void fun( int** par,   /* Violation */
          INTPTR par_t /* Violation */
        )
{
    int* a[10];        /* Violation */
    int ** ptr;        /* Violation */
    INTPTR ptr_t = 0;  /* Violation */
}
REPAIR 

Do not declare more than one level of pointer indirection.
