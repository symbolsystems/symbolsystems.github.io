The raw pointer constructors of 'std::shared_ptr' and 'std::unique_ptr' should not be used [CODSTA-MCPP-57-3]

DESCRIPTION 

When std::shared_ptr or std::unique_ptr that take ownership of the
raw pointer passed as an argument are used, then a memory leak can occur
if std::make_unique throws an exception.

This rule detects calls to the constructors of std::shared_ptr
and std::unique_ptr that take the raw pointer as the first argument.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent memory leaks.
EXAMPLE 

#include <memory>
struct A
{
    int8_t i;
};

void f1() {
    A *p1 = new A;
    auto v1 = std::shared_ptr<A>(p1);    // Violation
    auto v2 = std::unique_ptr<A>(new A); // Violation
}
REPAIR 

#include <memory>
struct A
{
    int8_t i;
};

void f1() {
    auto v1 = std::make_shared<A>();  // OK
    auto v2 = std::make_unique<A>();  // OK
}

void f2() {
    auto p = std::make_shared<A>();   // OK
    int8_t* pi = &(p->i);
    std::shared_ptr<int8_t> q(p, pi); // OK - not taking ownership
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
