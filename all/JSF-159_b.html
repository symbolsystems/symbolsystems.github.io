Operator unary & shall not be overloaded [JSF-159_b-2]

DESCRIPTION 

"Taking the address of an object of incomplete type where the complete type
contains a user declared operator & leads to undefined behaviour"
SINCE 

v7.2
BENEFITS 

Rule prevents undefined behaviour.
EXAMPLE 

class A
{
public:
    A * operator & ( );   // Violation
};
REPAIR 

Do not overload the unary '&' operator
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M5-3-3. The unary & operator shall not be overloaded

2. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.21 Operators, AV Rule 159

3. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 5, Rule 5-3-3

4. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 16.5.2 The address-of operator shall not be overloaded
