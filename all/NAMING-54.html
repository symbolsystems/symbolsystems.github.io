Use visually distinct identifiers [NAMING-54-3]

DESCRIPTION 

"Use visually distinct identifiers with meaningful names to eliminate errors
resulting from misreading the spelling of an identifier during the development
and review of code. . . Depending on the fonts used, certain characters appear
visually similar or even identical." [CERT-DCL02-C]

The rule detects when two identifiers vary only with respect to one or more
visually similar characters.

The rule assumes that the following characters are visually similar:
- the letter 'O' and the number '0'
- the letter 'O' and the letter 'D'
- the letter 'O' and the letter 'Q'
- the letter 'D' and the number '0'
- the letter 'D' and the letter 'Q'
- the letter 'Q' and the number '0'
- the letter 'I' and the number '1'
- the letter 'I' and the letter 'l'
- the letter 'l' and the number '1'
- the letter 'Z' and the number '2'
- the letter 'S' and the number '5'
- the letter 'B' and the number '8'
- the letter 'n' and the letter 'h'
- the letter cluster 'rn' ('r' followed by 'n') and the letter 'm'
SINCE 

2020.2
NOTES 

This rule compares identifiers in the same scope. Identifiers that are declared
in different structures or functions are not compared.
BENEFITS 

This rule helps you improve code readability and avoid errors that may result
from confusing identifiers.
EXAMPLE 

int g_O;
int g_0;                        // Violation

void foo()
{
    char locVar_ZZZ111;
    char locVar_222III;         // Violation
}
REPAIR 

To fix the code, use visually distinct identifiers

int g_1;
int g_2;                        // OK

void foo()
{
    char locVar_ZZZ111_1;
    char locVar_222III_2;       // OK
}
