Function-like macros shall not be defined [MISRACPP2023-19_0_2-a-2]

DESCRIPTION 

In most cases it is possible to replace a function-like macro by a function.
Functions have more advantages over function-like macros, for example their
parameters and return types are type-checked.

This rule detects definitions of function-like macros with exceptions
described in the section below.
SINCE 

2023.2
EXCEPTIONS 

In some cases it is not possible to replace a function-like macro
by a function, so this rule allows function-like macros containing
the __LINE__, __FILE__, and __func__ identifiers or #, and ## operators.
BENEFITS 

This rule helps you avoid errors and improve maintainability
of the code.
EXAMPLE 

#define SUM(A,B) ((A)+(B))     // Violation

void foo( int x, int y ) {
    /* ... */
    SUM( x, y );
    /* ... */
}
REPAIR 

int sum( int a, int b ) {      // OK
    return (a + b);
}

void foo( int x, int y ) {
    /* ... */
    sum( x, y );
    /* ... */
}

#define NAME( name ) int name = #name        // OK - Exception

#define LOGGING( msg ) log( __func__, msg ); // OK - Exception
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 19.0.2 Function-like macros shall not be defined
