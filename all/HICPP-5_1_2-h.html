The increment (++) and decrement (--) operators should not be mixed with other operators in an expression [HICPP-5_1_2-h-3]

DESCRIPTION 

"It is the intention of the rule that when the increment or decrement operator
is used, it should be the only side-effect in the statement." [MISRA-C:2004]

"The use of increment and decrement operators in combination with other
arithmetic operators is not recommended, because:
 - It can significantly impair the readability of the code.
 - It introduces additional side effects into a statement, with the potential
   for undefined behaviour." [MISRA C++:2008]

This rule detects when the result of an increment or decrement operation is
assigned to another object or used as one of the following:
- the operand of an arithmetic operator
- an initializer
- an array index
- an argument of a function call
- the value of a return statement
BENEFITS 

This rule helps you improve code readability and prevent undefined behavior.
EXAMPLE 

int g1 = 0;

void bar(int arg);
int foo1() {
    int x, y;
    int tab[10];
    int index = 0;
    int var = g1++;            // Violation

    tab[index++];              // Violation
    x = --y + x++;             // Violation
    bar(x++);                  // Violation
    return ++y;                // Violation
}
REPAIR 

You can fix the code by moving the increment and decrement operations to
separate statements.

int g1 = 0;

void bar(int arg);
int foo() {
    int x, y;
    int tab[10];
    int index = 0;
    int var = g1;

    g1++;
    tab[index];
    index++;                   // OK
    --y;                       // OK
    x = y + x;
    x++;                       // OK
    bar(x);
    x++;                       // OK
    ++y;                       // OK
    return y;
}
