Each variable declaration should be commented [COMMENT-05-3]

DESCRIPTION 

This rule requires that every declaration of a local or a global variable
should have a comment. The rule accepts comments that are placed in the
line of the declaration or in the line directly before the declaration
(only if this line does not contain any other code).
SINCE 

v7.1
NOTES 

1. The rule does not check the content of the comment.
2. If a declaration is used inside a macro definition, then this rule
   requires comments on every expansion of this macro.
3. If a variable has a declaration and a definition, then the comment
   is required for both.
BENEFITS 

The rule improves the readability of code.
EXAMPLE 

// Violations are reported on every declaration of the local or global
// variable in this section.

extern int x1;
int x1 = 10;
// comment
void foo1() {  // comment
    int v1;

    // comment

    int v2;
    int * ptr1, ptr2;
}

#define defVar1(var) { int j = var; }

void foo1(int v, int k) {
    defVar1(v)
    defVar1(k)
}
REPAIR 

// All declarations in this section are commented.

// comment
extern int x1;
int x1 = 10;   // comment
// comment
void foo1() {  // comment
    int v1;    // comment

    /*
     * comment
     */
    int v2;
    /* comment */
    int * ptr1, ptr2;
}

#define defVar1(var) { int j = var; }

void foo1(int v, int k) {
    // comment
    defVar1(v)
    defVar1(k)  // comment
}
