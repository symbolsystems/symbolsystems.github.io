There shall be at most one occurrence of the # or ## preprocessor operators in a single macro definition [AUTOSAR-M16_3_1-a-2]

DESCRIPTION 

"There is an issue of unspecified order of evaluation associated 
with the # and ## preprocessor operators.
To avoid this problem only one occurrence of either operator 
shall be used in any single macro definition 
(i.e. one #, or one ## or neither)."
BENEFITS 

Rule improves readability and clarity of code.
EXAMPLE 

#define TEST1(A,B,C) A # B # C      /* Violation */
#define TEST2(A,B,C) A ## B # C     /* Violation */
#define TEST3(A,B,C) A ## B ## C    /* Violation */
REPAIR 

#define TESTa(A,B) A # B            /* OK */
#define TESTb(A,B) A ## B           /* OK */
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M16-3-1. There shall be at most one occurrence of the # or ## operators in
   a single macro definition

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 16, Rule 16-3-1

3. MISRA-C:2004 Guidelines for the use of the C language in critical systems
   Chapter 6, Section 19

4. Origin: Misra Guidelines - Rule 98
