The parameters in all overrides of a function should either be unnamed or have identical names [NAMING-57-3]

DESCRIPTION 

"The name given to a parameter helps document the purpose of the parameter.
If a function parameter is renamed in a subsequent declaration, then having
different names for the same object may lead to developer confusion."
[MISRA C++:2023]

This rule detects when a member function in a derived class and an overridden
member function in a base class have different corresponding parameters
names.
SINCE 

2023.2
NOTES 

This rules does not apply to template functions.
This rule only checks declarations in a single translation unit.
BENEFITS 

This rule helps you improve readability of the code.
EXAMPLE 

class Base
{
public:
  virtual void f1(int a);
};

class Derived : public Base
{
public:
  void f1(int b) override;   // Violation - using 'b' instead of 'a'
};
REPAIR 

class Base
{
public:
  virtual void f1(int a);
  virtual void f2(int a);
};

class Derived : public Base
{
public:
  void f1(int a) override;   // OK
  void f2(int) override;     // OK
};
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 13.3.3 The parameters in all declarations or overrides of a function shall either be unnamed or have identical names
