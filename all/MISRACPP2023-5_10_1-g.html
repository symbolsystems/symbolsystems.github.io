Do not declare identifiers with names reserved in the standard library [MISRACPP2023-5_10_1-g-2]

DESCRIPTION 

"For each type T from the C standard library, the types ::T and std::T
are reserved to the implementation and, when defined, ::T shall be identical
to std::T. These types are clock_t, div_t, FILE, fpos_t, lconv, ldiv_t,
mbstate_t, ptrdiff_t, sig_atomic_t, size_t, time_t, tm, va_list, wctrans_t,
wctype_t, and wint_t." [ISO/IEC 14882:2017]

This rule detects when identifiers with the following names are declared:
defined, final, override, clock_t, div_t, FILE, fpos_t, lconv, ldiv_t,
mbstate_t, ptrdiff_t, sig_atomic_t, size_t, time_t, tm, va_list, wctrans_t,
wctype_t and wint_t.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid using reserved identifiers.
EXAMPLE 

int final;        // Violation
int override;     // Violation
int clock_t;      // Violation
REPAIR 

Do not define identifiers with names that are reserved in the standard
library.
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   20.5.4.3.4 Types

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
