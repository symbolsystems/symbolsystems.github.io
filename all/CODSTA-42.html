The macro TRUE should be defined as 1 [CODSTA-42-5]

DESCRIPTION 

The rule reports a violation if the macro 'TRUE' is not defined as 1.
It is allowed to use decimal, octal or hexadecimal values (e.g. 0001, 0x0001)
and the number 1 can be followed by a suffix (e.g. 1u, 1l, 1UL).

See also: CODSTA-24, CODSTA-25, CODSTA-43
BENEFITS 

The rule improves readability and maintainability of code.
EXAMPLE 

#define TRUE !0   // Violation
REPAIR 

#define TRUE 1    // OK
