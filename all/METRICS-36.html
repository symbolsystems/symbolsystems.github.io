A global function should not be called from more than 5 different functions [METRICS-36-3]

DESCRIPTION 

Calling a function in more than five other functions might result in too
complicated implementation logic in a single translation unit.

This rule detects when a global function is called in more than five other
functions defined in the same translation unit.
SINCE 

v7.3
NOTES 

This rule does not report template functions.
BENEFITS 

This rule helps you simplify code structure.
EXAMPLE 

// file.h
void func();    /* Violation */

// file.c
void f1() {     /* 1 */
    func();
}
void f2() {     /* 2 */
    func();
}
void f3() {     /* 3 */
    func();
}
void f4(){      /* 4 */
    func();
}
void f5(){      /* 5 */
    func();
}
void f6(){      /* 6 */
    func();
}
REPAIR 

You can improve your code by splitting the implementation into multiple files.

// file.h
void func();    /* OK */

// file1.c
void f1() {     /* 1 */
    func();
}
void f2() {     /* 2 */
    func();
}
void f3() {     /* 3 */
    func();
}

// file2.c
void f4(){      /* 1 */
    func();
}
void f5(){      /* 2 */
    func();
}
void f6(){      /* 3 */
    func();
}

/* Example of code not detected by this rule */
void func2();    /* OK */

void f0() {      /* 1 */
    func2();
    func2();
    func2();
    func2();
    func2();
    func2();
}
