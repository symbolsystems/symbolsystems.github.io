Don't use hex constant '0xffffffff' as an error marker [PORT-04-2]

DESCRIPTION 

"Sometimes the value '-1' is used as an error code or other special marker
and it is written as '0xffffffff'. This expression is incorrect on a 64-bit
platform, so you should explicitly define the value '-1'."
BENEFITS 

The rule prevents writing an error prone code.
EXAMPLE 

#ifndef size_t
typedef unsigned int size_t;
#endif

#define INVALID_RESULT (0xFFFFFFFFu)          // Violation
#define NULL 0

size_t MyStrLen(const char *str) {
   size_t n;
   if (str == NULL)
     return INVALID_RESULT;
   // ... 
   return n;
}
REPAIR 

#ifndef size_t
typedef unsigned int size_t;
#endif

#define INVALID_RESULT (size_t(-1))          // OK
#define NULL 0

size_t MyStrLen(const char *str) {
   size_t n;
   if (str == NULL)
     return INVALID_RESULT;
   // ...
   return n;
}
