The enumeration constant named 'TRUE' should be explicitly initialized by 1 value [CODSTA-43-5]

DESCRIPTION 

The enumeration types are sometimes used to define boolean values.
If the enumeration type contains an enumeration constant named 'TRUE', then 
this constant should be explicitly initialized by 1 value. 
It is allowed to use decimal, octal or hexadecimal values (e.g. 0001, 0x0001)
and the number 1 can be followed by a suffix (e.g. 1u, 1l, 1UL).

See also: CODSTA-24, CODSTA-25, CODSTA-42
BENEFITS 

The rule improves readability of code and prevents confusions.
EXAMPLE 

void foo1()
{
    enum BOOL
    {
        FALSE, 
        TRUE            // Violation
    };
}

void foo2()
{
    enum BOOL
    {
        FALSE = 0u,
        TRUE = !FALSE   // Violation
    };
}
REPAIR 

void foo1()
{
    enum BOOL
    {
        FALSE = 0,
        TRUE = 1         // OK
    };
}

void foo2()
{
   enum BOOL
    {
        FALSE = 0u,
        TRUE = 1u        // OK
    };
}
