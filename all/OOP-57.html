Structs should only contain public data members and should not be a base or inherit [OOP-57-3]

DESCRIPTION 

"A type defined as struct shall: (1) provide only public data members,
(2) not provide any special member functions or methods, (3) not be a
base of another struct or class, (4) not inherit from another struct or
class." [AUTOSAR A12-4-2]

This rule checks structs for functions, non-public members, and inherits
from another class or struct. It also checks for uses of a struct as a
base for another class or struct.
SINCE 

v10.4
BENEFITS 

Rule improves code readability, helps differentiate a struct from a class,
and helps you achieve compliance with some coding standards.
EXAMPLE 

#include <cstdint>

struct D              // Violation - struct has private/protected members
{
public:
  std::int32_t x;

protected:
  std::int32_t y;

private:
  std::int32_t z;
};

struct F : public D   // Violation - struct inherits from another struct
{
    int a;
};
REPAIR 

#include <cstdint>

class D              // OK
{
public:
  std::int32_t x;

protected:
  std::int32_t y;

private:
  std::int32_t z;
};

struct F              // OK
{
    int a;
};

1. AUTOSAR C++14 Coding Guidelines

   members, (2) not provide any special member functions or methods, (3) not
   be a base of another struct or class, (4) not inherit from another struct
   or class
