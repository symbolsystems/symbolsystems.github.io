The sizeof operator will not be used on expressions that contain side effects [JSF-166-3]

DESCRIPTION 

"The operand of a sizeof operator may be either an expression or may specify
a type. If the operand contains an expression, a possible programming error
is to expect that expression to be evaluated when it is actually not evaluated
in most circumstances." [MISRA C:2012 Rule 13.6]

The rule reports a violation if as the sizeof operand is used assignment,
increment, decrement, new, delete or throw operator.

See also: PB-02, PB-22
BENEFITS 

"Any expressions appearing in the operand of a sizeof operator are not normally
evaluated. This rule mandates that the evaluation of any such expression shall
not contain side effects, whether or not it is actually evaluated."
[MISRA C:2012 Rule 13.6]
EXAMPLE 

void foo(int p){
    int k;
    
    sizeof(k = p);  // Violation  
    sizeof(p++);    // Violation
    sizeof(p--);    // Violation
}
REPAIR 

void foo(int p){
    int k;
    
    k = p;
    sizeof(k);      // OK
    p++;
    sizeof(p);      // OK
    p--;
    sizeof(p);      // OK
}
