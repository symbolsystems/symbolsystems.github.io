The built-in unary + operator should not be used [CODSTA-236-3]

DESCRIPTION 

"The operand of the unary + operator shall have arithmetic, unscoped
enumeration, or pointer type and the result is the value of the argument.
Integral promotion is performed on integral or enumeration operands.
The type of the result is the type of the promoted operand."
[ISO/IEC 14882:2011]

This rule detects the built-in unary + operator.
SINCE 

2023.2
BENEFITS 

This rule helps you improve readability and prevents bug-prone code.
EXAMPLE 

void foo()
{
  short i = 0;
  auto sum = +i;                                // Violation
  for (i = 0; i < 10; i++)
  {
    sum =+ i;                                   // Violation
  }
}
REPAIR 

void foo()
{
  short i = 0;
  auto sum = i;                                 // OK
  for (i = 0; i < 10; i++)
  {
    sum += i;                                   // OK
  }
}
