All 'if...else-if' constructs shall be terminated with an 'else' clause [CODSTA-241-3]

DESCRIPTION 

This rule detects when an 'if' statement followed by one or more 'else-if'
statements is not terminated with a final 'else' clause.
There are no requirements for the contents of the final 'else' clause; it may
contain only an empty statement or an empty compound block.

See also: CODSTA-23, MISRA2004-14_10
SINCE 

2023.2
BENEFITS 

Providing, even an empty, explicit 'else' clause constitutes defensive
programming, showing that all cases have been considered.
EXAMPLE 

void foo(int a)
{
    if(a > 0)
    {
    }
    else if (a > 10)   // Violation
    {
    }
}
REPAIR 

void foo(int a)
{
    if(a > 0)
    {
    }
    else if (a > 10)   // OK
    {
    }
    else
    {
    }
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
