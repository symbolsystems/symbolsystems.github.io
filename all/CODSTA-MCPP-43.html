The 'set_unexpected' identifier should not be used [CODSTA-MCPP-43-3]

DESCRIPTION 

"All code shall conform to ISO/IEC 14882:2014 - Programming Language C++ and
shall not use deprecated features." [AUTOSAR-A1-1-1]

The 'set_unexpected' function from the Standard Library was deprecated in
the C++11 and removed in the C++17 version of the C++ Language Standard.

This rule detects when the 'set_unexpected' identifier is used in code.
SINCE 

v10.4
NOTES 

This rule reports all occurrences of the 'set_unexpected' identifier. For this
reason, the rule may report 'set_unexpected' that does not represent a function
form the Standard Library. For example:

int set_unexpected; // Violation
BENEFITS 

This rule helps you ensure that your code complies with the up-to-date version
of the C++ Language Standard. This allows you to increase longevity of
an application and address the defects and vulnerabilities that may emerge
as deployment environments evolve.
EXAMPLE 

void my_unexpected() {
    throw 1;
}

void bar() throw(int) {
    char c;
    trhow c;
}

void foo() {
    std::set_unexpected(my_unexpected);           // Violation
    try {
        bar();
    } catch(int e) {
    }
}
REPAIR 

Do not use the 'set_unexpected' identifier.
