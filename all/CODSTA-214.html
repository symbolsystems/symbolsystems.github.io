The '_Atomic' type specifier and the '_Atomic' type qualifier should not be used [CODSTA-214-3]

DESCRIPTION 

"Updates to the C Standard have introduced new language features. The following
shall not be used:
- The _Atomic type specifier, the _Atomic type qualifier and the facilities 
  that are specified as being provided by <stdatomic.h>;" 
[MISRA C:2012 Amendment 2]

This rule detects declarations and uses of atomic types.
SINCE 

2020.1
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

#include <stdatomic.h>

_Atomic const int * p1;                         // Violation

const atomic_int * p2;                          // Violation

_Atomic(int) * p3;                              // Violation
REPAIR 

To fix the code, use neither the '_Atomic' type specifier nor
the '_Atomic' type qualifier.
