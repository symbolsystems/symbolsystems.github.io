Objects with external linkage should not be initialized in a header file [INIT-01-3]

DESCRIPTION 

If a non static object is initialized in a header file and multiple source
files including this header, then multiple definition error can be thrown
by the linker. Instead of initialization in a header file declare objects
as extern and initialize them in implementation files.

See also: MISRA2004-8_5, PFO-01
NOTES 

The rule is enabled only for C language.
A header file is defined as any file that is included via #include directive
BENEFITS 

The rule improves readability and maintainability of code, and prevents linker
errors.
EXAMPLE 

/* header file - file.h */ 
int i = 0;	             /* Violation */
REPAIR 

/* header file - file.h */ 
extern int i;	             /* OK */

/* implementation file - file.c */ 
#include "file.h"
int i = 0;	             /* OK */
