Avoid expressions in return statements of void functions [MISRA-084-3]

DESCRIPTION 

"If any return statements are used in a function which returns a void type,
expressions shall not be placed after the return" [MISRA-C:1998]

This rule detects when a function that returns the 'void' type contains
the 'return' statement not directly followed by the ';' semicolon.
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

void f1(void)
{
    return 0;     // Violation
}
REPAIR 

void f1(void)
{
    return;       // OK
}
REFERENCES 

1. MISRA-C:1998 Guidelines for the use of the C language in Vehicle Based Software
   Chapter 7, Rule 84.
