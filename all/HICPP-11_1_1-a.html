Avoid "public" data members [HICPP-11_1_1-a-2]

DESCRIPTION 

This rule detects "public" data members that could be accessed 
by untrusted code.
Only "class" types are checked ("struct" and "union" are not checked).
See also: OOP-19
NOTES 

Rule skips 'static const' variables.
BENEFITS 

Prevents the use of "public" data members. Public data members can be directly 
accessed by any user code. 
Using public accessor member functions to return data instead 
will prevent unauthorized access.
EXAMPLE 

class A {
public:
    int iData;      // Violation
};
REPAIR 

class A {
private:
    int iData;      // OK
public:
    const int accessiData( ); 
};
