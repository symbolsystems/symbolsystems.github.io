NULL shall not be used as an integer value [CODSTA-CPP-62-3]

DESCRIPTION 

"In C++, the literal 0 is both an integer type and the null-pointer-constant.
To meet developer expectations, NULL should be used as the
null-pointer-constant, and 0 for the integer zero."
SINCE 

v7.2
BENEFITS 

"As a result of this rule, NULL is considered to have pointer type."
EXAMPLE 

#include <cstddef>

void f1( int );

void foo( int& ref )
{
    f1( NULL );  // Violation
    ref = NULL;  // Violation
}
REPAIR 

#include <cstddef>

void f1( int );

void foo( int& ref )
{
    f1( 0 ); // OK
    ref = 0; // OK
}

// Examples of code not reported by rule

void foo1 (int*& rptr) {
    int* ptr;
    int a1[5];
    
    ptr = NULL;        // OK
    rptr = NULL;       // OK
    
    if(a1 == NULL) {}  // OK
}
