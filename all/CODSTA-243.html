The 'switch' statement should not contain labels other than 'case' and 'default' [CODSTA-243-3]

DESCRIPTION 

A 'switch' statement contains a label group that is directly enclosed
by a 'switch' body. If labels other than 'case' or 'default' are used,
it can be misleading and potentially introduce unstructured control
flow.

This rule detects when a label other than 'case' or 'default' is used
in a 'switch' label group.
SINCE 

2023.2
BENEFITS 

This rule helps you prevent confusion.
EXAMPLE 

void f(int i)
{
    switch (i)
    {
    LABEL1:               // Violation
        break;
    case 0:
        LABEL2:           // Violation
        break;
    default:
    LABEL3:               // Violation
        break;
    }
}
REPAIR 

void f(int i)
{
    switch (i)
    {
    case 0:
        {
            LABEL1:      // OK
            break;
        }
    case 1:
        if(i > 0)
        {
            LABEL2:      // OK
            i = i + 2;
        }
        break;
    default:
        break;
    }
}
