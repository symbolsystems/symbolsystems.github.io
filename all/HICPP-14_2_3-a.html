Declare 'extern' an explicitly instantiated template [HICPP-14_2_3-a-3]

DESCRIPTION 

"Declaring the template with extern will disable implicit instantiation
of the template when it is used in other translation units."
[HIC++ Coding Standard]

This rule detects when a non-extern explicit instantiation of a template type,
function, or variable does not have a corresponding 'extern' explicit template
instantiation in the current translation unit.
SINCE 

v10.4
BENEFITS 

This rule helps you save time, and reduce compile-time dependencies.
EXAMPLE 

// file.h
template <typename T> class C {
    T m1;
};
template <typename T> void func1(){
    // ...
}

// file.cpp
#include "file.h"

template class C<int>;               // Violation
template void func1<int>();          // Violation
REPAIR 

template <typename T> class C {
    T m1;
};
template <typename T> void func1(){
    // ...
}

extern template class C<int>;        // OK
extern template void func1<int>();   // OK

// file.cpp
#include "file.h"

template class C<int>;               // OK
template void func1<int>();          // OK
REFERENCES 

1. High Integrity C++ Coding Standard
