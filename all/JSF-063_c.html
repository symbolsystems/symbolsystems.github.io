There will be no space between a unary operator '!' or '~' and its operand [JSF-063_c-3]

DESCRIPTION 

The rule reports a violation if there is space, tab, comment or new line
character between a unary complement operator "~" or unary logical
negation operator "!" and its operand. 

See also: FORMAT-17, FORMAT-18, FORMAT-22, FORMAT-45, FORMAT-46
NOTES 

The rule reports violations on operators used inside preprocessor
directives.

The rule does not report a violation on "operator" construct (declaration and
explicit call), for example:
bool operator! (const A&); // no violation reported
bool b = operator! (a); // no violation reported
BENEFITS 

The rule promotes continuity in the relationship between the operators
and the operands on which they act.
DRAWBACKS 

The rule reports a violation when "~" character is used in class
destructor declaration, for example:
class A { ~ A(); } // Violation
EXAMPLE 

#if ! defined(XXX)       // Violation
#define XXX ! a          // Violation
bool foo(int p) {
    int x = ~ (p + 1) +  // Violation
            ~/* Violation */p;
    // Violation in next line:
    return !
        x;
}
struct A {
    ~ A();                  // Violation
};
bool operator! (const A&);  // OK - no violation on "operator" construct
void bar(A& a) {
    bool b = operator! (a); // OK - no violation on "operator" construct
    b = ! a;                // Violation
}
#endif
REPAIR 

#if !defined(XXX)           // OK
#define XXX !a              // OK
bool foo(int p) {
    int x = ~(p + 1) +      // OK
            ~p;             // OK
    return !x;              // OK
}
struct A {
    ~A();                   // OK
};
bool operator!(const A&);   // OK
void bar(A& a) {
    bool b = operator! (a);
    b = !a;                 // OK
}
#endif
