A function should not contain unused label declarations [MISRA2012-RULE-2_6-4]

DESCRIPTION 

"If a label is declared but not used, then it is unclear to a reviewer
if the label is redundant or it has been left unused by mistake."
SINCE 

v9.5
BENEFITS 

The rule improves legibility of code.
EXAMPLE 

int foo1(){
    int i;
    label1:             /* Violation */
        i++;
    label2:             /* Violation */
        return 10;
}
REPAIR 

int foo1(){
    int i;
    label1:             /* OK - function contains goto label1 */
        i++;
    if(i < 10)
        goto label1;
                        /* label2 was redundant, so it was removed */
    return 10;
}
REFERENCES 

1. MISRA C:2012 Guidelines for the use of the C language in critical systems
   Section 8: Rules, Rule 2.6
