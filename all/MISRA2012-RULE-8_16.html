The alignment specification of zero should not appear in an object declaration [MISRA2012-RULE-8_16-4]

DESCRIPTION 

"The alignment requirement of an entity is the strictest nonzero
alignment specified by its alignment-specifiers, if any; otherwise,
the alignment-specifiers have no effect." [ISO/IEC 14882:2017]

"If the alignment of an object is important, it should be specified
explicitly." [MISRA C:2012 Amendment 3]

The rule detects zero-alignment specifiers in object declarations.
SINCE 

2022.2
NOTES 

"This rule applies to any integer constant expression operand
to _Alignas that evaluates to zero." [MISRA C:2012 Amendment 3]
BENEFITS 

This rule helps you improve code readability and maintainability.
EXAMPLE 

#include <stdalign.h>

#define SIZE 0

alignas (SIZE) int g1;                           // Violation
alignas (0) int g2;                              // Violation
REPAIR 

You can fix the code by removing the zero-alignment specifier.

#include <stdalign.h>

int g1;                                          // OK
alignas (8) int g2;                              // OK
