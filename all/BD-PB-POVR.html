Avoid overwriting method parameters [BD-PB-POVR-1]

DESCRIPTION 
This rule detects when a method parameter value is overwritten before it 
is used on paths for which the parameter already exists.
SINCE 
v10.1
DEPRECATED 
This rule is deprecated and has been replaced by BD-PB-VOVR.
NOTES 
Only parameters that are inside the actual method are analyzed. The rule 
assumes that parameters used to invoke other methods are used correctly.
SECURITY RELEVANCE 
N/A
PARAMETERS 
N/A
BENEFITS 
Parameters that are always overwritten are indicators of poor quality of 
code or bug in the program.
DRAWBACKS 
N/A
EXAMPLE 
Here is sample code that triggers a violation:

class StackElement
{
public:
    StackElement* previous;
    int value;
    
    StackElement(StackElement* previousElement, int value)
    {
        previous = previousElement;
        value = previousElement->value; // VIOLATION
    }
};
REPAIR 
In the example above improper value is initialized by improper field.

class StackElementFix
{
public:
    StackElementFix* previous;
    int value;

    StackElementFix(StackElementFix* previousElement, int value)
    {
        this->previous = previousElement;
        this->value = value; // FIXED
    }
};
