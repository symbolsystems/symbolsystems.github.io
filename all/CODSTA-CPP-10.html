Do not define structs that contain member functions [CODSTA-CPP-10-3]

DESCRIPTION 

Member functions should be contained in classes, not structs, because classes 
support both multiple instances and encapsulation. Structs are also often 
entirely public: whereas the default access level for members and base classes 
of a class is private. This rule detects if you define structs that contain 
member functions.
BENEFITS 

Readability. It is generally expected that types with member functions 
are classes, and many programmers still think of structs in terms
of their original meaning in C.
EXAMPLE 

struct A
{
    public:
        int foo( );     // Violation
};
REPAIR 

class A
{
    public:
        int foo( );     // OK
};
