Function types shall be in prototype form [MISRAC2012-RULE_8_2-c-2]

DESCRIPTION 

This rule checks if a function types are in prototype form.
It reports a violations if a declaration of a function type:
- is in K&R style, or
- has an empty parameter list

See also: MISRA2004-16_3, CODSTA-158
NOTES 

"The early version of C, commonly referred to as K&R C, did not provide
a mechanism for checking the number of arguments or their types against
the corresponding parameters. The type of an object or function did not
have to be declared in K&R C since the default type of an object and the
default return type of a function was int." [MISRA C:2012]

"An empty parameter list is not valid in a prototype. If a function type
has no parameters its prototype form uses the keyword void." [MISRA C:2012]
BENEFITS 

"The mismatch between the number of arguments and parameters, their types
and the expected and actual return type of a function provides potential 
for undefined behaviour." [MISRA C:2012]
EXAMPLE 

int func1();   /* Violation - not in prototype form */

int func2(n)   /* Violation - K&R style identifier and declaration list */
int n;
{
    return n;
}

void (*fp1)(); /* Violation - not in prototype form */
REPAIR 

int func1(void);    /* OK - prototype specifies 0 parameters */

int func2(int n)    /* OK */
{
    return n;
}

void (*fp1)(void);  /* OK */
