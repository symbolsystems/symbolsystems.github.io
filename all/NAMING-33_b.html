An identifier shall not contain a double underscore '__' [NAMING-33_b-3]

DESCRIPTION 

"Each identifier that contains a double underscore '__' (...) is reserved
to the implementation for any use." [ISO/IEC 14882:2017]

Using a double underscore might result in the creation of a reserved identifier.

This rule detects when a double underscore '__' is used inside a declaration
of an identifier.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid using reserved identifiers.
EXAMPLE 

int var__1;    // Violation
REPAIR 

int var_1;     // OK
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
