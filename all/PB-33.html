Avoid implicit type conversions in assignments involving enum types [PB-33-3]

DESCRIPTION 

Implicit conversions between:
* enum and not enum types
* enum and different enum types
are not allowed.
SINCE 

v7.2
BENEFITS 

Prevents depending on the value of enum constant which can change
EXAMPLE 

enum ABC0{A0, B0, C0};

void foo1(int arg1){}

int foo(int p)
{
    ABC0 x;
    int i = x; // Violation
    foo1(x);   // Violation
    return x;  // Violation
}
REPAIR 

enum ABC0{A0, B0, C0};

void foo1(int arg1){}

int foo(int p)
{
    ABC0 x;
    int i = (int)x; // OK
    foo1((int)x);   // OK
    return (int)x;  // OK
}
REFERENCES 

1. ISO/DIS 26262
   point 8.4.4
