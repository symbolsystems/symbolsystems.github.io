Braces ("{}") which enclose a block will have nothing else on the line except comments [JSF-060_a-3]

DESCRIPTION 

Rule reports a violation if a line with opening or closing brace
contains another code.

The rule checks opening and closing braces only for blocks. Not all 
braces are "block" braces. The block is a compound statement and can
be used, for example:
 - by itself
 - to enclose a body of statements:
   'if', 'else', 'while', 'do-while', 'for', 'switch', 'try', 'catch'
 - to enclose a function's body
Examples of braces which are not "block" braces:
 - class definition
 - variable initializer

See also: FORMAT-02, FORMAT-03, FORMAT-34, FORMAT-43, FORMAT-44
SINCE 

v9.4
BENEFITS 

Rule improves readability and maintainability.
EXAMPLE 

void bar() {}                   // Violation (opening and closing brace)
int foo(int i) {                // Violation
    if (i > 0) {                // Violation
        { int j; }              // Violation (opening and closing brace)
        return i;
    } else if (i < 0) {         // Violation (opening and closing brace)
        i /= 2;
        return -i;
    } else { return 0; }        // Violation (opening and closing brace)
}
REPAIR 

void bar()
{
}
int foo(int i)
{
    if (i > 0)
    {
        {
            int j;
        }
        return i;
    }
    else if (i < 0)
    {
        i /= 2;
        return -i;
    }
    else
    {
        return 0;
    }
}
// Braces that are not "block" braces are not checked
enum E {EN1 = 2, EN2 = 1};
class FooBar{};
struct S{ int i; float f; } ss;
int array[] = {1,2,3,4,5};
