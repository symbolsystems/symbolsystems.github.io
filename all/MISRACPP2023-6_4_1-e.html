Identifiers declared in an inner class scope should not hide identifiers declared in outer class scope [MISRACPP2023-6_4_1-e-2]

DESCRIPTION 

"If an identifier is declared in an inner scope and it uses the same name
as an identifier that already exists in an outer scope, then the innermost
declaration will "hide" the outer one. This may lead to developer confusion."

The rule reports a violation if an identifier declared in an inner class scope
hides an identifier declared in an outer class scope or in public/protected 
section outer class's parents.

See also: CODSTA-CPP-83, CODSTA-CPP-84, MISRA2004-5_2_a, MISRA2004-5_2_b
SINCE 

v9.4
BENEFITS 

Hiding names of identifiers can lead to errors and confusions.
EXAMPLE 

class C {
public:
    C();
    typedef enum E {EN} ET;
	
    class D {
    public:
        struct ET{       // Violation - typedef to enum is hidden
            int m;
        };
    private:
        int x;           // Violation - static member is hidden
    };
	
private:
    static int x;
};
REPAIR 

class C {
public:
    C();
    typedef enum E {EN} ET;
	
    class D {
    public:
        struct ST{       // OK - the name of struct was changed
            int m;
        };
    private:
        int x;           // OK - non static members are not hidden
    };
	
private:
    int x;
};
