The #include pre-processor directive will only be used to include header (*.h) files [PREPROC-08-3]

DESCRIPTION 

"The #include pre-processor directive will only be used to include
header (*.h) files."

The rule reports a violation if a file with extension other than '.h' 
is included via #include directive.
SINCE 

v7.1
NOTES 

JSF standard contains the following exception:

"In the case of template class or function definitions, the code may be
partitioned into separate header and implementation files. In this
case, the implementation file may be included as a part of the header
file. Rule allows for mentioned exception if the implementation file
has the same name as a header file."

The rule does not take this exception into account, so the user should himself
suppress such violations.
BENEFITS 

The rule improves the clarity of code. The only files included in a .cpp
file should be the relevant header (*.h) files.
EXAMPLE 

// file.cpp
#include <file2.cpp>  // Violation
#include <file3.hh>   // Violation

// file.h
#include <file2.cpp>  // Violation
#include <file3.hh>   // Violation
REPAIR 

// file.cpp
#include <file2.h>    // OK - header (*.h) file
#include <file3.h>    // OK - header (*.h) file

// file.h
#include <file2.h>    // OK - header (*.h) file
#include <file3.h>    // OK - header (*.h) file
