The names of identifiers should not be longer than 64 characters [PORT-25-3]

DESCRIPTION 

"User-specified identifiers (internal and external) will not rely
on significance of more than 64 characters." [JSF]

This rule detects when an identifier has more than 64 characters in its name.

The rule checks the following identifiers:
- variables, functions, function parameters
- classes, structures, unions, typedefs
- enumerations and enumeration constants
- namespaces
- labels
- template parameters and template parameter constants
SINCE 

v7.1
NOTES 

This rule does not check the names of conversion operators.
BENEFITS 

This rule helps you improve code portability and readability.
EXAMPLE 

void f1_45678901234567890123456789012345678901234567890123456789012345();        // Violation
void f1_45678901234567890123456789012345678901234567890123456789012345(){}       // Violation

void f1(int p1_45678901234567890123456789012345678901234567890123456789012345) { // Violation
    int v1_45678901234567890123456789012345678901234567890123456789012345;       // Violation
}
REPAIR 

You can fix the code by modifying the names of identifiers.

void f1_4567890123456789012345678901234567890123456789012345678901234();        // OK
void f1_4567890123456789012345678901234567890123456789012345678901234(){}       // OK

void f1(int p1_4567890123456789012345678901234567890123456789012345678901234) { // OK
    int v1_4567890123456789012345678901234567890123456789012345678901234;       // OK
}
