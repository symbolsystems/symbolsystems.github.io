Do not use break in for loops [CODSTA-08-2]

DESCRIPTION 

The rule reports a violation if a 'break' statement causes exit
from the 'for' loop. If the 'for' loop is interrupted, then the rest
of the code in the block, and the remaining iterations are skipped.
This behaviour may not be consistent with developer expectations.
BENEFITS 

Avoiding breaks in the for loop makes your code easier to follow.
EXAMPLE 

void func() 
{
    for (int j = 0; j < 10; j++) {        
        if (j==5) 
        {
            break;                     // Violation
        }
    }
  
    for (;;) {        
        //...
        break;                         // Violation    
    }
}
REPAIR 

/* Examples of correct code */

void func() 
{
    for (int j = 0; j < 5; j++) {
         // OK - 'for' loop does not contain 'break'
    }
    
    while (true) 
    {
        //...
        break; // OK - 'break' interrupts 'while' loop
    } 
  
    for (int j = 0; j < 5; j++) {
        while (true) 
        {
            //...
            break; // OK - 'break' interrupts 'while' loop
        } 
    }
}
