The '_Alignas' alignment specifier and the '_Alignof' operator should not be used [CODSTA-213-3]

DESCRIPTION 

"Updates to the C Standard have introduced new language features. The following
shall not be used:
- The _Alignas alignment specifier, the _Alignof operator and the <stdalign.h>
  header file;" [MISRA C:2012 Amendment 2]

This rule detects when the '_Alignas' alignment specifier or the '_Alignof'
operator are used.
SINCE 

2020.1
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

_Alignas(16) int v1;          /* Violation */

int f1()
{
    return _Alignof(float);   /* Violation */
}
REPAIR 

To fix the code, use neither the '_Alignas' alignment specifier nor
the '_Alignof' operator.
