Do not use std::move on objects declared with the const or const & type [CODSTA-MCPP-53-3]

DESCRIPTION 

"An object declared with the 'const' or 'const &' type will never actually be
moved as a result of calling 'std::move'." [HIC++ Coding Standard]

This rule detects when 'std::move' is called with an argument of 'const' or
'const &' type.
SINCE 

2021.1
BENEFITS 

This rule helps you prevent unexpected behavior and improve code clarity.
EXAMPLE 

In the following example, 'std::move' is called on an object of 'const' type.

#include <utility>

class C
{
};

void foo()
{
  const C c{};
  C c2 = std::move(c);                 // Violation
}
REPAIR 

#include <utility>

class C
{
};

void foo()
{
  const C c{};
  C c2 = c;                            // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A18-9-3. The std::move shall not be used on objects declared const or
   const&

2. High Integrity C++ Coding Standard
   17.3.1. Do not use std::move on objects declared with const or const & type

3. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 28.6.1 The argument to std::move shall be a non-const lvalue
