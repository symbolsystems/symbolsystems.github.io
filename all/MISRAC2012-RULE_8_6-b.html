An identifier with external linkage shall have external definition [MISRAC2012-RULE_8_6-b-2]

DESCRIPTION 
An identifier with external linkage shall have external definition [GLOBAL-NOEXTERNDEF]


DESCRIPTION


"The behaviour is undefined if an identifier is used for which multiple
definitions exist (in different files) or no definition exists at all."
[MISRA C:2012 Rule 8.6]

This rule detects when identifiers with external linkage (objects or functions)
do not have any external definition.

See also: GLOBAL-ONEEXTERNDEF
SINCE 

v10.3
NOTES 

The rule analyzes declarations and definitions in files in the testing scope.
For this reason, the set of findings reported for the same file may vary
depending on the scope of analysis.

The rule operates on all files in the testing scope. For this reason, it is
important to specify a testing scope that reflects the system architecture
(for example, it may be appropriate to define a testing scope that corresponds
to a single, complete, linkable module).
EXCEPTIONS 

The rule does not check template functions and members of template
classes.
The rule does not report violations on objects and functions that are declared
in unnamed namespaces.
BENEFITS 

This rule prevents undefined behaviors.
EXAMPLE 

Assumption: the testing scope contains only 'test.c' and 'test.h'.

// test.h
extern int x1;                // Violation - there is no definition of 'x1'
                              // in testing scope
extern void foo1(void);       // Violation - there is no definition of 'foo1()'
                              // in testing scope
extern void foo2(void);       // Violation - there is no definition of 'foo2()'
                              // in testing scope
// test.c
#include "test.h"

void foo2(void);              // Violation - there is no definition of 'foo2()'
                              // in testing scope
REPAIR 

You can improve your code by adding a definition or removing an unnecessary declaration

// test.h
extern int x1;                // OK
extern void foo2(void);       // OK

// test.c
#include "test.h"
int x1 = 0;                   // OK
void foo2(void)               // OK
{
   // ...
}
