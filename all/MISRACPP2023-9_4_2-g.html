The condition of a 'switch' statement should only be preceded by an optional simple-declaration [MISRACPP2023-9_4_2-g-2]

DESCRIPTION 

The init-statement in a 'switch' can be either a simple-declaration that allows
the declaration of a variable with restricted scope or an expression-statement.
Using an expression-statement in the init-statement introduces complexity
without any extra benefit.

This rule detects when an expression-statement is used in the init-statement
of a 'switch'.
SINCE 

2023.2
NOTES 

A simple-declaration is a statement that introduces, creates, and optionally
initializes one or several variables.
BENEFITS 

This rule helps you improve the readability and maintainability of the code.
EXAMPLE 

void f(int i)
{
  switch (i = 0; i)               // Violation
  {
    case 0:
      // ...
      break;
    case 1:
      // ...
      break;
    default:
      // ...
      break;
  }
}
REPAIR 

void f()
{
  switch (int i = 0; i)           // OK
  {
    case 0:
      // ...
      break;
    case 1:
      // ...
      break;
    default:
      // ...
      break;
  }
}
