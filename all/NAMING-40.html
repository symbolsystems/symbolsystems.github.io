Only the first word of the name of a class, structure, namespace, enumeration, or typedef will begin with an uppercase letter [NAMING-40-3]

DESCRIPTION 

"The first word of the name of a class, structure, namespace, enumeration,
or type created with typedef will begin with an uppercase letter.
All others letters will be lowercase."

See also: NAMING-10, NAMING-33
SINCE 

v7.1
NOTES 

If one wants to eliminate some abbreviations from checking it can be done by
adding abbreviation string to array 'allowed_abbreviation' in rule's python
method 'add_abbreviation' e.g. allowed_abbreviation = ["RGB", "IBM"]

The rule does not check names beginning with underscore. Underscore is often
used as the first character in the name of identifier from standard library,
so in order to avoid name collisions, identifiers should not begin with '_'.
EXCEPTIONS 

The first letter of a typedef name may be in lowercase if the typedef
is a class member or when it is used as a replacement for primitive types.
BENEFITS 

Rule improves readability and maintainability.
EXAMPLE 

class sample_class                  // Violation
{
   // ...
};

struct Sample_Struct                // Violation
{
   // ...
};

enum RGB_colors {red, green, blue}; // Violation
REPAIR 

class Sample_class                  // OK
{
   // ...
};

struct Sample_struct                // OK
{
   // ...
};

// When array 'allowed_abbreviation' in rule's python method
// 'add_abbreviation' will be modified in the following way:
// allowed_abbreviation = ["RGB"]
// then violation in line below won't be reported
enum RGB_colors {red, green, blue}; // OK

typedef int integral_type;          // OK - exception
