A function declared with a _Noreturn function specifier shall have void return type [MISRA2012-RULE-17_10-2]

DESCRIPTION 

"By definition, use of the _Noreturn function specifier indicates
unambiguously that a function will not return to its caller.
A function that cannot return cannot provide a return value and shall
therefore be defined with a void return type." [MISRA C:2012 Amendment 3]

This rule detects functions whose return type is other than 'void' that are
declared as 'noreturn' with one of the following:
  * "_Noreturn" function specifier
  * "noreturn" macro from stdnoreturn.h
  * __attribute__((noreturn))
  * [[ noreturn ]] attribute
SINCE 

v10.4
BENEFITS 

This rule helps you avoid unexpected behavior and prevent unexpected failures
at runtime.
EXAMPLE 

In the following example, the "foo" function declared with the "_Noreturn"
specifier returns an int value.

_Noreturn int foo(void);                                    // Violation
REPAIR 

To fix the code, change the return type of the "foo" function to "void".

_Noreturn void foo(void);                                   // OK
