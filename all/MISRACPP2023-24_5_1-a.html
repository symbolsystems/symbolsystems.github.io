The character handling functions from <cctype> and <cwctype> shall not be used [MISRACPP2023-24_5_1-a-2]

DESCRIPTION 

When values other than in the range of type unsigned char or a negative value
corresponding to EOF are passed as an argument to a function declared
in the Standard Library header <cctype> or <cwctype> (or its equivalent
<ctype.h> or <wctype.h>), then this results in undefined behavior.

This rule detects when functions with the following names that are defined
with extern C linkage (in external library or locally by the user) are used
(called or their addresses are taken):
'isalnum', 'isalpha', 'isblank', 'iscntrl', 'isdigit', 'isgraph', 'islower',
'isprint', 'ispunct', 'isspace', 'isupper', 'isxdigit', 'tolower', 'toupper',
'iswalnum', 'iswalpha', 'iswblank', 'iswcntrl', 'iswctype', 'iswdigit',
'iswgraph', 'iswlower', 'iswprint', 'iswpunct', 'iswspace', 'iswupper',
'iswxdigit', 'towctrans', 'towlower', 'towupper', 'wctrans', 'wctype'.

This rule detects function-like macros with the above names, when one
of the headers: <cctype>, <ctype.h>, <cwctype> or <wctype.h> is included
in a translation unit.
SINCE 

2023.2
NOTES 

This rule recommends using equivalent classification and case mapping functions
from the <locale> header that are safer to use.
BENEFITS 

This rule helps you prevent undefined behavior.
EXAMPLE 

#include <cctype>

void f1(int a)
{
  std::isalnum(a);                    // Violation
}
REPAIR 

You can improve your code by using functions from the <locale> header.

#include <locale>

void f1(int a, const char* loc)
{
  std::isalnum(a, std::locale(loc));  // OK
}
