All direct, non-static data members of an aggregate class should be initialized as soon as the object is created [INIT-20-3]

DESCRIPTION 

Non-static data members declared in an aggregate class can be initialized
using either default member initialization or aggregate initialization
when declaring an object.

This rule detects when not all non-static data members are initialized
when declaring an object of aggregate type.
SINCE 

2023.2
NOTES 

An aggregate is a class or struct with
  - no user-provided, explicit, or inherited constructors, and
  - no private or protected non-static data members, and
  - no virtual functions, and
  - no virtual, private, or protected base classes
EXCEPTIONS 

This rule does not apply to data members whose type has a constructor.
This rule does not apply to objects of the 'va_list' type.
BENEFITS 

This rule helps you reduce the risk of an invalid state existing after
successful construction.
EXAMPLE 

struct S1
{
    int m1;
    int m2;
    int m3 = 0;
};

void f1()
{
    S1 s;              // Violation
}
REPAIR 

struct S1
{
    int m1 = 0;
    int m2 = 0;
    int m3 = 0;
};

void f1()
{
    S1 s;              // OK
}

struct S2
{
    int m1;
    int m2;
    int m3 = 0;
};

void f2()
{
    S2 s{};            // OK
}
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   11.6.1 Aggregates

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 15.1.4 All direct, non-static data members of a class should be initialized before the class object is accessible
