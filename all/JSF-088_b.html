Multiple inheritance shall not use any public implementations [JSF-088_b-2]

DESCRIPTION 

Multiple inheritance shall only be allowed if "implementation" classes are not
inherited with public access specifier.

Rule checks only classes which have more than one direct parent class.

A violation is reported if a direct base class is found, which is inherited
with public access specifier, and the class is not an "interface".

An "interface" is a class which has no member variables and all
member functions (except for destructor) are pure virtual.

See also: OOP-07_a, OOP-07, OOP-06, OOP-05, OOP-04, OOP-03
BENEFITS 

Simpler and easier to maintain and understand inheritance hierarchy.
EXAMPLE 

// Non-interface (implementation)
class A { int x; };

// Non-interface (implementation)
struct B { virtual void foo(); };

// Interface
struct C { virtual void foo() = 0; };

// Examples of non-compliant code:

// Implementation class 'A' with public inheritance
struct D : public A, private B { };   // Violation

// Implementation class 'B' with public inheritance
class E : protected C, public B { };  // Violation
REPAIR 

// Non-interface (implementation)
class A { int x; };

// Non-interface (implementation)
struct B { virtual void foo(); };

// Interface
struct C { virtual void foo() = 0; };

// Examples of compliant code:

// Implementation class 'A' with non-public inheritance
struct D : protected A, private B { };  // OK

// Implementation class 'B' with non-public inheritance
class E : protected C, private B { };   // OK

// No multiple inheritance:
struct F : public A { int x; };         // OK
class G : public F { int x; };          // OK
