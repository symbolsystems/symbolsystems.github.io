If a member function can be made static then it shall be made static, otherwise if it can be made const then it shall be made const [CODSTA-CPP-78-3]

DESCRIPTION 

"Declaring a member function static or const limits its access
to the non-static data members."
See also: CODSTA-CPP-54
SINCE 

v7.2
BENEFITS 

Rule prevents unintentional modification of the data.
EXAMPLE 

class A
{
public:
    int foo1 ()         // Violation - can be static
    {
        return m_s;
    }
    int foo2 ()         // Violation - can be const
    {
        return m_i;
    }
    
private:
    int m_i;
    static int m_s;
};
REPAIR 

class A
{
public:
    static int foo1 ()        // OK
    {
        return m_s;
    }
    int foo2 () const         // OK
    {
        return m_i;
    }
    
private:
    int m_i;
    static int m_s;
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M9-3-3. If a member function can be made static then it shall be made
   static, otherwise if it can be made const then it shall be made const

2. General Principles of Software Validation;
   Final Guidance for Industry and FDA Staff
   Document issued on: January 11, 2002

3. High Integrity C++ Coding Standard
   9.1.1. Declare static any member function that does not require this.
   Alternatively, declare const any member function that does not modify the
   externally visible state of the object

4. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 9, Rule 9-3-3
