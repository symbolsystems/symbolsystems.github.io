Parameters in an overriding virtual function shall not specify different default arguments [CODSTA-CPP-210-2]

DESCRIPTION 

This rule reports when an overriding function specifies a default argument
value and:
* it is not a constant expression, or
* the overridden function does not specify a default value for this
parameter, or
* the overridden function specifies a default value for this parameter
that is not a constant expression, or
* the overridden function specifies a different value of the argument.
SINCE 

2023.2
NOTES 

The rule does not report when either the overridden or overriding function is
a template instance or a template specialization.
BENEFITS 

The rule helps you avoid confusion about which default argument value will
be used when calling a virtual function via a pointer or reference to the base
or derived class.
EXAMPLE 

class Base
{
public:
    virtual void f1(int32_t a = 0);
    virtual void f2(int32_t a = 0);
    virtual void f3(int32_t a = x());
    virtual void f4(int32_t a = x());
    virtual void f5(int32_t a);
};
class Derived: public Base
{
public:
    void f1(int32_t a = 1) override;   // Violation - different value
    void f2(int32_t a = 1) override;   // Violation - different value
    void f3(int32_t a = x()) override; // Violation - not a constant
    void f4(int32_t a = 1) override;   // Violation - different value
    void f5(int32_t a = 1) override;   // Violation - introducing a default value
};
REPAIR 

class Base
{
public:
    virtual void f1(int32_t a = 0);
    virtual void f2(int32_t a = 0);
    virtual void f3(int32_t a = x());
    virtual void f4(int32_t a = x());
    virtual void f5(int32_t a);
};
class Derived: public Base
{
public:
    void f1(int32_t a = 0) override;   // OK
    void f2(int32_t a) override;       // OK
    void f3(int32_t a) override;       // OK
    void f4(int32_t a) override;       // OK
    void f5(int32_t a) override;       // OK
};
