NULL identifier shall not be used; use plain 0 instead [JSF-175-2]

DESCRIPTION 

"A pointer shall not be compared to NULL or be assigned NULL;
use plain 0 instead. The NULL macro is an implementation-defined
C++ null pointer constant that has been defined in multiple ways
including 0, 0L, and (void*)0."

The rule reports a violation on each NULL identifier in code
(not only compared or assigned to pointer) due to it assumes
that each use of NULL shall be replaced by 0.
NOTES 

If a NULL identifier is not a macro (e.g. is a constant variable)
then violation is reported by rule as well (on declaration and use).
BENEFITS 

Due to C++â€™s stronger type-checking, the rule advises
the use plain 0 rather than any suggested NULL macro.
EXAMPLE 

#include <stddef.h>

void foop(int* p);
int* ptr = NULL;      // Violation

int* foo(int* par ) {
    int* ptr1 = NULL; // Violation
    
    if(par == NULL){  // Violation
        return NULL;  // Violation	
    } else {
    	foop(NULL);   // Violation
    	return par;
    }
}
REPAIR 

#include <stddef.h>

void foop(int* p);
int* ptr = 0;          // OK

int* foo(int* par ) {
    int* ptr1 = 0;     // OK
    
    if(par == 0){      // OK
        return 0;      // OK	
    } else {
    	foop(0);       // OK
    	return par;
    }
}
