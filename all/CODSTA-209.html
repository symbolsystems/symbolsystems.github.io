The facilities that are specified as being provided by <stdatomic.h> should not be used [CODSTA-209-3]

DESCRIPTION 

"Updates to the C Standard have introduced new language features.
The following shall not be used:
- The _Atomic type specifier, the _Atomic type qualifier and the facilities
  that are specified as being provided by <stdatomic.h>;"
[MISRA C:2012 Amendment 2]

This rule detects when the following are used in code:
- the <stdatomic.h> header file
- types, macros, functions, or typedefs defined in the <stdatomic.h> header
  file
SINCE 

2020.1
BENEFITS 

"Use of the language features restricted by this rule may have instances
of undefined, unspecified or implementation-defined behaviour associated
with them. In addition, features may also exhibit well defined behaviour
that does not meet developer expectations." [MISRA C:2012 Amendment 2]
EXAMPLE 

#include <stdatomic.h>                         /* Violation */

void f1(volatile atomic_flag* obj) {           /* Violation */
    int i;
    atomic_int* ai;                            /* Violation */
    atomic_init(ai, i);                        /* Violation */
    atomic_flag_test_and_set(obj);             /* Violation */
}
REPAIR 

Do not use the <stdatomic.h> header file or the function, type, macro,
and typedef defined in that file.
