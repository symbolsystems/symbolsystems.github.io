Every non-empty case clause in a switch statement shall be terminated with a break statement [JSF-193-2]

DESCRIPTION 

"The last statement in every switch clause shall be a break statement, 
or if the switch clause is a compound statement, then the last statement
in the compound statement shall be a break statement."

The rule reports a violation if a non-empty case clause is not terminated
by break or return statement.
NOTES 

It is recommended to add a break statement at the end of default clause.
BENEFITS 

"Terminating a non-empty case clause in a switch statement with a break
statement eliminates potentially confusing behavior by prohibiting control
from falling through to subsequent statements."
EXAMPLE 

void foo( int i )
{
    switch( i )
    {
        case 0 :         // Violation
            i = 4;
            break;
            i++;
        case 1 :         // Violation
            i = 7;
            if (i > 0)
            {
                i = 5;
                break;
            }
        case 2 :         // Violation
            {
                i = 3;
            }
        default:
            i = 8;
    }
}
REPAIR 

void foo( int i )
{
    switch( i )
    {
        case 0 :         // OK
            i = 4;
            i++;
            break;
        case 1 :         // OK
            i = 7;
            if (i > 0)
            {
                i = 5;
            }
            break;
        case 2 :         // OK
            {
                i = 3;
                break;
            }
        case 3 :         // OK
            {
                i = 3;
            }
            break;
        case 4 :         // OK - empty case
        default:
            i = 8;
            break;
    }
}
