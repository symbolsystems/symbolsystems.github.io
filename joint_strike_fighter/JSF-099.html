Namespaces will not be nested more than two levels deep [JSF-099-3]

DESCRIPTION 

Namespaces will not be nested more than two levels deep.
Please see Example section on how the nesting level is defined.
SINCE 

v7.1
NOTES 

It is possible to change the allowed nesting level by modifying 
rule in Rule Wizard. The level can be changed by adding 
(for example by using copy-paste) or removing the 
MemberOf:namespace elements.
BENEFITS 

Simplicity and clarity. 
Deeply nested namespaces can be difficult to comprehend and use correctly.
EXAMPLE 

namespace A {               // OK: depth = 1
    namespace B {           // OK: depth = 2
        namespace C {       // Violation: depth = 3
            namespace D {   // Violation: depth = 4
            }
        }
        namespace {         // Violation: depth = 3
        }
    }
    namespace X {           // OK: depth = 2
    }
}
namespace A {               // OK: depth = 1
    namespace X {           // OK: depth = 2
        namespace Y {       // Violation: depth = 3
        }
    }
}
REPAIR 

It is not allowed to use more than 2 namespace nesting levels.
REFERENCES 

JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
Chapter 4.11 Namespaces, AV Rule 99
