Multiple inheritance shall be limited to at most 1 protected implementation [JSF-088-2]

DESCRIPTION 

Multiple inheritance shall only be allowed if at most one protected
implementation is used.

Rule counts number of base classes which:
* are directly inherited, and
* are inherited with protected access specifier, and
* are not "interfaces"

A violation is reported when more than one such class is found.

An "interface" is a class which has no member variables and all
member functions (except for destructor) are pure virtual.

See also: OOP-07_b, OOP-07, OOP-06, OOP-05, OOP-04, OOP-03
BENEFITS 

Simpler and easier to maintain and understand inheritance hierarchy.
EXAMPLE 

// Non-interface (implementation)
class A { int x; };

// Non-interface (implementation)
struct B { virtual void foo(); };

// Examples of non-compliant code:

// More than one protected implementation inheritance:
struct D : protected A, protected B { };   // Violation
REPAIR 

// Non-interface (implementation)
class A { int x; };

// Non-interface (implementation)
struct B { virtual void foo(); };

// Interface
struct C { virtual void foo() = 0; };

// Examples of compliant code:

// One protected implementation inheritance:
struct D : protected A, private B { };   // OK

// One protected implementation inheritance, one protected interface:
class E : protected A, protected C { };   // OK
