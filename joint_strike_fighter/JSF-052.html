Identifiers for constant and enumerator values shall be lowercase [JSF-052-2]

DESCRIPTION 

"Although it is an accepted convention to use uppercase letters for constants
and enumerators, it is possible for third party libraries to replace
constant/enumerator names as part of the macro substitution process
(macros are also typically represented with uppercase letters)." [JSF]

This rule detects enumeration constants and const variables whose names contain
an uppercase letter.

See also: NAMING-05, NAMING-06, NAMING-07, NAMING-34, NAMING-35, NAMING-44
SINCE 

v7.1
EXCEPTIONS 

The Joint Strike Fighter coding standard allows uppercase letters in acronyms.
For the rule to support this exception, you must:
1. Create a text file that contains a whitespace-separated list of acronyms
   you want this rule to recognize. For example, the file content may resemble
   the following:
     RGB XYZ
     UTF8
2. Create an environment variable called CPPTEST_JSF_ACRONYMS and set its value
   to the path of the above file.

This will prevent the rule from reporting uppercase letters in the acronyms
listed in the file. For example:
  enum E {
      RGB_value,   // OK if "RGB" is an acronym
      UTF8,        // OK if "UTF8" is an acronym
      a_XYZ_b      // OK if "XYZ" is an acronym
  };

Note that the acronym must be separated from other parts of the identifier name
with an underscore. For example:
    RGBvalue     // Violation - "RGB" is not recognized as an acronym
BENEFITS 

This rule helps you improve code readability and maintainability.
EXAMPLE 

const int Max_Pressure = 100;    // Violation
enum Switch_position {
    Up,       // Violation
    DOWN,     // Violation
    notKnown  // Violation
};
REPAIR 

To fix the code, replace the uppercase letters with lowercase letters in
the names of the variables.

const int max_pressure = 100;    // OK
enum Switch_position {
    up,       // OK
    down,     // OK
    notknown  // OK
};
