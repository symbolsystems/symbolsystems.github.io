>, >=, <, <= shall not be applied to pointer types except where they point to the same array [MISRA2004-17_3-3]

DESCRIPTION 

"Attempting to make comparisons between pointers will produce undefined
behaviour if the two pointers do not point to the same object."
[MISRA-C:2004]

This rule detects comparisons between expressions of pointer or array type
that do not point to the same object or array.

See also: BD-PB-PTRCMP
NOTES 

This rule only detects cases when the direct operand of a relational operator
is one of the following: a variable, a parameter, an expression '&a', 'a.b',
or 'a->b'.
The rule does not check the flow, which may result in reporting false positive
or false negative violations, depending on how pointers are assigned on
different paths.
BENEFITS 

This rule allows you to avoid undefined behavior.
EXAMPLE 

void f1( int* p ) {
    int a1[3];
    int a2[4];
    int* p1 = a1;
    int* p2 = a2;
    int* p3 = 0;

    if (p1 < p2);   // Violation
    if (p1 < p3);   // Violation
    if (p1 < p);    // Violation - it is not known where 'p' points
}

struct S {
    int ms1;
    int ms2;
};

void fs1() {
    S s1;
    S s2;
    &s1.ms1 > &s2.ms1;  // Violation
}
REPAIR 

You can fix your code by only comparing pointers that point to the same
object or array.

/* Examples of correct code */
void f1( ) {
    int a1[3];

    int* p1 = a1;
    int* p2 = &a1[2];

    if (p1 < p2);           // OK
}

struct S {
    int ms1;
    int ms2;
};

void fs1() {
    S s1;
    &s1.ms1 > &s1.ms2;      // OK
}
