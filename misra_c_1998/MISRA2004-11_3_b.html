A cast should not convert an integral type to a pointer type [MISRA2004-11_3_b-3]

DESCRIPTION 

"A cast should not be performed between a pointer type and an
integral type." [MISRA-C:2004]

This rule detects when an expression of an integral type
is explicitly cast to a pointer type.
SINCE 

v7.0
EXCEPTIONS 

The rule allows casting the integer constant '0' to a pointer type.

The rule assumes that UINT_PTR, INT_PTR, uintptr_t, or intptr_t are integral
types capable of holding a pointer and allows casting them to pointers.
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

#include <stdint.h>

void f1(int32_t *pint) {
    int32_t si;
    pint = (int32_t*)si;       // Violation
}
REPAIR 

#include <stdint.h>

void f1(int32_t *pint) {
    intptr_t si;
    pint = (int32_t*)si;       // OK
}
