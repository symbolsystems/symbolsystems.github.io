External objects should not be declared in implementation files [MISRA-027_b-3]

DESCRIPTION 

"Adherence to this rule will normally mean declaring external objects
in header files which will then be included in all those files that
use those objects (including the files which define the objects)."

The rule reports a violation if an external object is declared
in implementation file. Declaration should be placed in a header
file to make sure the same declaration is used in different files.
SINCE 

v9.4
BENEFITS 

The rule allows to avoid inconsistent declarations.
EXAMPLE 

// In File_1.cpp

int a = 3;

// In File_2.cpp

extern int a;      // Violation
REPAIR 

// In File_1.h

extern int a;      // OK

// In File_1.cpp

#include "File_1.h"

int a = 3;

// In File_2.cpp

#include "File_1.h"
