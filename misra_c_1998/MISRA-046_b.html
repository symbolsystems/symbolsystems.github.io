Assignment statements should not be nested within other assignment statements [MISRA-046_b-3]

DESCRIPTION 

"Assignments nested within expressions cause additional side effects. The best 
way to avoid any chance of this leading to a dependence on order of evaluation 
is to not embed assignments within expressions."

The rule reports a violation if an assignment statement is nested in other
assignment statement.

See also: MISRA2004-12_2_e, PB-02
SINCE 

v9.4
BENEFITS 

The rule improves readability of code and prevents evaluation of expression
dependent on compiler version.
EXAMPLE 

void foo( int x, int y, int z ) {
    x = y = z / 3;  // Violation
    x = y = y++;    // Violation
}
REPAIR 

void foo( int x, int y, int z ) {
    y = z / 3;  // OK
    x = y;      // OK
}
