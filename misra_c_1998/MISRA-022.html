Declare objects at function scope [MISRA-022-5]

DESCRIPTION 

Declarations of objects should be at function scope
unless a wider scope is necessary.

This rule reports on any global (or namespace) variable declaration.

See also: MISRA2004-8_7, OPT-01
NOTES 

A violations is reported also if:
- a variable has external linkage and could be used from different translation
  units
- a variable is used by several functions
BENEFITS 

Prevents using global variables.
EXAMPLE 

int globalVar;  // Violation
REPAIR 

void foo( ) {
    int localVar;   // OK 
}
