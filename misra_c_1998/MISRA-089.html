The #include directive shall be followed by either a <filename> or "filename" sequence [MISRA-089-3]

DESCRIPTION 

"The #include directive shall be followed by either a <filename>
or "filename" sequence."

See also: PREPROC-09
EXCEPTIONS 

Standards MISRA-C:2004, MISRA C++:2008 and MISRA C:2012 permit to use a macro
after #include directive. However, it is not allowed by the rule, so it reports
the violation on the following code:

#define MY_HEADER "header.h"
#include MY_HEADER                   /* Violation */
BENEFITS 

The rule allows to use only permitted forms for the #include directive.
EXAMPLE 

/* Following statement is not compliant and may not compile */

#include foo.h    /* Violation */
REPAIR 

#include <foo1.h> /* OK */
#include "foo2.h" /* OK */
