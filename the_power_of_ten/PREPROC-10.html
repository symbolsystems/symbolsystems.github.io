The #ifndef pre-processor directive will only be used to prevent multiple inclusions of the same header file [PREPROC-10-3]

DESCRIPTION 

"Conditional code compilation should be kept to a minimum as it can
significantly obscure testing and maintenance efforts." [JOINT STRIKE FIGHTER]

This rule detects when the #ifndef preprocessor directive is used in code.
SINCE 

v7.1
EXCEPTIONS 

The rule allows using the #ifndef directive in the mechanism that prevents
multiple inclusions of the same header file.

#ifndef FOO_HH        // OK
#define FOO_HH
//...
#endif
BENEFITS 

This rule helps you improve code maintainability.
EXAMPLE 

#ifndef MAX                  // Violation
#define MAX 10
#endif
REPAIR 

static const int MAX = 10;   // OK
REFERENCES 

1. High Integrity C++ Coding Standard
   16.1.1. Use the preprocessor only for implementing include guards, and
   including header files with include guards

2. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.6 Pre-Processing Directives, AV Rule 28

3. MISRA C++:2008 - Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 16, Rule 16-2-1

4. The Power of Ten - Rules for Developing Safety Critical Code.
