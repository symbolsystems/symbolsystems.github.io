CWE-190 
CWE-190 Integer Overflow or Wraparound
RULES

Avoid wraparounds when performing arithmetic integer operations [CWE-190-a-3]
Avoid possible integer overflow in expressions in which the result is cast to a wider integer type [CWE-190-b-3]
Avoid possible integer overflow in expressions in which the result is assigned to a variable of a wider integer type [CWE-190-c-3]
Avoid possible integer overflow in expressions in which the result is compared to an expression of a wider integer type [CWE-190-d-3]
Integer overflow or underflow in constant expression in '+', '-', '*' operator [CWE-190-e-3]
Integer overflow or underflow in constant expression in '<<' operator [CWE-190-f-4]
Evaluation of constant unsigned integer expressions should not lead to wrap-around [CWE-190-g-5]
Avoid signed integer overflows [CWE-190-h-1]
Evaluation of constant unsigned integer expressions in preprocessor directives should not lead to wraparound [CWE-190-i-5]
