CWE-704 
CWE-704 Incorrect Type Conversion or Cast
RULES

Conversions shall not be performed between a pointer to a function and any other type than pointer to function [CWE-704-a-3]
Conversions shall not be performed between non compatible pointer to a function types [CWE-704-b-3]
Conversions shall not be performed between a pointer to an incomplete type and any other type [CWE-704-c-3]
A cast shall not be performed between a pointer to object type and a pointer to a different object type [CWE-704-d-3]
A conversion should not be performed between a pointer to object type and an integer type other than 'uintptr_t' or 'intptr_t' [CWE-704-e-4]
A conversion should not be performed from pointer to void into pointer to object [CWE-704-f-4]
A cast shall not be performed between pointer to void and an arithmetic type [CWE-704-g-3]
An implicit conversion shall not be performed between pointer to void and an arithmetic type [CWE-704-h-3]
A cast shall not be performed between pointer to object and a non-integer arithmetic type [CWE-704-i-3]
Implicit conversions from wider to narrower integral type which may result in a loss of information shall not be used [CWE-704-j-3]
Implicit conversions from integral to floating type which may result in a loss of information shall not be used [CWE-704-k-3]
Implicit conversions from integral constant to floating type which may result in a loss of information shall not be used [CWE-704-l-3]
