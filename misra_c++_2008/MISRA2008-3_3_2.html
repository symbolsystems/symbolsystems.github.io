If a function has internal linkage then all re-declarations shall include the static storage class specifier [MISRA2008-3_3_2-2]

DESCRIPTION 

"If the declaration of a function includes the 'static' storage class
specifier, then it has internal linkage. A re-declaration of such a function
is not required to have the 'static' keyword, but it will still have internal
linkage. However, this is implicit and may not be obvious to a developer.
It is therefore good practice to apply the 'static' keyword consistently
so that the linkage is explicitly stated." [MISRA C++:2008]

This rule detects when the declaration of a function with internal linkage
does not contain the explicit 'static' keyword.

See also: MISRA2004-8_11
SINCE 

v7.2
NOTES 

"Static storage duration class specifier is redundant and does not need to be
specified if a function is placed in an unnamed namespace." [AUTOSAR M3-3-2]
BENEFITS 

This rule helps you improve the code readability and follow good coding
practices.
EXAMPLE 

static void f1 ( );
void f1 ( ) { }         // Violation
REPAIR 

To fix the code, add the explicit 'static' keyword.

static void f1 ( );
static void f1 ( ) { }  // OK
