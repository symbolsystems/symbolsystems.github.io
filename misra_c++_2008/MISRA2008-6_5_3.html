The loop-counter shall not be modified within condition or statement [MISRA2008-6_5_3-2]

DESCRIPTION 

"Numeric variables being used within a 'for' loop for iteration counting
(loop-counters) shall not be modified in the body of the loop.

A loop-counter is a loop-control-variable that is:
  - Initialized in, or prior to, for-init-statement; and
  - an operand to a relational operator in condition; and
  - modified in expression.

A loop-control-variable is any variable occurring in for-init-statement,
condition or expression.

A loop-control-variable (other than loop-counter) representing logical value 
may be modified in the loop, for example a flag to indicate that something 
has been completed, which is then tested in the for statement."

The rule reports a violation if a loop-counter is modified within a condition
or within a body of 'for' loop.
NOTES 

The rule assumes that an object is modified if it is passed to the function
as a reference to a non-const object (except a universal reference) or its
address is passed to the function as a pointer to a non-const object 
(except functions which are called via function pointers).
BENEFITS 

Modification 'for' loop counter within a body of the loop can lead to 
errors and confusion.
DRAWBACKS 

The rule reports a violation only if a loop-counter is a simple parameter
or variable. If it is an expression (e.g. a.b, a->b, a[b], *a), then
violation is not reported.

struct S { int a; };

void foo(){
    S s;
    S* sp;
  
    int* ptr;
    int arr[3];
  
    for(s.a = 10; s.a < 100; s.a++){
        s.a = 10;      // should be Violation - no violations for expressions
    }
  
    for(sp->a = 10; sp->a < 100; sp->a++){
        sp->a = 10;    // should be Violation - no violations for expressions
    }
  
    for(*ptr = 10; *ptr < 100; (*ptr)++){
        *ptr = 10;     // should be Violation - no violations for expressions
    }

    for(arr[2] = 10; arr[2] < 100; arr[2]++){
        *arr = 10;     // should be Violation - no violations for expressions
    }
}
EXAMPLE 

void f(int* ptr);

void foo( int p ){
    int i;
  
    for(p = 10; p < 100; p++){
        p = p + 10;                            // Violation
    }
  
    for(p = 10; p < 100, p = p + 10; p++){     // Violation
        // ...                   
    }
  
    i = 10;
    for(; i < 100; i++){
        f(&i);                                 // Violation
    } 
}
REPAIR 

void f(const int* ptr);

void foo( int p ){
    int i;
    int flag;
  
    for(p = 10; p < 100; p = p + 11){          // OK
        // ... 
    }
  
    i = 10;
    for(; i < 100; i++){
        f(&i);                                 // OK
    } 
    
    for(i = 10; i < 100 && flag != 0; i++){
       flag = 0;                               // OK
    } 
}
