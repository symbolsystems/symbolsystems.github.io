Digraphs should not be used [MISRA2008-2_5_1-4]

DESCRIPTION 

"Be aware of trigraph and digraph character sequences and avoid them. It is
possible to avoid such sequences arising accidentally by using spaces."
[High Integrity C++]

This rule detects when digraphs are used in code. The rule assumes that
digraphs are the following two-character sequences: <%, %>, <:, :>, %:.
SINCE 

v7.1
NOTES 

The rule doesn't report digraphs in comments and strings.
BENEFITS 

"In most modern environments there is no longer a need for trigraphs or
digraphs, and their use will either result in hard to diagnose compiler errors
or code that is difficult to maintain." [High Integrity C++]

This rule helps you improve code readability, maintainability, and portability.
EXAMPLE 

int tab <: 2 :> = <%0,1%>;                            // Violation

%:define FOO(a, b) a %:%: b                           // Violation
REPAIR 

To fix the code, replace the digraphs with their one-character equivalents.

int tab[2] = {0,1};                                   // OK

#define FOO(a, b) a ## b                              // OK
