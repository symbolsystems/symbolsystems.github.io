The types used for an object, a function return type, or a function parameter shall be token-for-token identical in all declarations and re-declarations [MISRA2008-3_9_1-2]

DESCRIPTION 

"If a re-declaration has compatible types but not types which are
token-for-token identical, it may not be clear to which declaration
that re-declaration refers."
SINCE 

v7.2
BENEFITS 

Rule improves readability and maintainability of code.
DRAWBACKS 

The rule does not detect when a member variable with the constexpr
specifier has a different type in the declaration than in the definition.

typedef int INT;

struct S
{
  static constexpr INT i = 1;                  /* should be Violation */
};

constexpr int S::i;
EXAMPLE 

typedef int INT;

INT i;                 // Violation 
extern int i;

void foo(const int i); 
void foo(int i){}      // Violation
REPAIR 

typedef int INT;

INT i;                  // OK
extern INT i;

void foo(const int i);
void foo(const int i){} // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M3-9-1. The types used for an object, a function return type, or a function
   parameter shall be token-for-token identical in all declarations and re-
   declarations

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 3, Rule 3-9-1
