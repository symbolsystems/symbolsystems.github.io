An object shall not be assigned to an overlapping object [MISRA2008-0_2_1_b-2]

DESCRIPTION 

"An object shall not be assigned to an overlapping object."
Rule detects if two different members of the same union are assigned
to each other using the same object
SINCE 

v7.2
BENEFITS 

"Assigning between objects that have an overlap in their physical storage
leads to undefined behaviour."
EXAMPLE 

union U {
    int         iValue; 
    long        lValue;
    double      dValue;
};

void main( ) {
    union U a, b;
    union U *p;

    a.dValue  = a.lValue;    // Violation
    p->dValue = p->iValue;   // Violation
}
REPAIR 

union U {
    int         iValue; 
    long        lValue;
    double      dValue;
};

void main( ) {
    union U a, b;
    union U *p, *q;
    a.dValue  = b.lValue;     // OK
    p->dValue = q->iValue;    // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M0-2-1. An object shall not be assigned to an overlapping object

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 0, Rule 0-2-1
