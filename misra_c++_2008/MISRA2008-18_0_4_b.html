The time handling functions and macros of library <ctime> shall not be used [MISRA2008-18_0_4_b-2]

DESCRIPTION 

"The standard header file <time.h> shall not be #included, and none of the
features that are specified as being provided by <time.h> shall be used.

The time and date functions have unspecified, undefined
and implementation-defined behaviours associated with them."
[MISRA C:2012 Rule 21.10 with Amendment 3]

This rule detects the following identifiers with the names of functions
and macros defined in the <time.h> (or <ctime>) library:
time, clock, ctime, asctime, difftime, gmtime, localtime, mktime, strftime,
wcsftime, TIME_UTC, CLOCKS_PER_SEC
SINCE 

2022.2
NOTES 

This rule also detects identifiers that are not defined in <time.h>,
but have the same names as macros or functions from this header.

int clock(void);

void f1 ( void )
{
    int t = clock ( );    // Violation
}
BENEFITS 

The rule helps you prevent unspecified, undefined and implementation-defined
behavior.
EXAMPLE 

#include <time.h>

void dangerFunction(time_t time);
int main ()
{
    time_t rawtime;
    struct tm * ptm;

    time ( &rawtime );          // Violation
    ptm = gmtime ( &rawtime );  // Violation
    dangerFunction(rawtime);

    return 0;
}
REPAIR 

Do not use identifiers with the names of functions and macros defined in the
<time.h> header.
