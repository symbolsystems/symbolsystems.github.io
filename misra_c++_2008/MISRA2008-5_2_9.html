A cast should not convert a pointer type to an integral type [MISRA2008-5_2_9-4]

DESCRIPTION 

"The size of integer that is required when a pointer is converted to an integer
is implementation-defined. Casting between a pointer and an integer type should
be avoided where possible, but may be unavoidable when addressing memory mapped
registers or other hardware specific features." [MISRA C++:2008]

This rule detects when an expression of a pointer type is explicitly cast to
an integral type.
SINCE 

v7.0
EXCEPTIONS 

The rule assumes that UINT_PTR, INT_PTR, uintptr_t, or intptr_t are integral
types capable of holding a pointer and allows casting pointers to them.
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

#include <stdint.h>

void f1(int *pint) {
    int32_t si = (int32_t)pint;       // Violation
    uint32_t ui = (uint32_t)pint;     // Violation
}
REPAIR 

#include <stdint.h>

void f1(int *pint) {
    intptr_t si = (intptr_t)pint;     // OK
    uintptr_t ui = (uintptr_t)pint;   // OK
}
