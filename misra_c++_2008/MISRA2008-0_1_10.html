Every defined function with internal linkage shall be used [MISRA2008-0_1_10-2]

DESCRIPTION 

This rule detects definitions of unused functions that cannot be referred to
from scopes of other translation units.
The rule assumes that a function is used when it is called
or has its address taken.

This rule checks:
- functions with internal linkage
- functions with no linkage
- functions with external C++ linkage that are defined in anonymous namespaces
  (except default constructors, destructors and virtual functions)

See also: GLOBAL-UNUSEDFUNC
SINCE 

v7.2
NOTES 

"Note that an unused prototype is not a violation of this rule."
[MISRA C++:2008]
BENEFITS 

"Functions or procedures that are not called may be symptomatic
of a serious problem, such as missing paths." [MISRA C++:2008]
EXAMPLE 

/* The functions 'foo1', 'foo2', foo3' are not used in the translation unit */
static void foo1() { /* ... */ }   /* Violation */

namespace {
    void foo2() { /* ... */ }      /* Violation */
    struct S {
        void foo3() { /* ... */ }  /* Violation */
    };
}
REPAIR 

static void foo1() { /* ... */ }   /* OK */

namespace {
    void foo2() { /* ... */ }      /* OK */
    struct S {
        void foo3() { /* ... */ }  /* OK */
    };
}

static void foo4(int);             /* OK - prototype */

void f_use() {
    void (*fp)() = foo1;
    struct S* sp;
    foo2();
    sp->foo3();
}
