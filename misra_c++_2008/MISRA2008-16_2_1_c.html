The #ifdef, #else, #elif preprocessor directives should not be used [MISRA2008-16_2_1_c-2]

DESCRIPTION 

"Limit the use of the pre-processor to those cases where it is necessary."
[JOINT STRIKE FIGHTER]

Conditional code compilation can hinder testing and maintenance and should
therefore be kept to a minimum.

This rule detects when the #ifdef, #else, and #elif preprocessor directives are
used in code.
SINCE 

v7.1
BENEFITS 

This rule helps you improve code maintainability.
EXAMPLE 

#ifdef M1    // Violation
// ..
#elif M2     // Violation
// ..
#else        // Violation
//..
#endif
REPAIR 

Do not use the #ifdef, #else,or #elif preprocessor directives.
REFERENCES 

1. High Integrity C++ Coding Standard
   16.1.1. Use the preprocessor only for implementing include guards, and
   including header files with include guards

2. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.6 Pre-Processing Directives, AV Rule 26

3. MISRA C++:2008 - Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 16, Rule 16-2-1

4. The Power of Ten - Rules for Developing Safety Critical Code.
   Rule 8
