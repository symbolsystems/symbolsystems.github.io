All functions with void return type shall have external side effect(s) [MISRA2008-0_1_8_b-2]

DESCRIPTION 

"A function which does not return a value and which does not have external side
effects will only consume time and will not contribute to the generation of any
outputs, which may not meet developer expectations." [MISRA C++:2008]

This rule reports violations on empty functions with void return type.
The function is non empty if, after preprocessing, contains a code.
SINCE 

v9.6
NOTES 

It is possible that some functions intentionally have no code at all
(e.g. visitors, interface implementations, callbacks).
This rule reports these cases as well.
BENEFITS 

Rule prevents unnecessary time consumption.
EXAMPLE 

void foo1( void ) // Violation
{
}

void foo2( void ) // Violation
{
    // comments only
}

void foo3( void ) // Violation - no code after preprocessing
{
#if 0
    int i;
#endif
}
REPAIR 

int foo1( int p )   // OK
{
    return p*10;
}

void foo2( int* p ) // OK
{
    (*p) = 10;
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M0-1-8. All functions with void return type shall have external side
   effect(s)

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 0, Rule 0-1-8
