Every defined function with external linkage shall be used [MISRA2008-0_1_10_b-2]

DESCRIPTION 


This rule detects definitions of unused functions that can be referred to
from scopes of other translation units.
The rule assumes that a function is used when it is called
or has its address taken.

This rule checks:
- functions with external C linkage
- functions with external C++ linkage that are not defined in anonymous
  namespaces (except default constructors, destructors and virtual functions)

See also: OPT-30
SINCE 

v10.3
NOTES 

An unused function prototype or a definition of a 'main' function are not 
considered to be a violation of this rule. 

The rule analyzes declarations and definitions in files in the testing scope.
For this reason, the set of findings reported for the same file may vary
depending on the scope of analysis.

The rule operates on all files in the testing scope. For this reason, it is
important to specify a testing scope that reflects the system architecture
(for example, it may be appropriate to define a testing scope that corresponds
to a single, complete, linkable module).
BENEFITS 

"Functions or procedures that are not called may be symptomatic
of a serious problem, such as missing paths." [MISRA C++:2008]
EXAMPLE 

/* The functions 'foo1', 'foo2', foo3' are not used in the testing scope */
void foo1() { /* ... */ }              /* Violation */

struct S {
    void foo2();
    static void foo3() { /* ... */ }   /* Violation */
};

void S::foo2() { /* ... */ }           /* Violation */
REPAIR 

void foo1() { /* ... */ }              /* OK */

struct S {
    void foo2();                       /* OK */
    static void foo3() { /* ... */ }   /* OK */
};

void S::foo2() { /* ... */ }           /* OK */

extern void foo4(int);                 /* OK - prototype */

int main() {                           /* OK */
    void (*fp)() = foo1;
    struct S* sp;
    sp->foo2();
    S::foo3();
    return 0;
}
