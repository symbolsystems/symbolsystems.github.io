A function identifier shall either be used to call the function or it shall be preceded by '&' [MISRA2008-8_4_4-2]

DESCRIPTION 

"If the developer writes 'if( f )' then it is not clear whether the intent
is to test if the address of the function is NULL or if a call to the
function f() should be made and the brackets have been unintentionally omitted.
The use of the & (address-of) operator will resolve this ambiguity."
[MISRA C++:2008]

The rule detects when the function identifier is not preceded
by the & (address-of) operator and is not followed by parentheses.
SINCE 

v9.4
EXCEPTIONS 

"Passing the function by reference, or assigning it to a reference object
is not a violation of this rule." [MISRA C++:2008]

The rule does not report violations on the following identifiers representing
functions from the standard C++ library defined in the 'std' namespace:
boolalpha, noboolalpha, dec, hex, oct, endl, ends, fixed, scientific, flush,
internal, left, right, noshowbase, showbase, noshowpoint, showpoint, noshowpos,
showpos, noskipws, skipws, nounitbuf, nouppercase, uppercase, unitbuf, ws
BENEFITS 

"A function identifier can implicitly convert to a pointer to a function.
In certain contexts this may result in a well-formed program, but which
is contrary to developer expectations." [MISRA C++:2008]
EXAMPLE 

extern void f ( void );

void foo()
{
    if ( 0 == f )      // Violation
    {
        // ...
    }
}
REPAIR 

extern int f ( void );

void foo()
{
    if ( 0 == &f )    // OK
    {
        // ...
    }

    if ( 0 == f() )   // OK
    {
        // ...
    }
}
