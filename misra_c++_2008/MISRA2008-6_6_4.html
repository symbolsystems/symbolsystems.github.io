For any iteration statement there shall be no more than one break or goto statement used for loop termination [MISRA2008-6_6_4-2]

DESCRIPTION 

"For any iteration statement there shall be no more than one break or goto
statement used for loop termination."
SINCE 

v7.2
BENEFITS 

"Restricting the number of exits from a loop is done in the interests of good
structured programming. One break or goto statement is acceptable in a loop
since this allows, for example, for dual outcome loops or optimal coding."
EXAMPLE 

void foo( ) {
    int a;

    for (a = 0; a < 10; a++) { // Violation
        if (a == 5) {
            break;
        }
        if (a == 7) {
            break;
        }
    }
}
REPAIR 

void foo( ) {
    int a;

    for (a = 0; a < 10; a++) { // OK
        if (a == 5 || a == 7) {
            break;
        }
    }
}

2. MISRA C:2012 Guidelines for the use of the C language in critical systems
