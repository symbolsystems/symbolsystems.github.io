Enabled Rules

  • OWASP API Security Top 10 (2019) [OWASP2019]
    • OWASP API10 Insufficient Logging & Monitoring [OWASP2019-API10]
      • All exceptions should be rethrown or logged with standard logger [OWASP2019-API10-a-2]
      • Do not use 'syslog' function for logging purposes [OWASP2019-API10-b-2]
    • OWASP API2 Broken User Authentication [OWASP2019-API2]
      • Do not use weak encryption functions [OWASP2019-API2-a-2]
    • OWASP API3 Excessive Data Exposure [OWASP2019-API3]
      • Do not pass empty container iterators to std algorithms as destinations [OWASP2019-API3-a-2]
      • Avoid accessing arrays out of bounds [OWASP2019-API3-b-2]
      • Avoid accessing arrays and pointers out of bounds [OWASP2019-API3-c-2]
      • Avoid buffer overflow due to defining incorrect format limits [OWASP2019-API3-d-1]
      • Avoid overflow due to reading a not zero terminated string [OWASP2019-API3-e-1]
      • Avoid overflow when reading from a buffer [OWASP2019-API3-f-1]
      • Avoid overflow when writing to a buffer [OWASP2019-API3-g-1]
      • Avoid buffer overflow from tainted data due to defining incorrect format limits [OWASP2019-API3-h-1]
      • Avoid buffer read overflow from tainted data [OWASP2019-API3-i-1]
      • Avoid buffer write overflow from tainted data [OWASP2019-API3-j-1]
      • Properly seed pseudorandom number generators [OWASP2019-API3-k-2]
      • Avoid passing sensitive data to functions that write to log files [OWASP2019-API3-l-2]
      • Avoid race conditions while checking for the existence of a symbolic link [OWASP2019-API3-m-1]
      • Usage of system properties (environment variables) should be restricted [OWASP2019-API3-n-4]
      • Avoid functions which use time from MFC library [OWASP2019-API3-o-3]
      • Do not print potentially sensitive information, resulting from an application error into exception messages [OWASP2019-API3-p-2]
      • A pointer to a structure should not be passed to a function that can copy data to the user space [OWASP2019-API3-q-3]
    • OWASP API4 Lack of Resources & Rate Limiting [OWASP2019-API4]
      • Validate potentially tainted data before it is used to determine the size of memory allocation [OWASP2019-API4-a-2]
      • Ensure resources are freed [OWASP2019-API4-b-1]
    • OWASP API7 Security Misconfiguration [OWASP2019-API7]
      • Properly use errno value [OWASP2019-API7-a-2]
      • Where multiple handlers are provided in a single try-catch statement or function-try-block for a derived class and some or all of its bases, the handlers shall be ordered most-derived to base class [OWASP2019-API7-b-3]
      • Do not leave 'catch' blocks empty [OWASP2019-API7-c-3]
    • OWASP API8 Injection [OWASP2019-API8]
      • Protect against command injection [OWASP2019-API8-a-1]
      • Avoid printing tainted data on the output console [OWASP2019-API8-b-1]
      • Protect against environment injection [OWASP2019-API8-c-1]
      • Protect against file name injection [OWASP2019-API8-d-1]
      • Exclude unsanitized user input from format strings [OWASP2019-API8-e-1]
      • Protect against SQL injection [OWASP2019-API8-f-1]
      • Disable resolving XML external entities (XXE) in libxerces-c [OWASP2019-API8-g-2]
      • Use care to ensure that LoadLibrary() will load the correct library [OWASP2019-API8-h-2]
      • Avoid passing dynamically created strings into exec [OWASP2019-API8-i-2]
      • Avoid passing user input into methods as parameters [OWASP2019-API8-j-2]
    • OWASP API9 Improper Assets Management [OWASP2019-API9]
      • Assert liberally to document internal assumptions and invariants [OWASP2019-API9-a-5]
      • When using enum, the values of each member should be explicitly declared [OWASP2019-API9-b-5]
      • All usage of assembler shall be documented [OWASP2019-API9-c-3]
      • Use of floating-point arithmetic shall be documented [OWASP2019-API9-d-3]
      • Objects or functions with external linkage shall be declared in a header file [OWASP2019-API9-e-4]
      • Document integer division [OWASP2019-API9-f-5]
      • All uses of the #pragma directive shall be documented and explained [OWASP2019-API9-g-3]
