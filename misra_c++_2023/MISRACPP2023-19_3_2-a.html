A macro parameter immediately following a # operator shall not immediately be followed by a ## operator [MISRACPP2023-19_3_2-a-2]

DESCRIPTION 

"The order of evaluation associated with multiple #, multiple ## or a mix
of # and ## preprocessor operators is unspecified. In particular, the result of
a # operator is a string literal and it is extremely unlikely that pasting this
to any other preprocessing token will result in a valid token."

This rule detects macro parameters immediately following a # operator that are
immediately followed by a ## operator.

See also: MISRA2004-19_13
BENEFITS 

The rule prevents unspecified behaviours.
EXAMPLE 

#define M( x, y )  #x ## y         /* Violation */
REPAIR 

#define M1( x )    #x              /* OK */
#define M( x, y )  M1(x) ## y      /* OK */
