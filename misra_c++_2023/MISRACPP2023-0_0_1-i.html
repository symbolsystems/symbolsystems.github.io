Where multiple handlers are provided in a single 'try-catch' statement or 'function-try-block', any ellipsis (catch-all) handler shall occur last [MISRACPP2023-0_0_1-i-2]

DESCRIPTION 

"If the catch-all handler is found before any other handler, that behaviour
will be performed. Any handlers after the catch-all are unreachable code
and can never be executed." [MISRA C++:2008, Rule 15-3-7]

The catch-all clause 'catch (...)' matches exceptions of any type. If present,
it has to be the last catch clause in a sequence of handlers.
If the 'catch (...)' clause is not the last exception handler, most compilers
report an error.

This rule detects when a catch-all handler is not the last handler in a handler
sequence used in a 'try-catch' statement or a 'function-try-block'.
SINCE 

v10.4
BENEFITS 

This rule helps you identify unreachable code and ensure that all handlers are
executed.
EXAMPLE 

In the following example, the 'catch (...)' clause is not the last
handler in the 'try-catch' statement.

void foo() {
    try {
    } catch (...) {                        // Violation
    } catch (int e) {
    }
}
REPAIR 

You can fix the code by moving the catch-all handler to the last position
in the sequence of handlers.

void foo() {
    try {
    } catch (int e) {
    } catch (...) {                        // OK
    }
}
