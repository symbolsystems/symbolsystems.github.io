Any member function named 'allocate' or 'deallocate' enclosed by namespace 'std' should not be used [MISRACPP2023-21_6_2-b-2]

DESCRIPTION 

Using 'allocate' and 'deallocate' functions requires ensuring that memory
resources are appropriately released.

This rule detects when the 'allocate' or 'deallocate' member function declared
in namespace 'std' (or in any namespace in the Standard Library) is called
or when its address is taken.
SINCE 

2023.2
BENEFITS 

This rule helps you avoid memory leaks.
EXAMPLE 

#include <memory>

void foo() {
    std::allocator<int> alloc;
    int* pi = alloc.allocate(1); // Violation
    alloc.deallocate(pi, 1);     // Violation
}
REPAIR 

Do not use the 'allocate' or 'deallocate' member functions.
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 21.6.2 Dynamic memory shall be managed automatically
