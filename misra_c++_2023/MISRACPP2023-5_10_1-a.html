Do not declare entities in the standard library namespaces: posix, std and stdN [MISRACPP2023-5_10_1-a-2]

DESCRIPTION 

"Top level namespaces with a name starting with 'std' and followed
by a non-empty sequence of digits are reserved for future standardization.
The behavior of a C++ program is undefined if it adds declarations or
definitions to such a namespace." [ISO/IEC 14882:2017]

This rule detects when any identifiers are defined within namespaces
with the names 'posix', 'std', and 'stdN', where 'N' is any number.
SINCE 

2023.2
EXCEPTIONS 

This rule does not apply to full and partial template specializations.
BENEFITS 

This rule helps you avoid confusion and unpredicted behavior.
EXAMPLE 

namespace posix
{
    int a;            // Violation
}
namespace std
{
    class A           // Violation
    {
        int m1;
    };
}

namespace std55
{
    void f();         // Violation
}
REPAIR 

#include <functional>

namespace myPosix
{
    int a;            // OK
}

namespace myStd
{
    class A           // OK
    {
        int m1;
    };
    void f();         // OK
}

namespace std
{
    template <> struct hash< myStd::A >                 // OK - Exception
    {
        size_t operator()( const myStd::A & x ) const;
    };
}
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   20.5.4.2.3 Namespaces for future standardization

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
