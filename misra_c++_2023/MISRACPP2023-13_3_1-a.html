Each overriding virtual function shall be declared with the override or final specifier [MISRACPP2023-13_3_1-a-2]

DESCRIPTION 

If a member function declaration or definition is followed by the 'override'
specifier, it means that this function is virtual and overrides a virtual
function from the base class.

This rule detects when a member virtual function from the derived class that
overrides a function from the base class does not include an explicit
'override' or 'final' specifier.

This rule helps you avoid issues that may result form overloading issues
instead of overriding issues.
SINCE 

v9.6
BENEFITS 

This rule helps you avoid issues that may occur if a function that should
be overridden is overloaded instead.
If determines whether or not overriding is desired and allows the compiler
to emit a clear error message when a mistake has been made.
EXAMPLE 

class B
{
public:
  virtual void v();
};

class D : public B
{
public:
  virtual void v();                    // Violation
};

class E : public D
{
public:
  void v();                            // Violation
};
REPAIR 

class B
{
public:
  virtual void v();
};

class D : public B
{
public:
  void v() override;                   // OK
};

class E : public D
{
public:
  void v() final;                      // OK
};
