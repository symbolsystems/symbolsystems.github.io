Destructor should not be called manually [MISRACPP2023-21_6_3-a-2]

DESCRIPTION 

"Don't try to play with object lifetime by calling the destructor manually.
After you call an object's destructor, its lifetime is over and it is undefined
and generally unsafe to do anything with it."

The rule reports violations on explicit calls of destructors.
EXAMPLE 

class A {
public:
    A();
    ~A();
};

void foo( ) {
    A obj;
    obj.~A( );        // Violation
}
REPAIR 

Don't call destructor explicitly.
