MISRACPP2023-25_5_2 
Rule 25.5.2 (Mandatory) The pointers returned by the C++ Standard Library functions localeconv, getenv, setlocale or strerror must only be used as if they have pointer to const-qualified type
RULES

The pointers returned by the Standard Library functions 'localeconv', 'getenv', 'setlocale' or, 'strerror' shall only be used as if they have pointer to const-qualified type [MISRACPP2023-25_5_2-a-1]
Strings pointed by members of the structure 'lconv' should not be modified [MISRACPP2023-25_5_2-b-1]
The address of the 'localeconv', 'getenv', 'setlocale' or, 'strerror' function should not be taken [MISRACPP2023-25_5_2-c-1]
