The 'NULL' identifier should not be used [MISRACPP2023-7_11_1-b-2]

DESCRIPTION 

"The macro NULL, defined in any of <clocale>, <cstddef>, <cstdio>, <cstdlib>,
<cstring>, <ctime>, or <cwchar>, is an implementation-defined C++ null pointer
constant in this International Standard" [ISO/IEC 14882:2011]

The 'NULL' macro usually expands to literal constant '0' and can be used
as a pointer or integral type, which can be confusing for developers.

This rule detects when the 'NULL' identifier is used in code.
SINCE 

2023.2
BENEFITS 

This rule helps you improve the readability and maintainability of code.
EXAMPLE 

#define NULL 0              // Violation
void f1()
{
    int one = NULL + 1;     // Violation
}
#undef NULL                 // Violation

#include <cstdlib>
void f2()
{
    int* ptr = NULL;        // Violation
}
REPAIR 

#define MY_NULL 0            // OK
void f1()
{
    int one = MY_NULL + 1;   // OK
}
#undef MY_NULL               // OK

#include <cstdlib>
void f2()
{
    int* ptr = nullptr;      // OK
}
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 7.11.1 nullptr shall be the only form of the null-pointer-constant

2. Programming languages -- C++ ISO/IEC 14882:2011
   C.3 C standard library, C.3.2.4 Macro NULL
