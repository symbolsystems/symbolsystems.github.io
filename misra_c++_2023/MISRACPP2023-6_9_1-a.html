The same type aliases shall be used in all declarations of the same entity [MISRACPP2023-6_9_1-a-2]

DESCRIPTION 

If in a re-declaration type aliases are different from those in previous
declarations, then it may not be clear whether the declarations refer
to the same entity.

This rule detects when different aliases are used in declarations
of the same entity in a translation unit.
SINCE 

2023.2
NOTES 

This rule checks only type aliases and ignores other differences
in declarations i.e. the 'const' and 'volatile' qualifiers used
for of function parameters.

extern void foo(const int i);
       void foo(      int i){}  // OK
BENEFITS 

This rule helps you improve the readability and maintainability of your code.
DRAWBACKS 

1. The rule does not detect when a member variable with the constexpr
   specifier has a different type in the declaration than in the definition.

typedef int INT;

struct S
{
    static constexpr INT i = 1;   // no Violation
};

constexpr int S::i;

2. The rule does not check types in template functions.

template<typename T> int ft1();
template<typename T> INT ft1()   // no Violation
{
    return 0;
}
EXAMPLE 

using INT = int;

INT i;
extern int i;                   // Violation

void f1(INT p1);
void f1(int p1){}               // Violation
REPAIR 

using INT = int;

INT i;
extern INT i;                   // OK

void f1(int p1);
void f1(int p1){}               // OK
REFERENCES 

1. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 6.9.1 The same type aliases shall be used in all declarations of the same entity
