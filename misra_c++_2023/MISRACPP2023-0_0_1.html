MISRACPP2023-0_0_1 
Rule 0.0.1 (Required) A function shall not contain unreachable statements
RULES

There shall be no unreachable code in "else" block [MISRACPP2023-0_0_1-a-2]
There shall be no unreachable code after 'return', 'break', 'continue', 'goto', 'throw' statements, and after calls to functions with the 'noreturn' attribute [MISRACPP2023-0_0_1-b-2]
There shall be no unreachable code in "if/else/while/for" block [MISRACPP2023-0_0_1-c-2]
There shall be no unreachable code in switch statement [MISRACPP2023-0_0_1-d-2]
There shall be no unreachable code in 'for' loop [MISRACPP2023-0_0_1-e-2]
There shall be no unreachable code after 'if' or 'switch' statement [MISRACPP2023-0_0_1-f-2]
There shall be no unreachable code after "if" or "switch" statement inside while/for/do...while loop [MISRACPP2023-0_0_1-g-2]
Where multiple handlers are provided in a single try-catch statement or function-try-block for a derived class and some or all of its bases, the handlers shall be ordered most-derived to base class [MISRACPP2023-0_0_1-h-2]
Where multiple handlers are provided in a single 'try-catch' statement or 'function-try-block', any ellipsis (catch-all) handler shall occur last [MISRACPP2023-0_0_1-i-2]
A 'try' statement of a 'try-block' should contain a potentially-throwing statement [MISRACPP2023-0_0_1-j-2]
