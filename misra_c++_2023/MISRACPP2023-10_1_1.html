MISRACPP2023-10_1_1 
Rule 10.1.1 (Advisory) The target type of a pointer or lvalue reference parameter should be const-qualified appropriately
RULES

A pointer parameter in a function prototype should be declared as pointer to const if the pointer is not used to modify the addressed object [MISRACPP2023-10_1_1-a-4]
Declare a type of parameter as typedef to pointer to const if the pointer is not used to modify the addressed object [MISRACPP2023-10_1_1-b-4]
Declare reference parameters as const references whenever possible [MISRACPP2023-10_1_1-c-4]
