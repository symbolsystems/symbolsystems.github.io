The 'exit()' function from the 'stdlib.h' or 'cstdlib' library shall not be used [MISRACPP2023-18_5_2-b-4]

DESCRIPTION 

"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<stdlib.h> shall not be used." [MISRA-C:2004]
"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<cstdlib> shall not be used. The use of these functions leads to
implementation-defined behaviour." [MISRA C++:2008]

This rule detects calls to the 'exit()' function from the 'stdlib.h'
and 'cstdlib' standard libraries.

See also: SECURITY-48_b, SECURITY-52, PB-75
SINCE 

v10.4
BENEFITS 

This rule helps you prevent undefined or implementation-defined behavior.
EXAMPLE 

#include  <stdlib.h>

void f1( int i ) {
    /*...*/
    exit( 0 );                 /* Violation */
}
REPAIR 

You can fix the code by removing the call to the 'exit()' function.
