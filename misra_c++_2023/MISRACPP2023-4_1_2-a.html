Do not include headers deprecated in the C++17 standard [MISRACPP2023-4_1_2-a-4]

DESCRIPTION 

"For compatibility with prior C++ International Standards, the C++ standard
library provides headers <ccomplex>, <cstdalign>, <cstdbool>, and <ctgmath>.
The use of these headers is deprecated." [ISO/IEC 14882:2017]

"The header <strstream> defines three types that associate stream buffers
with character array objects and assist reading and writing such objects."
 [ISO/IEC 14882:2017]

"The header <codecvt> provides code conversion facets for various character
encodings."  [ISO/IEC 14882:2017]

This rule detects when one of the standard library headers <ccomplex>,
<cstdalign>, <cstdbool>, <ctgmath>, <strstream>, <codecvt> is included.
SINCE 

2023.2
BENEFITS 

This rule helps you eliminate deprecated features.
EXAMPLE 

#include <ccomplex>  // Violation
#include <cstdalign> // Violation
#include <cstdbool>  // Violation
#include <ctgmath>   // Violation
#include <strstream> // Violation
#include <ccomplex>  // Violation
REPAIR 

Do not use deprecated headers.
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   Annex D Compatibility features

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 4.1.2 Deprecated features should not be used
