The 'assert' macro shall not be used with a constant expression [MISRACPP2023-22_3_1-a-2]

DESCRIPTION 

This rule reports when a constant expression is used as the condition
of the 'assert' macro.
SINCE 

2023.2
NOTES 

The rule checks the use of some commonly used expansions of the 'assert'
macro.
EXCEPTIONS 

The rule does not report on the following:
* assert(false)
* assert(false && "string literal")
* assert(!"string_literal")

Additionally, for C code only, the rule does not report on the following:
* assert(0)
BENEFITS 

This rule helps you identify the use of the 'assert' macro, which could be
checked during compilation rather than during runtime. For C11 and C++11,
'static_assert' can be used instead of 'assert' in such cases.
EXAMPLE 

#ifdef __cplusplus
const int C = 42;
#define FALSE false
#else
enum E {
    A = 40, B, C
};
#define FALSE 0
#endif

void test(int x)
{
    assert(C == 42);                  // Violation
	assert(sizeof(x) < sizeof(long)); // Violation
	assert(FALSE);                    // OK by exception
}
REPAIR 

#ifdef __cplusplus
const int C = 42;
#else
enum E {
    A = 40, B, C
};
#endif

void test(int x)
{
    static_assert(C == 42);           // OK
	assert(x > 0);                    // OK
}
