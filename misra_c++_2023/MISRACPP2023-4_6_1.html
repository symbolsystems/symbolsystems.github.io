MISRACPP2023-4_6_1 
Rule 4.6.1 (Required) Operations on a memory location shall be sequenced appropriately
RULES

The value of an expression shall be the same under any order of evaluation that the standard permits [MISRACPP2023-4_6_1-a-2]
Don't write code that depends on the order of evaluation of function arguments [MISRACPP2023-4_6_1-b-2]
Don't write code that depends on the order of evaluation of function designator and function arguments [MISRACPP2023-4_6_1-c-2]
Don't write code that depends on the order of evaluation of expression that involves a function call [MISRACPP2023-4_6_1-d-2]
Between sequence points an object shall have its stored value modified at most once by the evaluation of an expression [MISRACPP2023-4_6_1-e-2]
Do not use more than one volatile between two adjacent sequence points [MISRACPP2023-4_6_1-f-2]
Don't write code that depends on the order of evaluation of function calls [MISRACPP2023-4_6_1-g-2]
