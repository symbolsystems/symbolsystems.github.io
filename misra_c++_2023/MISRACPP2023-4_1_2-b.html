The 'uncaught_exception' function deprecated in the C++17 standard should not be used [MISRACPP2023-4_1_2-b-4]

DESCRIPTION 

The 'uncaught_exception()' function was deprecated in the C++17 standard.
The function std::uncaught_exceptions(), which returns the number of uncaught
exceptions in the current thread, should be used instead.

This rule detects when the 'uncaught_exception()' function is called or when its
address is taken.
SINCE 

2023.2
BENEFITS 

This rule helps you eliminate deprecated features.
EXAMPLE 

#include <exception>

void f1()
{
    if (std::uncaught_exception()) {       // Violation
        throw 0;
    }
}
REPAIR 

#include <exception>

void f1()
{
    if (std::uncaught_exceptions()) {      // OK
        throw 0;
    }
}
REFERENCES 

1. ISO/IEC 14882:2017 Programming languages - C++
   Annex D Compatibility features

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 4.1.2 Deprecated features should not be used
