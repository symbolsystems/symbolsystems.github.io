User defined suffixes of the user defined literal operators shall start with underscore followed by one or more letters [MISRACPP2023-5_10_1-d-2]

DESCRIPTION 

"Suffixes that do not begin with the underscore character are reserved
for operators provided by the standard library." [AUTOSAR A13-1-2]

The rule detects user-defined suffixes of the user-defined literal operators
that do not start with an underscore followed by a letter.
SINCE 

v10.4
BENEFITS 

This rule helps you improve code readability.
EXAMPLE 

long double operator"" meters(long double mtrs) // Violation
{
  // ...
  return mtrs;
}
REPAIR 

long double operator"" _m(long double mtrs)     // OK
{
  // ...
  return mtrs;
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A13-1-2. User defined suffixes of the user defined literal operators shall
   start with underscore followed by one or more letters

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 5.10.1 User-defined identifiers shall have an appropriate form
