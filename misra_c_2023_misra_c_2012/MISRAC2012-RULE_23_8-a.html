The default association shall be placed first or last in the association list [MISRAC2012-RULE_23_8-a-2]

DESCRIPTION 

"A default association shall only appear as either the first generic
association in the association list, or the last - this rule only applies
when a generic selection contains a default association."
[MISRA C:2012 Amendment 3]

This rule detects when a generic selection contains a default association
that is not first or last in the association list.
SINCE 

2022.2
BENEFITS 

"Having a default association appear as either the first or last in the
association list makes it easy to locate, clarifying the intent of the
selection structure." [MISRA C:2012 Amendment 3]

This rule helps you improve the readability of the code.
EXAMPLE 

// Violation
#define cbrt(X) _Generic((X),           \
                    long double: cbrtl, \
                    default: cbrt,      \
                    float: cbrtf        \
                )(X)
REPAIR 

To fix your code, move the default association to the first or last
position in the association list.

// OK
#define cbrt(X) _Generic((X),           \
                    default: cbrt,      \
                    long double: cbrtl, \
                    float: cbrtf        \
                )(X)
