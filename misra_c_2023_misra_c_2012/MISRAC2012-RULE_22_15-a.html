A thread synchronization object should not be destroyed by a thread [MISRAC2012-RULE_22_15-a-2]

DESCRIPTION 
This rule detects cases when a thread synchronization object or 
a thread-specific storage pointer is destroyed inside of a thread.

For a list of recognized thread-creating functions, see the rule BD-TRS-THRTHR.

This rule recognizes the following destruction functions:

<threads.h> (C Standard Library)
	void mtx_destroy( mtx_t *mutex );
	void cnd_destroy( cnd_t* cond );
	void tss_delete( tss_t tss_id );
	
<pthread.h> (POSIX)
	int pthread_mutex_destroy(pthread_mutex_t *mutex);
	int pthread_cond_destroy(pthread_cond_t *cond);
	int pthread_key_delete(pthread_key_t key);

<synhapi.h> (Windows)
    VOID DeleteCriticalSection (LPCRITICAL_SECTION lpCriticalSection);
    WINBOOL DeleteSynchronizationBarrier (LPSYNCHRONIZATION_BARRIER lpBarrier);
    BOOL TlsFree (DWORD dwTlsIndex);
	
If any of these functions is called (directly or indirectly) from a function which itself
is a thread entry point, the rule will report a violation.
SINCE 
2023.1
NOTES 
This rule does not report a violation when an object of the C++ Standard Library mutex type is destroyed.
However, the rule BD-TRS-THRINIT will report a violation when an object of the C++ Standard Library mutex type is constructed.
    
The "CloseHandle(HANDLE hObject)" function which closes a mutex or a semaphore also 
closes more objects (e.g. File, Console input or Event). It is difficult to recognize 
when the function is called to close a mutex or a semaphore. To avoid false positive 
violations, the CloseHandle function is not supported by this rule.
SECURITY RELEVANCE 
Destroying a thread synchronization object or a thread-specific storage pointer 
inside of a thread can cause undefined behavior, because another thread might 
still have access to the destroyed resource.
BENEFITS 
This rule helps you prevent undefined behavior when thread synchronization 
objects or thread-specific storage pointers are destroyed.
EXAMPLE 

#include<pthread.h>

pthread_t thread1, thread2;

pthread_mutex_t mutex;      
static pthread_key_t key;     

void *t1( void* ignore )
{
    pthread_key_delete(key); // VIOLATION
    return 0;
}

void *t2( void* ignore )
{
    pthread_mutex_destroy(&mutex); // VIOLATION
    return 0;
}

int main(void)
{
    pthread_mutex_init(&mutex, NULL);
    pthread_key_create(&key, NULL);
    
    pthread_create(&thread1, NULL, &t1, NULL);
    pthread_create(&thread2, NULL, &t2, NULL);
    return 0;
}
REPAIR 
To repair the code, destroy the thread synchronization object and thread-specific storage pointer
outside of the created thread.

#include<pthread.h>

pthread_t thread1, thread2;

pthread_mutex_t mutex;      
static pthread_key_t key;     

void *t1( void* ignore )
{             
    return 0;
}

void *t2( void* ignore )
{             
    return 0;
}

int main(void)
{
    pthread_mutex_init(&mutex, NULL); 
    pthread_key_create(&key, NULL);
    
    pthread_create(&thread1, NULL, &t1, NULL);
    pthread_create(&thread2, NULL, &t2, NULL);

    pthread_join(thread1,NULL);
    pthread_join(thread2,NULL);

    pthread_key_delete(key); // NO VIOLATION
    pthread_mutex_destroy(&mutex); // NO VIOLATION
    return 0;
}
