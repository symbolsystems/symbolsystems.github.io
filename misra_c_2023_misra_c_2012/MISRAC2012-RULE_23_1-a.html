A generic selection should only be expanded from a macro [MISRAC2012-RULE_23_1-a-4]

DESCRIPTION 

"A generic selection expression should only be expanded from a function-like
macro, with one of the macro's arguments expanded into the controlling expression
of the generic selection." [MISRA C:2012 Amendment 3]

This rule detects generic selections that are used outside of the macro
definitions.
SINCE 

2022.2
BENEFITS 

"Generic selections are useful to implement type queries or checks, or generic
functions. If the generic selection is applied directly rather than expanded
from a macro body, the type of the operand is already known locally and querying
it is not necessary." [MISRA C:2012 Amendment 3]
EXAMPLE 

int x;
int y = _Generic(x, int: 1, default : 0);                      // Violation
REPAIR 

#define M_IS_INT(X) _Generic((X), int : 1, default : 0)        // OK
int x;
int y =  M_IS_INT(x);
