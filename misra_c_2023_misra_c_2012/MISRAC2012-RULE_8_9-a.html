Objects shall be defined at block scope if they are only accessed from within a single function [MISRAC2012-RULE_8_9-a-4]

DESCRIPTION 

"Declaring an object at block scope reduces the possibility that the object
might be accessed inadvertently and makes clear the intention that it should
not be accessed elsewhere.
Within a function, whether objects are declared at the outermost or innermost
block is largely a matter of style." [MISRA C:2012 Rule 8.9 with Amendment 4]

This rule detects when a global variable with internal linkage is only used
inside a single function in a translation unit.
NOTES 

This rule does not apply to const variables of POD-type declared in header
files.
This rule does not detect variables that are not used at all.
BENEFITS 

The rule helps you improve code readability and maintainability.
DRAWBACKS 

This rule might report false positive violations when a const or constexpr
variable is used as a template argument.

template <int C> class AT { };

static constexpr int LOC1 = 10U;  // false positive

void f1()
{
    int x1 = LOC1;
}

void f2()
{
    AT<LOC1> var1;
}
EXAMPLE 

static int max = 10;        // Violation
void foo( ) {
    while(max >= 0) {
        max--;
    }
}
REPAIR 

void foo( ) {
    int max = 10;           // OK
    while(max >= 0) {
        max--;
    }
}
