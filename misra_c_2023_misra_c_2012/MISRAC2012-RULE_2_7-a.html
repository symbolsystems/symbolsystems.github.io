There should be no unused parameters in functions [MISRAC2012-RULE_2_7-a-4]

DESCRIPTION 

"Most functions will be specified as using each of their parameters. 
If a function parameter is unused, it is possible that the implementation
of the function does not match its specification. This rule highlights
such potential mismatches."

See also: OPT-03, OPT-31
SINCE 

v9.5
NOTES 

This rule is enabled for C only - to check C++ code please see rules OPT-03
or OPT-31 which better support unnamed parameters and virtual methods.
EXCEPTIONS 

The rule does not report violation on 'main' function.
BENEFITS 

The rule improves legibility and increases efficiency of code.
EXAMPLE 

int foo(int i, int k) {      /* Violation */
    i = 5 * i;
    return i;
}
REPAIR 

int foo(int i) {             /* OK */
    i = 5 * i;
    return i;
}
REFERENCES 

1. MISRA C:2012 Guidelines for the use of the C language in critical systems
   Section 8: Rules, Rule 2.7
