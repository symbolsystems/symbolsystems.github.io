MISRAC2012-RULE_10_5 
Rule 10.5 (Advisory) The value of an expression should not be cast to an inappropriate essential type
RULES

The cast operation to essentially enumeration type is not allowed [MISRAC2012-RULE_10_5-a-4]
Do not cast from or to essentially Boolean type [MISRAC2012-RULE_10_5-b-4]
Do not use casts between essentially character types and essentially floating types [MISRAC2012-RULE_10_5-c-4]
