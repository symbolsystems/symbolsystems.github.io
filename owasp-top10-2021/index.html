Enabled Rules

  • OWASP Top 10 Most Critical Web Application Security Risks (2021) [OWASP2021]
    • OWASP A1 Broken Access Control [OWASP2021-A1]
      • Protect against file name injection [OWASP2021-A1-a-1]
      • Observe correct revocation order while relinquishing privileges [OWASP2021-A1-b-3]
      • Ensure that privilege relinquishment is successful [OWASP2021-A1-c-3]
    • OWASP A2 Cryptographic Failures [OWASP2021-A2]
      • Properly seed pseudorandom number generators [OWASP2021-A2-a-2]
    • OWASP A3 Injection [OWASP2021-A3]
      • Avoid passing unvalidated binary data to log methods [OWASP2021-A3-a-2]
      • Protect against command injection [OWASP2021-A3-b-1]
      • Avoid printing tainted data on the output console [OWASP2021-A3-c-1]
      • Protect against environment injection [OWASP2021-A3-d-1]
      • Exclude unsanitized user input from format strings [OWASP2021-A3-e-1]
      • Protect against SQL injection [OWASP2021-A3-f-1]
    • OWASP A4 Insecure Design [OWASP2021-A4]
      • Avoid passing sensitive data to functions that write to log files [OWASP2021-A4-a-2]
    • OWASP A5 Security Misconfiguration [OWASP2021-A5]
      • Where multiple handlers are provided in a single try-catch statement or function-try-block for a derived class and some or all of its bases, the handlers shall be ordered most-derived to base class [OWASP2021-A5-a-3]
      • Do not leave 'catch' blocks empty [OWASP2021-A5-b-3]
      • Properly use errno value [OWASP2021-A5-c-2]
      • Disable resolving XML external entities (XXE) in libxerces-c [OWASP2021-A5-d-2]
    • OWASP A7 Identification and Authentication Failures [OWASP2021-A7]
      • Do not use weak encryption functions [OWASP2021-A7-a-2]
    • OWASP A8 Software and Data Integrity Failures [OWASP2021-A8]
      • Use care to ensure that LoadLibrary() will load the correct library [OWASP2021-A8-a-2]
    • OWASP A9 Security Logging and Monitoring Failures [OWASP2021-A9]
      • All exceptions should be rethrown or logged with standard logger [OWASP2021-A9-a-2]
