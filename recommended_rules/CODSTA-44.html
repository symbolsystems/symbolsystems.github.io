Local variables should not use the same names as member variables [CODSTA-44-1]

DESCRIPTION 

If a local variable uses the same name as member variable, then it hides
this variable, what may lead to developer confusion.

The rule reports a violation if a local variable declared in a member
function uses the same name as member variable of current class
or as public/protected member variable of base class.

See also: CODSTA-45, CODSTA-CPP-83
BENEFITS 

Hiding names of identifiers can lead to errors and confusions.
EXAMPLE 

class Base {
public:
    int basem;
};

class Derived : public Base {
public:
    void foo1(){
        int basem;           // Violation
    }
    void foo2();

private:
    int member;
};

void Derived::foo2(){ 
    int member;              // Violation
    member = 10;             // possible mistake
}
REPAIR 

class Base {
public:
    int basem;
};

class Derived : public Base {
public:
    void foo1(){
        int lbasem;          // OK - the name of local variable was changed
    }
    void foo2();

private:
    int member_D;
};

void Derived::foo2(){ 
    int member;              // OK - the name of member variable was changed
    member = 10;
}
