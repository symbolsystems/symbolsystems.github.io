Enabled Rules

  • Flow Analysis [BD]
    • Incorrect API usage [BD-API]
      • Do not pass negative values to functions expecting non-negative arguments [BD-API-NEGPARAM-1]
    • Possible Bugs [BD-PB]
      • Avoid accessing arrays out of bounds [BD-PB-ARRAY-2]
      • Avoid conditions that always evaluate to the same value [BD-PB-CC-2]
      • Do not check for null after dereferencing [BD-PB-DEREF-2]
      • Always catch exceptions [BD-PB-EXCEPT-1]
      • Avoid signed integer overflows [BD-PB-INTUB-1]
      • Avoid use before initialization [BD-PB-NOTINIT-1]
      • Atomic objects shall be properly initialized before being accessed [BD-PB-NOTINITAT-1]
      • Avoid null pointer dereferencing [BD-PB-NP-1]
      • Avoid buffer overflow due to defining incorrect format limits [BD-PB-OVERFFMT-1]
      • Avoid overflow due to reading a not zero terminated string [BD-PB-OVERFNZT-1]
      • Avoid overflow when reading from a buffer [BD-PB-OVERFRD-1]
      • Avoid overflow when writing to a buffer [BD-PB-OVERFWR-1]
      • Suspicious setting of stream flags [BD-PB-STREAMFLAGS-3]
      • Restore stream format [BD-PB-STREAMFMT-3]
      • Avoid division by zero [BD-PB-ZERO-1]
    • Resources [BD-RES]
      • Properly deallocate dynamically allocated resources [BD-RES-BADDEALLOC-1]
      • Do not use resources that have been freed [BD-RES-FREE-1]
      • Do not free resources using invalid pointers [BD-RES-INVFREE-1]
      • Ensure resources are freed [BD-RES-LEAKS-1]
    • Threads & Synchronization [BD-TRS]
      • Avoid double locking [BD-TRS-DLOCK-1]
      • Avoid race conditions when using fork and file descriptors [BD-TRS-FORKFILE-1]
      • Do not abandon unreleased locks [BD-TRS-LOCK-1]
      • Do not acquire locks in different order [BD-TRS-ORDER-1]
      • Avoid race conditions while checking for the existence of a symbolic link [BD-TRS-SYMLINK-1]
      • The Standard Library function mtx_timedlock() shall not be invoked on a mutex that does not support timeout [BD-TRS-TIMEDLOCK-2]
      • Do not use blocking functions while holding a lock [BD-TRS-TSHL-1]
  • Code Duplication Detection [CDD]
    • Avoid function duplication [CDD-DUPM-2]
  • Coding Conventions [CODSTA]
    • Coding Conventions for C++ [CODSTA-CPP]
      • Constructors allowing for conversion should be made explicit [CODSTA-CPP-04-1]
    • Pointer should not be compared with NULL using relational operators <, >, >=, <= [CODSTA-147-3]
    • Do not use string literals as operands of equality or relational operators [CODSTA-148-3]
    • Local variables should not use the same names as member variables [CODSTA-44-1]
    • Avoid infinite loops [CODSTA-82-3]
  • Exceptions [EXCEPT]
    • Throw by value, catch by reference [EXCEPT-02-1]
    • Do not throw from within destructor [EXCEPT-03-1]
  • Initialization [INIT]
    • All member variables should be initialized in constructor [INIT-06-1]
  • MISRA C 2004 [MISRA2004]
    • Floating-point expressions shall not be tested for equality or inequality [MISRA2004-13_3-3]
    • All exit paths from a function with non-void return type shall have an explicit return statement with an expression [MISRA2004-16_8-3]
    • The address of an object with automatic storage shall not be returned from a function [MISRA2004-17_6_a-3]
  • Memory and Resource Management [MRM]
    • Do not invoke malloc/realloc for objects having constructors [MRM-08-1]
    • Never provide brackets ([]) for delete when deallocating non-arrays [MRM-35-3]
    • Always provide empty brackets ([]) for delete when deallocating arrays [MRM-36-3]
    • Declare a copy assignment operator for classes with dynamically allocated memory [MRM-37-1]
    • Declare a copy constructor for classes with dynamically allocated memory [MRM-38-1]
    • Do not use 'delete' on pointers to a void type [MRM-51-3]
  • Object Oriented [OOP]
    • Class cannot inherit other class more than once unless it is virtual inheritance [OOP-03-1]
    • Avoid calling virtual functions from constructors [OOP-16-1]
    • If a class has virtual functions it shall have a virtual destructor [OOP-23-2]
  • Optimization [OPT]
    • Pass objects by reference instead of by value [OPT-14-3]
  • Possible Bugs [PB]
    • Do not call delete on non-pointers [PB-13-1]
    • Properly terminate character strings [PB-21-1]
    • Do not cast from or to incomplete class at the point of casting [PB-54-2]
    • Do not delete objects with incomplete class at the point of deletion [PB-55-2]
    • Suspicious argument to malloc [PB-60-3]
    • Pointer arithmetic performed on freshly allocated memory [PB-61-3]
    • Boolean condition always evaluates to the same value due to enumeration with only zero or only non-zero constants [PB-68-2]
