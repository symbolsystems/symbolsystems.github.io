Pointer arithmetic performed on freshly allocated memory [PB-61-3]

DESCRIPTION 

Rule reports a violation if you do pointer arithmetic directly on result of
'malloc', 'calloc', 'realloc' or 'new'. For example:
    malloc(s) + x
    T *p = new T - 10;
    return calloc(n, s) - ptr;

When you do pointer arithmetic on freshly allocated memory, you may:
- leak memory because you don't have pointer to the beginning of the
  allocated memory region or you don't have any pointer at all
- cause invalid memory access when you point before allocate memory region
SINCE 

v9.5
BENEFITS 

Detects possible memory leaks and invalid memory accesses.
EXAMPLE 

#include <stdlib.h>
#include <stddef.h>

ptrdiff_t foo(int x)
{
    /* Addition */
    int *p = x + (int*) malloc(10); /* Violation */

    /* Subtraction */
    (int*)malloc(20) - 2; /* Violation */

    /* Pointer difference */
    /* Following code is also invalid because pointers
     do not point to the same array */
    return (int*) malloc(sizeof(int)) - p; /* Violation */
}
void bar(size_t nmemb, size_t size)
{
    void *p = (int*)calloc(nmemb, size) + 1; /* Violation */
    (int*)realloc(p, nmemb * size + 1) - 1; /* Violation */
}
REPAIR 

/* Example of code which does not cause a violation */

#include <stdlib.h>
void bar(int x)
{
    int *p = (int*) malloc(10); /* OK */
    int *q = x + p; /* OK */
}
