The 'abort()' function from the 'stdlib.h' or 'cstdlib' library shall not be used [MISRA2004-20_11-3]

DESCRIPTION 

"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<stdlib.h> shall not be used." [MISRA-C:2004]
"The library functions 'abort', 'exit', 'getenv' and 'system' from library
<cstdlib> shall not be used. The use of these functions leads to
implementation-defined behaviour." [MISRA C++:2008]

This rule detects calls to the 'abort()' function from the 'stdlib.h'
and 'cstdlib' standard libraries.

See also: SECURITY-48_b, SECURITY-52, PB-75_b
SINCE 

v10.4
BENEFITS 

This rule helps you prevent undefined or implementation-defined behavior.
EXAMPLE 

#include  <stdlib.h>

void f1( int i ) {
    /*...*/
    abort( );                 /* Violation */
}
REPAIR 

You can fix the code by removing the call to the 'abort()' function.
