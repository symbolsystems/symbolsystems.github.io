Lambda expressions should not be defined inside another lambda expression [CODSTA-MCPP-44-3]

DESCRIPTION 

"Lambda expressions should not be defined inside another lambda
expression." [AUTOSAR A5-1-8]

This rule reports a violation if a lambda expression is nested inside another 
lambda expressions.
SINCE 

v10.4
BENEFITS 

This rule improves code readability and consistency with the C++ standard.
EXAMPLE 

#include <cstdint>

void example()
{
  std::int16_t x = 0;
  auto f1 = [&x]() {

    auto f2 = []() {};      // Violation
    f2();

    auto f4 = []() {};      // Violation
    f4();
  };

  f1();
}
REPAIR 

#include <cstdint>

void example()
{
  std::int16_t x = 0;
  auto f1 = [&x]() {};

  auto f2 = []() {};      // OK

  auto f4 = []() {};      // OK

  f1();
}
