Include a parameter list in every lambda expression [CODSTA-MCPP-46-3]

DESCRIPTION 

"The lambda-declarator is optional in a lambda expression and results in
a closure that can be called without any parameters. To avoid any visual
ambiguity with other C++ constructs, it is recommended to explicitly
include (), even though it is not strictly required." [AUTOSAR A5-1-3]

This rule detects when a lambda expression fails to include a (possibly empty)
parameter list.
SINCE 

10.4
BENEFITS 

This rule helps you avoid visual ambiguity by differentiating lambda
expressions from other C++ constructs.
EXAMPLE 

#include <cstdint>
void Fn()
{
    std::int32_t i = 0;
    std::int32_t j = 0;
    auto lambda1 = [&i, &j] {  // Violation
        ++i;
        ++j;
    };
}
REPAIR 

You can fix the code by including a parameter list () in the lambda expression.

#include <cstdint>
void Fn()
{
    std::int32_t i = 0;
    std::int32_t j = 0;
    auto lambda1 = [&i, &j]() {  // OK
        ++i;
        ++j;
    };
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A5-1-3. Parameter list (possibly empty) shall be included in every lambda
   expression

2. High Integrity C++ Coding Standard
   5.1.5. Include a (possibly empty) parameter list in every lambda expression
