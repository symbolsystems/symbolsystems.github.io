Do not introduce virtual functions in a final class [CODSTA-MCPP-52-3]

DESCRIPTION 

"Declaring a class as final explicitly specifies that the class cannot be
inherited. Declaring a virtual function inside a class specifies that
the function can be overriden in the inherited class, which is inconsistent"
[AUTOSAR A10-3-3]

"Declaring a class as final explicitly documents it as a leaf class because it
cannot be used as a base class. Introducing a virtual function in such a class
is, therefore, redundant because the function can never be overridden in a
derived class." [HIC++ v4.0, Rule 9.1.5]

The rule detects when a virtual function is declared in a final class.
SINCE 

2020.2
EXCEPTIONS 

The rule does not trigger if the virtual function declared in a final class
overrides a function from the base class and is declared as 'final'.
BENEFITS 

This rule helps you improve code readability and increase code consistency.
EXAMPLE 

class Base {
  public:
    virtual void foo();                   // OK
};

class Derived final : public Base {
  public:
    void foo() override;                  // Violation
    virtual void bar();                   // Violation
};
REPAIR 

class Base {
  public:
    virtual void foo();                   // OK
};

class Derived final : public Base {
  public:
    void foo() final;                     // OK
    void bar();                           // OK
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A10-3-3. Virtual functions shall not be introduced in a final class

2. High Integrity C++ Coding Standard
