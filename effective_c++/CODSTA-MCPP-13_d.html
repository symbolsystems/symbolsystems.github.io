The 'std::move' function should be used to forward rvalue references [CODSTA-MCPP-13_d-3]

DESCRIPTION 

"The 'std::move' function unconditionally casts an rvalue reference to rvalue,
while the 'std::forward' function does the same if and only if the argument was
initialized with an rvalue.(...)
'std::move' should be used for forwarding rvalue references to other functions,
as rvalue reference always bounds to rvalue" [AUTOSAR A18-9-2]

This rule detects when an rvalue reference parameter that is not a forwarding
reference is passed as an argument to a function other than 'std::move'.
SINCE 

2023.2
NOTES 

Forwarding references are checked by the CODSTA-MCPP-13_b rule.
BENEFITS 

The rule helps you improve code consistency and clarity.
EXAMPLE 

class C { };

void foo(C& c){}
void foo(C&& c){}

void f1(C&& arg)
{
    foo(arg);                   // Violation
}
REPAIR 

To fix the code, use the 'std::move' function to pass the rvalue reference.

#include <utility>
class C { };

void foo(C& c){}
void foo(C&& c){}

void f1(C&& arg)
{
    foo(std::move(arg));       // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A18-9-2. Forwarding values to other functions shall be done via:
   (1) std::move if the value is an rvalue reference,
   (2) std::forward if the value is forwarding reference.

2. High Integrity C++ Coding Standard
   17.3.2. Use std::forward to forward universal references
