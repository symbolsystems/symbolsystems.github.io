Declare copy constructor and copy assignment operators with the 'delete' specifier instead of using a base class with private methods to prevent copying of class [CODSTA-MCPP-06_b-3]

DESCRIPTION 

One of the ways to make a class uncopyable is to make it a derived class
of another base class that has a privately defined copy constructor and copy
assignment, with no implementation. This moves subtle run-time bugs into
compile- or link- time. In C++11, we can now explicitly declare a class with
deleted functions with the 'delete' specifier, instead of using a base class
with private methods.

This rule detects when a class inherits from a base class that includes
only private undefined copy constructor and copy assignment operators.
SINCE 

v9.6
BENEFITS 

The rule improves the readability of code and makes your code clearer and safer.
EXAMPLE 

class NonCopyable
{
private:
    NonCopyable(const NonCopyable&);
    NonCopyable& operator=(const NonCopyable&);
};

class Foo : public NonCopyable                   // Violation
{
    // copying is disabled
};
REPAIR 

class Foo                                        // OK
{
public:
    Foo(const Foo&) = delete;
    Foo& operator=(const Foo&) = delete;
    // copying is disabled
};
