Prefer to use cbegin(), crbegin, cend(), crend() functions [CODSTA-MCPP-10_b-3]

DESCRIPTION 

In Modern C++ (from C++11) the use of const_iterators doesn't have
limitations that exist in the previous versions of the C standard. You can use
them always when an object pointed by an iterator does not need to be modified.
If an iterator type is deduced based on its initializer, then you can change
it replacing standard library functions begin(), end(), rbegin(), rend()
by their const versions cbegin(), cend(), crbegin(), crend().

This rule reports a violation if a local variable with the deduced
iterator type points to an object that is not modified in the function
and its initializer contains the call to one of the following functions:
begin(), end(), rbegin(), rend().
SINCE 

v10.3
NOTES 

This rule detects only the container member functions begin(), end(),
rbegin(), rend() from the standard library.
BENEFITS 

Using of const_iterators can prevent exploits that could lead to modifying data.
Dereferencing const_iterators produces a const object, which results in better
overload resolution and helps you avoid security concerns.
EXAMPLE 

#include <vector>
#include <algorithm>

void foo()
{
    std::vector<int> values;
    auto it = find(values.begin(), values.end(), 90);         // Violation
}
REPAIR 

#include <vector>
#include <algorithm>

void foo()
{
    std::vector<int> values;
    auto it = find(values.cbegin(), values.cend(), 90);       // OK
}

Scott Meyers, "Effective Modern C++, 42 specific ways to improve your use of
C++11 and C++14", Item 13
