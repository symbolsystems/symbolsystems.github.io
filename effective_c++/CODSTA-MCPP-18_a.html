Avoid unnecessary default capture modes in lambda expressions [CODSTA-MCPP-18_a-3]

DESCRIPTION 

A lambda capture can include one of the following default capture modes:
- the by-reference[&] mode captures by reference the current object (if it
  exists) and all automatic variables in the body of a lambda
- the by-value [=] mode captures by value the current object (if it exists)
  and all automatic variables in the body of a lambda

Captures apply only to non-static local variables (including parameters)
that are visible in the scope where the lambda is created.

This rule detects when a lambda capture contains only a capture default,
but the lambda expression does not have any automatic variables.
SINCE 

v10.3
BENEFITS 

This rule improves readability of code and helps you avoid confusions.
EXAMPLE 

int g_i;
void func(void)
{
    [=]{ return 1;};        /* Violation - no captured variables */
    [&]{ return g_i;};      /* Violation - global variable is not captured */
}
REPAIR 

int g_i;
void func(void)
{
    []{ return 1;};        /* OK */
    []{ return g_i;};      /* OK */
}
