Do not use throw exception specifications [AUTOSAR-A15_4_1-a-2]

DESCRIPTION 

Using throw( type-id-listopt ) to specify an exception specification is
deprecated in the 2011 C++ Language Standard and may be removed in a future
version. The new syntax is to use noexcept or noexcept( expr ).

The exception specification was confusing and gave the mistaken impression that
it guaranteed certain kinds of behavior. This is why it was replaced by the
noexcept specification.
SINCE 

v10.4
BENEFITS 

You should keep your code up to date with the version of your programming
language standard. Doing so ensures the longevity of your application, as well
as addresses defects and vulnerabilities that may emerge as deployment
environments evolve.
EXAMPLE 

#include <cstdint>

void f1() throw();        // Violation
void f2() throw(int32_t); // Violation
REPAIR 

void f1() noexcept;        // OK
void f2() noexcept(false); // OK
