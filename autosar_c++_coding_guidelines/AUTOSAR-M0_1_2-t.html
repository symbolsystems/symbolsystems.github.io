Boolean operations whose results are invariant shall not be permitted [AUTOSAR-M0_1_2-t-2]

DESCRIPTION 

The rule reports violation if a result of boolean operation with variable 
of short type is always true or always false.
BENEFITS 

"If a Boolean operator yields a result that can be proven to be always "true"
or always "false", it is highly likely that there is a programming error."
EXAMPLE 

void myFunc()
{
    unsigned short a;
    signed short b;

    if (a < 65536)    // Violation
        a++;
    if (b <= 32767)   // Violation
        b++;
}
REPAIR 

void myFunc()
{
    unsigned short a;
    signed short b;

    if (a < 65535)    // OK
        a++;
    if (b <= 32766)   // OK
        b++;
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M0-1-2. A project shall not contain infeasible paths

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 0, Rule 0-1-2

3. MISRA-C:2004 Guidelines for the use of the C language in critical systems
   Chapter 6, Section 13
