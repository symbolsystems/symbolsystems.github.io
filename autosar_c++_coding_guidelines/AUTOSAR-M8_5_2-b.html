Arrays shall not be partially initialized [AUTOSAR-M8_5_2-b-2]

DESCRIPTION 

"If any element of an array object or subobject is explicitly initialized,
then the entire object or subobject shall be explicitly initialized."
[MISRA C:2012 Rule 9.3]

This rule detects when an aggregate initializer for an array object
is partially initialized.

See also: MISRA2004-9_2, MISRA2004-9_2_c
EXCEPTIONS 

"An initializer of the form { 0 } may be used to explicitly initialize all
elements of an array object or subobject." [MISRA C:2012 Rule 9.3]

"An array whose initializer consists only of designated initializers may be
used, for example to perform a sparse initialization." [MISRA C:2012 Rule 9.3]

"An array initialized using a string literal does not need an initializer
for every element." [MISRA C:2012 Rule 9.3]
BENEFITS 

"Providing an explicit initialization for each element of an array makes
it clear that every element has been considered." [MISRA C:2012 Rule 9.3]
EXAMPLE 

int arr1[3] = { 1, 2 };                       /* Violation */
int arr2[3][2] = { {1, 2}, {5, 6} };          /* Violation */
int arr3[3][2] = { {1}, {3, 4}, {5, 6} };     /* Violation */
REPAIR 

int arr1[3] = { 1, 2, 3 };                    /* OK */
int arr2[3][2] = { {1, 2}, {3, 4}, {5, 6} };  /* OK */
