AUTOSAR-A1_1_1 
AUTOSAR A1-1-1 All code shall conform to ISO/IEC 14882:2014 - Programming Language C++ and shall not use deprecated features
RULES

The 'register' storage class specifier shall not be used [AUTOSAR-A1_1_1-a-2]
A copy assignment operator should be declared when a copy constructor is declared (and vice versa) [AUTOSAR-A1_1_1-b-2]
Both copy constructor and copy assignment operator should be declared for classes with a nontrivial destructor [AUTOSAR-A1_1_1-c-2]
The C library shall not be used [AUTOSAR-A1_1_1-d-2]
Prefer lambdas over std::bind, std::bind1st and std::bind2nd [AUTOSAR-A1_1_1-e-2]
The 'binder1st' and 'binder2nd' identifiers should not be used [AUTOSAR-A1_1_1-f-2]
Prefer to use std::unique_ptr instead of std::auto_ptr [AUTOSAR-A1_1_1-g-2]
The 'random_shuffle' identifier should not be used [AUTOSAR-A1_1_1-h-2]
Do not use the increment operator (++) on an operand of type 'bool' [AUTOSAR-A1_1_1-i-2]
The 'set_unexpected' identifier should not be used [AUTOSAR-A1_1_1-j-2]
Do not use throw exception specifications [AUTOSAR-A1_1_1-k-2]
