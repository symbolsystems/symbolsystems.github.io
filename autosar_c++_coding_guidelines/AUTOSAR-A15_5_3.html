AUTOSAR A15-5-3 The std::terminate() function shall not be called implicitly
RULES

The execution of a function registered with 'std::atexit()' or 'std::at_quick_exit()' should not exit via an exception [AUTOSAR-A15_5_3-a-2]
Never allow an exception to be thrown from a destructor, deallocation, and swap [AUTOSAR-A15_5_3-b-2]
Do not throw from within destructor [AUTOSAR-A15_5_3-c-2]
There should be at least one exception handler to catch all otherwise unhandled exceptions [AUTOSAR-A15_5_3-d-2]
An empty throw (throw;) shall only be used in the compound-statement of a catch handler [AUTOSAR-A15_5_3-e-2]
Exceptions shall be raised only after start-up and before termination of the program [AUTOSAR-A15_5_3-f-2]
Each exception explicitly thrown in the code shall have a handler of a compatible type in all call paths that could lead to that point [AUTOSAR-A15_5_3-g-2]
Where a function's declaration includes an exception-specification, the function shall only be capable of throwing exceptions of the indicated type(s) [AUTOSAR-A15_5_3-h-2]
Function called in global or namespace scope shall not throw unhandled exceptions [AUTOSAR-A15_5_3-i-2]
Always catch exceptions [AUTOSAR-A15_5_3-j-2]
Properly define exit handlers [AUTOSAR-A15_5_3-k-2]
