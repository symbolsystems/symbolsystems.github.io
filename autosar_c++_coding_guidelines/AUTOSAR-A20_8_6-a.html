Prefer 'std::make_shared' to the direct use of new [AUTOSAR-A20_8_6-a-2]

DESCRIPTION 

Compared to the direct use of the 'new' operator, the std::make_shared function
is more efficient and exception safe, as well as increases the speed of code
execution, potentially reducing the total memory footprint for a program.

This rule detects when a 'new' operator is used when a smart pointer
'std::shared_ptr' is created.
SINCE 

v10.3
EXCEPTIONS 

There are several cases when std::make_shared cannot or should not be used:
  1. When a custom deleter needs to be used. For example:

       void logAndDelete(int* name);
       std::shared_ptr<int> custShPtr(new int, logAndDelete);          // OK

  2. When braced initializers need to be passed. For example:

       std::shared_ptr<int> s2(new int[5] {1,2,3,4,5});                // OK
BENEFITS 

The std::make_shared function eliminates source code duplication, improves
exception safety, and generates code that is more concise and faster.
EXAMPLE 

#include <memory>

std::shared_ptr<int> s(new int);                   // Violation

void foo() {
  std::shared_ptr<int> sp;
  sp.reset(new int);                               // Violation
  sp.reset();
}
REPAIR 

#include <memory>

std::shared_ptr<int> s(std::make_shared<int>());   // OK

void foo() {
  std::shared_ptr<int> sp;
  sp = std::make_shared<int>();                    // OK
}

   std::shared_ptr

2. High Integrity C++ Coding Standard
   17.4.2. Use API calls that construct objects in place
