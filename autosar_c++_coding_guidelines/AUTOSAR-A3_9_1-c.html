A typedef should be used in place of the plain char type [AUTOSAR-A3_9_1-c-2]

DESCRIPTION 

This rule detects when the plain 'char' (not explicitly declared as
'signed' or 'unsigned') type is used in the code.

The signedness of the plain 'char' type is implementation-defined.
The 'char' type declared explicitly as 'signed' or 'unsigned' should
be used for numeric data and plain 'char' should be used for character data.

Using defined typedefs for 'char' types allows the user to determine exactly
what the type's meaning is.
SINCE 

2023.1
EXCEPTIONS 

The rule allows:
- the use of plain 'char' as a parameter type of the 'main' function
- to declare typedefs for the 'plain' char types
BENEFITS 

This rule helps you improve readability of the code and prevent confusion.
EXAMPLE 

static char a;                      // Violation
REPAIR 

typedef char char_t;                // OK
static char_t a;                    // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A3-9-1. Fixed width integer types from <cstdint>, indicating the size and
   signedness, shall be used in place of the basic numerical types
