Literal suffixes shall use uppercase rather than lowercase letters [AUTOSAR-M2_13_4-a-2]

DESCRIPTION 

The rule reports a violation if a literal suffix use lowercase letter.
Using upper case letters makes the recognition of literal suffixes
easier, e.g. removes the potential ambiguity between '1' (digit 1) and 'l'
(letter el) for declaring literals.

See also: CODSTA-50
SINCE 

v7.1
BENEFITS 

The rule improves readability of code.
EXAMPLE 

void foo(long param = 64l)    // Violation
{
    unsigned int ui = 10000u; // Violation
    float f = 0.1f;           // Violation
    long hex = 0xfffffl;      // Violation
}
REPAIR 

void foo(long param = 64L)    // OK
{
    unsigned int ui = 10000U; // OK
    float f = 0.1F;           // OK
    long hex = 0xfffffL;      // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M2-13-4. Literal suffixes shall be upper case

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 2, Rule 2-13-4
