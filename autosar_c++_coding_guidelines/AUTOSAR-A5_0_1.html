AUTOSAR A5-0-1 The value of an expression shall be the same under any order of evaluation that the standard permits
RULES

Don't write code that depends on the order of evaluation of function calls [AUTOSAR-A5_0_1-a-2]
Do not use more than one volatile between two adjacent sequence points [AUTOSAR-A5_0_1-b-2]
Between sequence points an object shall have its stored value modified at most once by the evaluation of an expression [AUTOSAR-A5_0_1-c-2]
Don't write code that depends on the order of evaluation of expression that involves a function call [AUTOSAR-A5_0_1-d-2]
Don't write code that depends on the order of evaluation of function designator and function arguments [AUTOSAR-A5_0_1-e-2]
Don't write code that depends on the order of evaluation of function arguments [AUTOSAR-A5_0_1-f-2]
The value of an expression shall be the same under any order of evaluation that the standard permits [AUTOSAR-A5_0_1-g-2]
