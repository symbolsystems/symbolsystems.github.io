All usage of assembler shall be documented [AUTOSAR-M7_4_1-a-2]

DESCRIPTION 

"All usage of assembler shall be documented. Assembly language code
is implementation-defined and therefore is not portable."
SINCE 

v7.2
NOTES 

The rule assumes that the usage of assembler is documented if there is
a comment in the same line as 'asm' instruction or in line directly
before 'asm' instruction. For macros containing assembler, the violation
is reported on the use, not on the definition.
BENEFITS 

The rule improves readability of code.
EXAMPLE 

#define ASM_MAC asm ( "NOP" )
/* OK - reported on the macro use */

void fn1 (int i)
{
    asm ( "NOP" ); 
    /* Violation - line above */
    
    /*
     * 
     *  comment
     *  
     */
  
    asm ( "NOP" ); 
    /* Violation - line above */
  
    ASM_MAC;
    /* Violation - line above */
}
REPAIR 

#define ASM_MAC asm ( "NOP" )

void fn1 (int i)
{
    asm ( "NOP" );   /* comment */
    /* OK - line above */
    
    /*
     * 
     *  comment
     *  
     */
    asm ( "NOP" ); 
    /* OK - line above */
  
    // comment
    ASM_MAC;
    /* OK - line above */
}
