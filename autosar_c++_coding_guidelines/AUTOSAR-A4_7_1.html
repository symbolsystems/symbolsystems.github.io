AUTOSAR A4-7-1 An integer expression shall not lead to data loss
RULES

Avoid implicit conversions from signed to unsigned type [AUTOSAR-A4_7_1-a-2]
Implicit conversions from integral constant to floating type which may result in a loss of information shall not be used [AUTOSAR-A4_7_1-b-2]
Avoid conversions of constant values to a narrower type [AUTOSAR-A4_7_1-c-2]
Implicit conversions from wider to narrower integral type which may result in a loss of information shall not be used [AUTOSAR-A4_7_1-d-2]
Avoid implicit conversions from wider to narrower floating type [AUTOSAR-A4_7_1-e-2]
Avoid implicit conversions from floating to integral type [AUTOSAR-A4_7_1-f-2]
Implicit conversions from integral to floating type which may result in a loss of information shall not be used [AUTOSAR-A4_7_1-g-2]
Avoid wraparounds when performing arithmetic integer operations [AUTOSAR-A4_7_1-h-2]
Avoid data loss when converting between integer types [AUTOSAR-A4_7_1-i-2]
Avoid signed integer overflows [AUTOSAR-A4_7_1-j-2]
Avoid value change when converting between integer types [AUTOSAR-A4_7_1-k-2]
