Assembly language shall be encapsulated and isolated in C/C++ functions [AUTOSAR-M7_4_3-a-2]

DESCRIPTION 

"Where assambly language instructions are needed, they shall be 
encapsulated and isolated in either assembler functions or C/C++ functions."

The rule reports a violation if in one function is used assembly language 
instruction and C/C++ code. 

See also: MISRA2004-2_1
SINCE 

v9.5
NOTES 

This rule does not allow to use macros containing assembly language instructions
and C/C++ code in one function.
BENEFITS 

"Ensuring that assembly language code is encapsulated and isolated aids
portability"
EXAMPLE 

#define ASM_NOP asm ( "NOP" )
#define ASM_MOV asm ( "mov eax, 01h" )

void fn1 (int i)             /* Violation */
{
    if(i > 0)
    {
        asm ( "NOP" );      
    }
    else
    {
        asm ( "mov eax, 01h" );
    }
}

void fn2 (int i)            /* Violation */
{
    if(i > 0)
    {
        ASM_NOP;            /* assembler code in macro */
    }
    else
    {
        ASM_MOV;            /* assembler code in macro */
    }
}
REPAIR 

inline void asm_nop()       /* OK - no other code than assembler in function */
{
    asm ( "NOP" );
}

void asm_mov()              /* OK - no other code than assembler in function */
{
    asm ( "mov eax, 01h" );
}

void fn2 (int i)            /* OK */
{
    if(i > 0)
    {
        asm_nop();          /* assembler code in inline function - recommended */
    }
    else
    {
        asm_mov();          /* assembler code in function */
    }
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   M7-4-3. Assembly language shall be encapsulated and isolated

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 7, Rule 7-4-3
