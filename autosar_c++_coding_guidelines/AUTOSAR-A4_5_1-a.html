Expressions with type enum shall not be used as operands to built-in operators other than [ ], =, ==, !=, <, <=, >, >=, and the unary & operator [AUTOSAR-A4_5_1-a-2]

DESCRIPTION 

"Expressions with type enum shall not be used as operands to built-in
operators other than the subscript operator [ ], the assignment operator =,
the equality operators == and !=, the unary & operator, and the relational
operators <, <=, >, >=."
SINCE 

v7.2
BENEFITS 

"Enumerations have implementation-defined representation
 and so should not be used in arithmetic contexts."
EXAMPLE 

/* Examples of incorrect code */
enum { COLOUR_0, COLOUR_1, COLOUR_2, COLOUR_3, COLOUR_COUNT } colour;
void foo()
{
    if ( ( COLOUR_0 + COLOUR_1 ) == colour ){} // Violation
}
REPAIR 

/* Examples of correct code */
enum { COLOUR_0, COLOUR_1, COLOUR_2, COLOUR_3, COLOUR_COUNT } colour;
void foo()
{
    if ( ( COLOUR_0 < colour) && (COLOUR_3  > colour ) ){} // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A4-5-1. Expressions with type enum or enum class shall not be used as
   operands to built-in and overloaded operators other than the subscript
   operator [ ], the assignment operator =, the equality operators == and !=,
   the unary & operator, and the relational operators <, <=, >, >=

2. MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
   Chapter 6, Section 4, Rule 4-5-2
