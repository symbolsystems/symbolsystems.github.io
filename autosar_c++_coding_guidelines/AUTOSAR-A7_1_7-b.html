Multiple variable declarations shall not be allowed on the same line [AUTOSAR-A7_1_7-b-2]

DESCRIPTION 

Rule triggers if some variables are declared on the same line.
See also: FORMAT-33
NOTES 

Violation is not reported if variables are declared by macros.
BENEFITS 

Rule increases readability and prevents confusion.
EXAMPLE 

void foo( ) {
    int a, b;   // Violation
}
REPAIR 

void foo( ) {
    int a;      // OK
    int b;      // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A7-1-7. Each expression statement and identifier declaration shall be
   placed on a separate line

2. High Integrity C++ Coding Standard
   7.1.1. Declare each identifier on a separate line in a separate declaration

3. JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
   Chapter 4.19 Variables, AV Rule 152
