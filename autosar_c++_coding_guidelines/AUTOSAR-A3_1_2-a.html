Header files should have a file extension of: ".h", ".hpp" or ".hxx" [AUTOSAR-A3_1_2-a-2]

DESCRIPTION 

"This is consistent with developer expectations to provide header files with one 
of the standard file name extensions." [AUTOSAR A3-1-2]

This rule checks #include directives and reports a violation if a header file
name uses an extension other than '.h', '.hpp' or '.hxx'.
SINCE 

v10.4
NOTES 

This rule only checks header files that are included with double quotation
marks (""). Any file included using angle brackets (<>) are ignored.
BENEFITS 

This rule improves code readability, is consistent with developer expectations,
and helps you achieve compliance with some coding standards.
EXAMPLE 

// h1.cpp
int i1;

// h2.hh
extern int i2;

// file.cpp
#include "h1.cpp"                        // Violation
#include "h2.hh"                         // Violation
REPAIR 

// h1.h
int i1;

// h2.hpp
extern int i2;

// file.cpp
#include "h1.h"                          // OK
#include "h2.hpp"                        // OK

#include <iostream>                      // OK
#include <boost/test/impl/framework.ipp> // OK
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A3-1-2. Header files, that are defined locally in the project, shall have a
   file name extension of one of: ".h", ".hpp" or ".hxx"
