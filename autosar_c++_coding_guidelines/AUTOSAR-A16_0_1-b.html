#error directive shall not be used [AUTOSAR-A16_0_1-b-2]

DESCRIPTION 

"Using the pre-processor #error directive may lead to code that is complicated
and not clear for developers. The #error directive can not be applied
to templates as it will not be evaluated as a per-instance template deduction."
[AUTOSAR A16-6-1]

This rule detects '#error' pre-processor directives.

See also: PREPROC-11, PREPROC-23
SINCE 

v10.4
NOTES 

"Static assertion, similarly to #error directive, provides a compile-time error
checking. However, static_assert behaves correctly in all C++ concepts and makes
the code more readable and does not rely on pre-processor directives."
[AUTOSAR-16-6-1]
BENEFITS 

This rule helps you improve code readability.
EXAMPLE 

int i;
// ...
#if i > 0
#error "ERROR MESSAGE"                                  // Violation
#endif
REPAIR 

constexpr int i = 0;
// ...
static_assert(i == 0, "ERROR MESSAGE");                 // OK
