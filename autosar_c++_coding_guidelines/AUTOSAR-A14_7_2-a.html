Template specialization shall be declared in the same file as the primary template or a user-defined type, for which the specialization is declared [AUTOSAR-A14_7_2-a-2]

DESCRIPTION 

"Template specialization shall be declared in the same file (1) as the primary
template (2) as a user-defined type, for which the specialization is declared.

It is undefined behavior, when a compiler sees the (partial or explicit)
template specialization after it already has instantiated the primary or less
specialized template. Moreover, the case (2) allows compile-time interfaces
to be extensible, as developers can safely provide custom specializations
e.g. for traits classes or 'std::hash'." [AUTOSAR A14-7-2]

This rule detects definitions of template specializations when:
- the primary template is not declared or defined in the same file as
  the template specialization
- the template arguments do not have any user-defined type that is defined
  in the same file as the template specialization
SINCE 

v10.4
BENEFITS 

This rule helps you prevent undefined behavior and comply with some coding
standards.
EXAMPLE 

// file.h
template <typename T> void f1(T t){}
struct S { int m1; };

// file.cpp
#include "file.h"

template <> void f1<>(S t){}     // Violation - 'S' is defined in a header file
template <> void f1<S*>(S* t){}  // Violation - 'S' is defined in a header file
REPAIR 

You can fix the code by defining the template specializations in the header
file that contains the primary template and the user-defined type for which
the specialization is declared.

// file.h
template <typename T> void f1(T t){}
struct S { int m1; };

template <> void f1<>(S t){}     // OK
template <> void f1<S*>(S* t){}  // OK

// file.cpp
#include "file.h"
