Do not declare member variables as bit-fields [AUTOSAR-A9_6_2-a-2]

DESCRIPTION 

"Usage of bit-fields increases code complexity and certain aspects of bit-field
manipulation can be error prone and implementation-defined." [AUTOSAR A9-6-2]

This rule detects member variables declared as bit-fields.
BENEFITS 

This rule helps you prevent implementation-defined behavior.
EXAMPLE 

struct S
{
    int var1 : 2;             // Violation
    int : 2;                  // Violation
};
REPAIR 

Do not declare member variables as bit-fields.
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A9-6-2. Bit-fields shall be used only when interfacing to hardware or
   conforming to communication protocols

2. MISRA C++:2023 Guidelines for the use of C++17 in critical systems
   Rule 12.2.1 Bit-fields should not be declared
