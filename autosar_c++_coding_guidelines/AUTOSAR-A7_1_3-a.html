CV-qualifiers shall be placed on the right hand side of the type that is a typedef or a using name [AUTOSAR-A7_1_3-a-2]

DESCRIPTION 

"If the type is a typedef or a using name, placing const or volatile qualifier
on the left hand side may result in confusion over what part of the type the
qualification applies to." [AUTOSAR A7-1-3]

This rule detects when the 'const' or the 'volatile' qualifier is placed
on the left side of the type it applies to.
This rule checks 'typedef' and 'using' types.

See also: FORMAT-47_a
SINCE 

2020.2
BENEFITS 

This rule helps you improve code readability.
EXAMPLE 

typedef int* PINT;
void foo(const PINT p1) {    // Violation
    volatile PINT vi;        // Violation
}
REPAIR 

To fix the code, place the 'const' and 'volatile' operators on the right side
of the type.

typedef int* PINT;
void foo(PINT const p1) {    // OK
    PINT volatile vi;        // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A7-1-3. CV-qualifiers shall be placed on the right hand side of the type
   that is a typedef or a using name
