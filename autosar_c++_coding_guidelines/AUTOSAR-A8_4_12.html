AUTOSAR-A8_4_12 
AUTOSAR A8-4-12 A std::unique_ptr shall be passed to a function as: (1) a copy to express the function assumes ownership (2) an lvalue reference to express that the function replaces the managed object
RULES

Do not pass std::unique_ptr by const reference [AUTOSAR-A8_4_12-a-2]
A smart pointer shall only be used as a parameter type if it expresses lifetime semantics [AUTOSAR-A8_4_12-b-2]
A parameter should only be declared as a non-const lvalue reference to 'std::shared_ptr' or 'std::unique_ptr' if the function replaces the managed object [AUTOSAR-A8_4_12-c-2]
Do not declare the type of a parameter as an rvalue reference to 'std::shared_ptr' or 'std::unique_ptr' [AUTOSAR-A8_4_12-d-2]
