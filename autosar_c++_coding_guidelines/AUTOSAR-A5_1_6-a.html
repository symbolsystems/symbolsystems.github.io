Return type of a non-void return type lambda expression should be explicitly specified [AUTOSAR-A5_1_6-a-4]

DESCRIPTION 

"If a non-void return type lambda expression does not specify its return type,
then it may be confusing which type it returns. It leads to developers
confusion." [AUTOSAR A5-1-6]

This rule detects when the return type of a non-void return type lambda
expression is not explicitly specified.
SINCE 

v10.4
NOTES 

When the return type is explicitly specified, an implicit conversion between
the type of the returned value and the return type specified in the lambda
expression may occur. This problem should be properly handled. The compiler
may emit warning messages about implicit conversions that may not be
appropriate.
BENEFITS 

This rule helps you avoid confusion about return types of lambda expressions.
Additionally, it allows the compiler to provide warning messages about implicit
conversions, and facilitates compliance with some coding guidelines.
EXAMPLE 

void Fn() noexcept
{
    auto lambda1 = []() { return 0U; };   // Violation
}
REPAIR 

You can fix the code by explicitly specifying the return type of the lambda
expression.

void Fn() noexcept
{
    auto lambda1 = []() -> unsigned int { return 0U; };   // OK
}
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A5-1-6. Return type of non-void return type lambda expression should be
   explicitly specified.
