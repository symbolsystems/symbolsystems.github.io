Define special members as =default when the behavior is equivalent to the compiler's behavior [AUTOSAR-A12_7_1-a-2]

DESCRIPTION 

"If a class contains a user defined version of a member with the same
definition as would be provided by the compiler, then it will be less
error prone and more maintainable to replace the definition with =default."
[HIC++ v4.0, Rule 12.5.2]

This rule checks if default constructor, copy constructor, move constructor
or destructor can be defaulted.
SINCE 

2020.2
BENEFITS 

This rule helps you improve code maintainability, reduce confusion, remove
extraneous code, and comply with coding standards.
EXAMPLE 

class A {
public:
    A();                    // Violation
    A(A& rhs);              // Violation
    ~A(){ }                 // Violation

private:
    int m;
};

A::A() { }                  // Violation
A::A(A& rhs) : m(rhs.m){ }  // Violation
REPAIR 

class A {
public:
    A() = default;         // OK
    A(A& rhs) = default;   // OK
    ~A() = default;        // OK

private:
    int m;
};
REFERENCES 

1. AUTOSAR C++14 Coding Guidelines
   A12-7-1. If the behavior of a user-defined special member function is
   identical to implicitly defined special member function, then it shall be
   defined "=default" or be left undefined

2. High Integrity C++ Coding Standard
   12.5.2. Define special members =default if the behavior is equivalent
