The 'register' storage class specifier shall not be used [AUTOSAR-A7_1_4-a-2]

DESCRIPTION 

"ISO compilers are not obliged to take any notice of the 'register' storage
class specifier, so at best it is only a hint to the compiler. A good optimiser
should be able to decide for itself what to put in registers."[MISRA-C:1998]
BENEFITS 

This rule allows to avoid an unnecessary 'register' storage class specifier
that serves very little function, offering no more than a hint that is typically
ignored.
EXAMPLE 

void f1(register int p)  /* Violation */ 
{
    register int a;      /* Violation */ 
}
REPAIR 

Do not use the 'register' storage class specifier.
