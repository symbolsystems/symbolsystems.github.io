The assignment-expression of a throw statement shall not itself cause an exception to be thrown [AUTOSAR-M15_1_1-a-2]

DESCRIPTION 

"If an exception is thrown when constructing the exception object, or when
evaluating the assignment expression that initializes the exception object,
it is that exception that propagates in preference to the one that was about
to be thrown."
SINCE 

v7.2
BENEFITS 

Rule improves readability of code
EXAMPLE 

class E
{
public:
    E ( )
    {
        throw 10;
    }
};
int foo1()
{
    try
    {
        if ( 0 )
        {
            throw E ( ); // Violation
        }
    }
    catch(...)
    {
    	
    }
}
REPAIR 

class E
{
public:
    E ( ){ }
};

int foo()
{
    try
    {
        if ( 0 )
        {
            throw E ( ); // OK
        }
    }
    catch(...)
    {
    	
    }
}
