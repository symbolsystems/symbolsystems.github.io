User defined literals operators shall only perform conversion of passed parameters [AUTOSAR-A13_1_3-a-2]

DESCRIPTION 

"It is expected behavior that the user-defined literals operators are only used
to convert passed parameters to the type of declared return value. User-defined
literals are not supposed to provide any other side-effects." [AUTOSAR A13-1-3]

This rule detects when a user-defined literal operator function is not declared
as 'constexpr', or when it does not use, in a return statement, the constructor
taking a parameter if a conversion to the complex type is needed.
SINCE 

v10.4
BENEFITS 

This rule improves code readability and consistency with the C++ standard
library. This rule also helps prevent unexpected side effects.
EXAMPLE 

struct Temperature
{
    unsigned long long int kelvins;
    constexpr explicit Temperature(unsigned long long int k) : kelvins(k) {}
};

static int currentTemperature = 0;

Temperature operator"" _K(unsigned long long int kelvins)  // Violation
{
    currentTemperature = kelvins;
    return Temperature(kelvins);
}
REPAIR 

You can fix the code by removing side effects literal operator function
and declaring it as constexpr.

struct Temperature
{
    unsigned long long int kelvins;
    constexpr explicit Temperature(unsigned long long int k) : kelvins(k) {}
};

static int currentTemperature = 0;

constexpr Temperature operator"" _K(unsigned long long int kelvins)  // OK
{
    return Temperature(kelvins);
}

void setCurrentTemperature(unsigned long long int kelvins) {
    currentTemperature = kelvins;
}
