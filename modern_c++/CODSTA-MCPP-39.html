A variable of type auto shall not be initialized using '{}' or '={}' braced-initialization [CODSTA-MCPP-39-3]

DESCRIPTION 

"If an initializer of a variable of type auto is enclosed in braces, then
the result of type deduction may lead to developer confusion, as the variable
initialized using {} or ={} will always be of std::initializer_list type."
[AUTOSAR A8-5-3]

The rule detects variables of type auto that are initialized using braces.
SINCE 

v10.4
EXCEPTIONS 

The rule does not trigger when the braced initializer contains a list of
elements.

#include <initializer_list>

auto e = { 1, 2, 3 };         // OK
BENEFITS 

The rule helps you improve code readability and prevent confusion.
EXAMPLE 

#include <initializer_list>

auto i{1};                    // Violation
auto j = {2};                 // Violation
REPAIR 

To fix the code, initialize the variables without using braces.

auto i = 1;                   // OK
auto j = 2;                   // OK

1. AUTOSAR C++14 Coding Guidelines
