Use constexpr to declare const variables whenever possible [CODSTA-MCPP-11_a_cpp11-4]

DESCRIPTION 

The 'constexpr' specifier indicates that an objects is a constant whose value is
known at compile time. It should be used instead of the 'const' qualifier
whenever possible.

A variable can be declared with 'constexpr' if it has a literal type, it is
immediately initialized and every full-expression that appears in its
initializer is a constant expression.

This rule detects const variables that can be declared with the 'constexpr'
specifier.
SINCE 

v10.3
NOTES 

This rule assumes that a full-expression is not a constant expression
if it contains at least one of the following:
- a function call;
- a non-constexpr variable;
- a parameter;
- a 'new' or 'typeid' operator;
- a left shift operator (<<) with a left hand operand of signed type.

The 'constexpr' specifier for static data members should be used only when they
are initialized inside a class body.

struct Z {
    static const int z1 = 0;  // Violation
    static const int z2;
};

const int Z::z2 = 0; // OK - the definition outside the class body
BENEFITS 

This rule helps you ensure that 'constexpr' is used whenever possible.
'constexpr' variables are stored in the read-only memory, which prevents
exploits and improves security.

Also, they can be employed in a wider range of contexts. For example, you can
use variables declared with 'constexpr' when the C++ Standard requires integral
constant expressions, such as array sizes, integral template arguments,
enumerator values, alignment specifiers and more.
EXAMPLE 

const int a = 1;              // Violation
const char b[] = "test";      // Violation

void foo()
{
  static const int m = 0;     // Violation
}
REPAIR 

constexpr int a = 1;           // OK
constexpr char b[] = "test";   // OK

void foo()
{
  static constexpr int m = 0;  // OK
}
