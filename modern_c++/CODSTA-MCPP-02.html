Prefer alias declarations to typedefs [CODSTA-MCPP-02-3]

DESCRIPTION 

Alias declarations are similar to typedefs, but they have the advantage of
working with templates, and they can improve the clarity of code. For this
reason, alias declarations should be preferred to typedef.

This rule detects user-defined types declared through typedef declarations.
SINCE 

v9.6
BENEFITS 

Alias declarations help you keep your code clear. Their syntax separates
the type name from the type, which makes complex types (such as a pointer to
a function) easier to read. In addition, alias declarations can be templatized,
unlike typedefs.
EXAMPLE 

typedef char CHAR;                                    // Violation
typedef char* PCHAR;                                  // Violation

template<typename T>
struct MyAllocList
{
    typedef std::list<T, std::allocator<T>> type;     // Violation
};
REPAIR 

using CHAR = char;                                    // OK
using PCHAR = char*;                                  // OK

template<typename T>
struct MyAllocList
{
    using type = std::list<T, std::allocator<T>>;     // OK
};
