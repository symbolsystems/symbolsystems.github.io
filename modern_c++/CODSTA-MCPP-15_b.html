Use the 'this' pointer explicitly in lambdas with a default by-reference capture [CODSTA-MCPP-15_b-3]

DESCRIPTION 

A by-reference capture [&] results in a closure that contains a reference to
a local variable or a parameter available in the scope where the lambda is
defined. It also captures the current object (if it exists) by reference.
If there is an identifier (a variable or a function) that has the same name
as the class member and can be used in a lambda body, the lambda code becomes
ambiguous. To avoid ambiguity, use 'this->' each time you want to refer to the
class member.
SINCE 

v10.3
BENEFITS 

This rule helps you keep your code clear and avoid mistakes when accessing
class members.
EXAMPLE 

class A
{
private:
    int z;
    void f();
    void f2(){};
};

void A::f()
{
    z = 0;
    [&]{
        f2();      // Violation
        return z;  // Violation
    };
}
REPAIR 

class A
{
private:
    int z;
    void f();
    void f2(){};
};

void A::f()
{
    z = 0;
    [&]{
        this->f2();      // OK
        return this->z;  // OK
    };
}
